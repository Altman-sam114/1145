# v3.0（反潜HUD信息）

## 版本号

v3.0

## 版本分配依据

- 人工未指定本轮版本；`update_log.md` 和 `md/prompt/` 当前最新正式玩法版本为 v2.9“旗点占领奖励”。
- v2 阶段主要围绕地图控制、旗点、侦测建筑、防空和 AI 争夺展开；反潜 HUD 信息是多轮遗留扩展点，属于海战 / 反潜可读性增强。
- 本轮开启一个小型 v3.0 海战反潜阶段，只做 HUD 信息展示，不改变反潜规则或数值。

## 背景

当前项目已有潜艇隐身、`revealedUntil`、声呐检测、Sonar Buoy、Battleship / Carrier / Helicopter / Submarine 的 sonar 能力，以及 `AIRS` / `BARR` 命中潜艇短暂暴露。但选择面板对声呐信息展示有限：Sonar Buoy 只显示 “Sonar detector No SCAN”，潜艇状态只显示 “Stealth Sonar yes” 或 detected 秒数，多选时不显示编队里有多少 sonar asset。

v3.0 的目标是让玩家在不改变玩法规则的情况下，更清楚地看见当前选择的反潜能力和潜艇状态。

## 目标

1. 单体选择面板显示 active sonar range。
   - 对已完工 Sonar Buoy、Battleship、Carrier、Helicopter、Submarine 显示 `Sonar <range>`。
   - 未完工 Sonar Buoy 不显示 active sonar range，继续显示 offline 状态。
   - 不改变 `EntityKind.hasSonar` 或 `sonarRange(for:)` 数值。
2. 潜艇单体状态更清楚。
   - 选中 Submarine 时，状态行应显示当前是 stealth 还是 temporary detected，并保留 detected 剩余秒数。
   - Sonar range 应通过统一 sonar 信息展示，而不是含糊的 “Sonar yes”。
3. 多选面板显示 sonar asset 概况。
   - 多选中存在 active sonar asset 时，在编队信息中显示 sonar asset 数量和最大 sonar range。
   - 不覆盖 hold / attack-move 状态优先级；可放在 combat summary 行，避免新增 HUD 行数。
4. Sonar Buoy 结构状态展示 range。
   - Operational Sonar Buoy 的结构状态从泛化描述改为显示 `Sonar <range>  No SCAN`。
   - 未完工结构仍显示 `Offline until operational`。
5. 保持全部反潜规则不变。
   - 不改变 `isSubmarineDetected(...)`、`revealedUntil`、`SCAN`、`AIRS` / `BARR` 命中暴露、声呐范围、潜艇可见性、AI 或战斗。
   - 不新增按钮、图标、外部素材或调试 HUD。

## 非目标

- 不新增反潜武器、反潜单位、科技树、升级或支援技能。
- 不改变 Sonar Buoy 放置规则、是否作为 `SCAN` 资产、是否攻击目标或是否赚钱。
- 不改变潜艇隐身、开火暴露、支援命中暴露、声呐检测、迷雾合法性或目标选择。
- 不改变 HUD 布局行数、按钮、命令输入或控制组逻辑。
- 不修改 Xcode project、CI workflow 或测试规范，除非编译必须且能说明原因。
- 按人工最新要求，本轮不得运行本地测试、本地构建或本地静态检查作为验证；验收只认 GitHub Actions artifact。

## 实现步骤

### 1. 开始前同步 main 并确认工作区归属

Agent B 必须先执行：

```sh
git fetch origin
git switch main
git pull --ff-only origin main
git status --short
git rev-parse HEAD
git rev-parse origin/main
```

要求：

- 当前分支必须是 `main`。
- `HEAD` 和 `origin/main` 必须是同步后的最新基线。
- 如果 `git status --short` 显示任何未提交、未跟踪或冲突文件，B 必须停止并确认归属。
- 本提示词文件 `md/prompt/v3（海战反潜）/v3.0（反潜HUD信息）.md` 属于本轮 Agent A 产物。

### 2. 增加 sonar HUD helper

建议新增：

```swift
private func isActiveSonarSensor(_ entity: GameEntity) -> Bool
private func sonarInfoLine(for entity: GameEntity) -> String?
private func groupSonarSummary(for selected: [GameEntity]) -> String?
```

要求：

- active sonar 必须满足 `kind.hasSonar`，并且若为结构则 `isOperational`。
- sonar range 使用既有 `sonarRange(for:)`。
- helper 只用于 HUD 文案，不改变检测逻辑。

### 3. 单体选择信息

- 在 `singleSelectionInfo(for:)` 的 stats 行追加 `Sonar <range>`，但只对 active sonar sensor 追加。
- `mobileStatusLine(for:)` 中 submarine 状态改为更清晰的 stealth / detected 文案，不再写 “Sonar yes”。
- `structureStatusLine(for:)` 中 Sonar Buoy 改为 `Sonar <range>  No SCAN`。

### 4. 多选信息

- 在 `groupSelectionInfo(for:)` 的 combat summary 行追加 sonar 摘要，例如 `Sonar 2/300` 表示 2 个 active sonar asset、最大 sonar range 300。
- 若没有 active sonar asset，则保持原文案。
- 不覆盖 hold / attack-move / veterancy 行。

### 5. 文档更新

- README 当前功能列表同步 selection panel 会显示 sonar range、submarine stealth/detected 状态和 group sonar summary。
- `md/flow/flow.md` HUD / 小地图小节同步选择面板的反潜信息展示来源。
- `md/flow/flowchart.md` HUD 节点可加入“反潜信息”。
- 不更新 `update_log.md`。正式 v3.0 记录由 Agent C 在最新 `origin/main` artifact 验收通过后追加。
