# v3.9（AI低血单位撤退回修）

## 版本号

v3.9

## 版本分配依据

- 当前 `md/prompt/` 与 `update_log.md` 最新海战 / AI 阶段为 v3.8「AI高价值老兵主攻保护」。
- v3.6-v3.8 已经让 Red routine attack-move 波次具备高价值海军护航、混编主攻和低血 Veteran / Elite 保护。
- v3.9 继续做一个小而可验证的 AI 战术增量：让 Red 低血机动战斗单位在 routine 主攻中主动脱离并回到维修 / 后撤锚点，恢复后再交回现有波次系统。

## 背景

当前 `GameScene.swift` 中 Red AI 的 routine assault 入口是 `commandEnemyAttackers(_:)`。该函数会筛选空闲作战单位，生成 mixed provisional wave，再通过 `enemyAssaultWave(from:)` / `canJoinEnemyAssaultWave(...)` 执行低血老兵保护和高价值海军护航过滤，最后调用共享 `issueFormationMove(... attackMove: true)`。

v3.8 的低血保护只覆盖 Veteran / Elite，并且只是让它们不进入 routine wave。普通 Recruit / Hardened 单位低血后仍可能继续被拉入主攻或保持前线攻击状态，缺少基础 RTS 常见的撤退回修行为。

本轮只在 Red routine assault 决策边界做最小改动：低血 Red 机动战斗单位若属于本轮 routine assault 候选，则清理主攻 / 攻击意图并后撤到最近己方 Mechanic 附近；没有 Mechanic 时退回 Red 基地锚点。现有 `updateRepair(dt:)` 和 `REPR` 支援负责实际回血，本轮不改维修数值。

## 目标

1. Red low-health routine assault 候选会脱离主攻并后撤。
   - 适用对象：Red、存活、非结构、operational、有伤害、非占点 reservation 的机动战斗单位。
   - HP 比例低于撤退阈值时触发，建议阈值 `0.38`。
   - 撤退对象覆盖 Recruit / Hardened / Veteran / Elite；v3.8 低血 Veteran / Elite 仍继续受主攻过滤保护。
2. 撤退单位必须清理当前 routine attack 意图。
   - 清理 `attackMoveDestination`。
   - 清理 `attackTarget`。
   - 可把 `holdPosition` 设置为撤退 / 维修锚点，复用既有 hold 回位语义避免低血单位立刻追击回前线；恢复到安全阈值后必须清理该 hold 点。
   - 通过 `setDestination(for:near:)` 后撤。
3. 撤退目标选择必须复用现有地图 / domain 移动规则。
   - 优先选择最近的己方存活 Mechanic 附近，让既有 `updateRepair(dt:)` 可自然维修。
   - 没有 Mechanic 时退到 `enemyBaseAnchorPoint()`。
   - 不直接生成单位，不瞬移，不绕过 `setDestination(for:near:)` / `navigablePoint(...)`。
4. 恢复后自然回到既有 AI 波次系统。
   - 不新增复杂状态机。
   - 允许使用一个小型 `enemyRetreatingUnitIDs` 集合跟踪正在回修的 Red 单位，但死亡、恢复和 `SKRM` 必须清理，不能变成长期 reservation。
   - HP 高于安全阈值后，单位若重新空闲，可被后续 `commandEnemyAttackers(_:)` 按既有混编 / 护航 / 老兵保护规则重新使用。
5. Red `REPR` 支援评分可轻量偏向低血撤退候选。
   - 只允许加评分，不改实际维修量、费用、冷却、半径、资产需求或玩家手动 `REPR` 行为。
6. 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`。
7. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

- 不改 `issueFormationMove(...)`、`issueAttackMoveOrder(...)`、`updateMovement(...)`、`updateCombat(...)`、`setDestination(...)`、`navigablePoint(...)`、`updateRepair(...)` 或 `applySupportRepair(...)`。
- 不改 Mechanic 维修半径 / 维修量，不改 `SupportPower.fieldRepair` 数值 / 费用 / 冷却 / 资产需求。
- 不改 AI 生产、动态防空、动态 ASW、`SCAN`、`AIRS`、`BARR`、经济、建筑补建、旗点 reservation、旗点防守响应或基地防守响应。
- 不改玩家 `AMOV`、普通移动、攻击命令、控制组、选择、HUD、迷雾、声呐、潜艇暴露、XP / 老兵加成、单位数值或 Xcode / workflow 配置。
- 不新增外部依赖、素材、XCTest target、模拟器脚本或平行 AI 系统。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。

2. 在 `GameScene.swift` 中增加小范围 helper / 常量。

建议新增：

- `private let enemyRetreatHealthThreshold: CGFloat = 0.38`
- `private let enemyRetreatRecoveryThreshold: CGFloat = 0.62`
- `private let enemyRepairAnchorOffset: CGFloat = 72`
- `private var enemyRetreatingUnitIDs = Set<Int>()`
- `private func updateEnemyRetreatingUnits(_ enemyUnits: [GameEntity])`
- `private func shouldRetreatEnemyAssaultUnit(_ unit: GameEntity) -> Bool`
- `private func retreatEnemyAssaultUnits(_ units: [GameEntity]) -> Set<Int>`
- `private func retreatDestination(for unit: GameEntity) -> CGPoint`
- `private func enemyRepairAnchor(for unit: GameEntity) -> CGPoint?`

要求：

- `shouldRetreatEnemyAssaultUnit` 必须要求 Red、存活、非结构、operational、有伤害、非占点 reservation。
- 使用 `healthRatio(of:) < enemyRetreatHealthThreshold` 判定撤退。
- 恢复阈值必须只用于避免刚恢复到低阈值附近反复进入撤退；`SKRM`、死亡、恢复和失效单位必须清理 `enemyRetreatingUnitIDs`。
- `enemyRepairAnchor(for:)` 只选择 Red 存活 Mechanic，不要求读取玩家信息。
- fallback 必须是 `enemyBaseAnchorPoint()`。

3. 改造 `commandEnemyAttackers(_:)` 的 objective wave 分支。

推荐顺序：

- 计算 `provisionalWave`。
- 从 `provisionalWave` 中找出需要撤退的单位。
- 对这些单位执行后撤，得到 `retreatingIDs`。
- 最终 `wave` 从 `enemyAssaultWave(from:)` 的结果中过滤 `retreatingIDs`。
- 若最终 `wave` 为空，return。
- 其余单位继续调用 `issueFormationMove(to:objective, units:wave, ..., attackMove:true)`。

注意：

- 改派撤退必须在 `commandEnemyAttackers(_:)` 中完成，不要在 filter 闭包里产生副作用。
- 被撤退单位不要再被本轮 `issueFormationMove` 覆盖目的地。
- 普通非低血单位、混编比例、高价值海军护航和低血 Veteran / Elite 过滤必须保持原语义。

4. 可轻量更新 `supportRepairScore(...)`。

允许增加：

- 若 `faction == .enemy && shouldRetreatEnemyAssaultUnit(entity)`，给 `REPR` 评分增加战略价值偏向。

禁止：

- 改 `applySupportRepair(...)`。
- 改 `SupportPower.fieldRepair`。
- 改玩家 `REPR`。
- 改 cooldown、cost、radius、repair amount 或 asset requirement。

- `md/flow/flow.md`：AI 小节同步低血撤退 / 回修行为，说明不改维修数值、不影响玩家命令和占点 reservation。

- 不更新 `md/test/test.md`。
- 不更新 `update_log.md`，由 Agent C 验收通过后追加 v3.9 正式记录。

6. 最后可用 `git diff`、`git diff --name-only`、`git diff --stat` 自查范围；这些只用于范围自查，不能写成验收结果。

## 验收标准

- Red routine assault provisional wave 中低血机动战斗单位会退出本轮主攻。
- 撤退单位会清理 `attackMoveDestination` 和 `attackTarget`，通过 `setDestination(for:near:)` 后撤，并可临时使用 `holdPosition` 锚定维修点；恢复后清理该 retreat hold 点。
- 有 Red Mechanic 时优先后撤到 Mechanic 附近；没有 Mechanic 时后撤到 `enemyBaseAnchorPoint()`。
- 撤退单位不会在同一轮被 `issueFormationMove(... attackMove: true)` 覆盖目的地。
- HP 恢复后，单位可作为空闲作战单位被后续 routine wave 重新使用。
- v3.6 Battleship / Carrier 护航门槛、v3.7 mixed provisional wave、v3.8 低血 Veteran / Elite 保护继续生效。
- 占点 reservation、防守响应、AI 生产、动态 AA / ASW、支援技能数值、维修数值、玩家命令、寻路、战斗、迷雾、声呐、单位数值和 Xcode / workflow 配置无变化。
- README、flow、flowchart 与真实行为一致。
- 本轮提交已 push 到 `origin/main` 并触发 GitHub Actions。
- Agent C 能下载最新 run artifact 并核对 manifest / JUnit / xcodebuild.log / failure summary / xcresult。

## 风险和禁止项

- 风险：撤退单位被本轮 `issueFormationMove` 覆盖，导致没有真正退出主攻。
- 风险：撤退状态和 `updateMovement` / `updateCombat` 自动追击互相抢状态，造成单位抖动。
- 风险：误伤占点 reservation 或基地 / 旗点防守响应。
- 风险：把 Mechanic 维修、`REPR` 数值或玩家命令一起改了，扩大范围。
- 禁止新建平行 AI 状态机；必须复用现有 `commandEnemyAttackers` / `setDestination` / `updateRepair` 链路。
- 禁止修改玩家 `AMOV` 或任何玩家输入语义。
