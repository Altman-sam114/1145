# v3.4（声呐覆盖选择反馈）

## 版本号

v3.4

## 版本分配依据

- 当前 `md/prompt/` 和 `update_log.md` 最新海战反潜阶段为 v3.3「反潜命中战场反馈」。
- v3.0-v3.3 已补 HUD 反潜信息、AI 已知潜艇压力补 ASW、ASW 命中反馈。
- v3.4 继续同一阶段，只做 visual-only 选择反馈，不改变反潜规则。

## 背景

当前 `GameScene.swift` 已有 `EntityKind.hasSonar`、`sonarRange(for:)`、`isActiveSonarSensor(_:)`、`isSubmarineDetected(...)` 和选择面板 sonar/ASW 文案。玩家选择 active sonar sensor 时，HUD 能看到 range，但地图上没有覆盖圈反馈。

## 目标

1. 选择玩家 active sonar sensor 时显示 sonar range 覆盖圈。
   - 适用：玩家 Submarine、Battleship、Carrier、Helicopter、operational Sonar Buoy。
   - 覆盖圈尺寸必须由既有 `sonarRange(for:)` 推导，不硬编码第二份 range。
2. 多选时显示多个覆盖圈。
   - 只对当前 `selectedIDs` 内满足条件的玩家 active sonar sensor 显示。
   - 不新增结构多选机制，只复用现有选择状态。
3. 未完工 Sonar Buoy 不显示覆盖圈。
   - 若选中施工中的 Sonar Buoy，完工前无圈；完工后应能刷新出圈。
4. 不显示敌方 sonar range。
5. 本轮 strictly visual-only。
   - 不改变 `sonarRange`、`isSubmarineDetected`、`revealedUntil`、fog、AI、damage、HUD buttons、targeting、support powers 或单位数值。
6. 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`。
7. 不更新 `update_log.md`，由 Agent C 在云端 artifact 通过后补正式记录。

## 非目标

- 不新增反潜单位、武器、科技、升级、按钮、支援技能、音效或外部素材。
- 不改变 Sonar Buoy 放置、施工、视野、是否 `SCAN` 资产、是否攻击目标或是否赚钱。
- 不改变潜艇隐身、开火暴露、支援命中暴露、玩家 / AI 可见性或迷雾合法性。
- 不改变选择机制、框选规则、控制组、HUD button layout 或 HUD button 高亮。
- 不修改 Xcode project、CI workflow 或 `md/test/test.md`。

## 实现步骤

1. 开始前同步 `main`：

```sh
git fetch origin
git switch main
git pull --ff-only origin main
git status --short
git rev-parse HEAD
git rev-parse origin/main
```

要求：当前分支必须是 `main`，目标远端为 `origin/main`，工作区不能混入无关改动。

2. 增加选择态 sonar 覆盖视觉。
   - 可在 `GameEntity` 增加 `sonarCoverageNode`，作为实体子节点，随移动单位自然移动。
   - 覆盖圈必须位于 `fogLayer` 下方，不画穿黑雾。
   - 覆盖圈只作为 world-space visual，不参与点击和实体逻辑。
3. 增加或复用 helper：
   - `private func shouldShowSonarCoverage(for entity: GameEntity) -> Bool`
   - `private func configureSonarCoverageNode(for entity: GameEntity)`
4. 覆盖圈显隐规则：
   - 必须要求 `selectedIDs.contains(entity.id)`。
   - 必须要求 `entity.faction == .player`。
   - 必须复用 `isActiveSonarSensor(entity)`，不要复制 operational 判断。
   - 覆盖圈尺寸必须来自 `sonarRange(for: entity.kind)`。
   - 使用半透明 cyan / blue stroke / fill，低透明度，不遮挡单位。
   - 不给覆盖节点设置 `entityID` / `userData`，不改变 `entity(at:)`。
5. 同步触发点：
   - `refreshSelection()` 更新覆盖圈显隐。
   - `updateConstruction(dt:)` 中玩家结构完工后同步一次，确保选中 Sonar Buoy 完工后显示圈。
   - `cullDestroyedEntities()`、取消选择、`SKRM` 重开继续复用现有 `refreshSelection()` / layer 清理，不残留旧圈。
6. 更新文档：
   - `README.md`：说明选择玩家 active sonar sensor 会显示 sonar range 覆盖圈，多选显示多个，未完工 Sonar Buoy 和敌方 sonar 不显示。
   - `md/flow/flow.md`：说明覆盖圈由选择状态和 `sonarRange(for:)` 派生，是 visual-only，不改变 detection / fog。
   - `md/flow/flowchart.md`：HUD 或 Render 节点同步加入选择态 sonar 覆盖圈描述。
   - 不更新 `update_log.md`。

- 风险：把覆盖圈画在 fog 之上，会在未探索或黑雾区域提供额外可读信息。

- 风险：误给敌方 sensor 显示覆盖圈会暴露敌方能力与位置。
- 禁止修改 `sonarRange(for:)`、`isSubmarineDetected(...)`、`isKnownToFaction(...)`、`revealedUntil`、`visibleTiles`、`exploredTiles`、`supportRevealTiles`、AI、攻击规则、单位数值或支援技能。
