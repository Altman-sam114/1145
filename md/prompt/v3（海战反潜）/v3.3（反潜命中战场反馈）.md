# v3.3（反潜命中战场反馈）

## 版本号

v3.3

## 版本分配依据

- 当前 `update_log.md` 和 `md/prompt/` 最新海战反潜阶段为 v3.2「AI 已知潜艇压力补 ASW」。
- v3.0 / v3.1 已补足反潜 HUD 可读性，v3.2 已补足 Red 已知潜艇压力下的 ASW 生产响应。
- v3.3 继续同一阶段，只做 direct-fire 命中潜艇时的 visual-only 战场反馈，不改变潜艇侦测、暴露、伤害、AI 或支援技能规则。

## 背景

当前 `fire(attacker:target:)` 负责直接火力伤害、潜艇开火短暂暴露、投射物视觉、armor multiplier、血条、under-attack、AI 防守触发、击杀 XP 和爆炸。项目已有 `showProjectile(...)`、`showSonarPing(at:faction:)`、`explode(...)` 等挂在 `effectsLayer` 的短生命周期特效。玩家已经能从 HUD 看见 ASW attack 与 sonar sensor，但直接火力命中玩家已知或可见潜艇时缺少明确命中反馈。

## 目标

1. 直接火力命中 `target.kind == .submarine` 时，在目标位置显示短暂 `ASW HIT` 和水下冲击圈。
2. 反馈必须遵守玩家可见性边界：
   - 玩家潜艇被命中时可以显示。
   - 敌方潜艇只有 `isKnownToFaction(target, observer: .player)` 为 true 时才显示。
   - 不得通过特效泄露隐藏敌方潜艇位置。
3. 新增 `shouldShowAntiSubmarineHitFeedback(for:)` 和 `showAntiSubmarineHit(at:faction:)` 这类 helper，保持视觉逻辑集中。
4. 只接入 `fire(attacker:target:)` direct-fire 路径。

## 非目标

- 不改变 `fire` 的伤害公式、`armorMultiplier`、击杀 / XP、under-attack alert 或 enemy defense trigger。
- 不改变 `EntityKind.canAttack(_:)`、`EntityKind.hasSonar`、`sonarRange(for:)`、`isSubmarineDetected(...)`、`revealedUntil`。
- 不改变 `SCAN`、`AIRS`、`BARR`、`applySupportDamage(...)` 或 `revealSubmarineHitBySupport(...)`。
- 不改变 AI、目标搜索、迷雾、单位数值、建筑规则、HUD 按钮或选择面板。
- 不新增外部素材、音效、依赖、测试 target、Xcode project 或 CI workflow 改动。
- 不更新 `update_log.md`；由 Agent C 在 artifact 验收通过后补正式记录。
- 按人工要求，本轮不得运行本地测试、本地构建或本地静态检查作为验证。

2. 在 `fire(attacker:target:)` 中扣血前计算：

```swift
let showASWHit = shouldShowAntiSubmarineHitFeedback(for: target)
```

扣血和 `updateHealthBar(target)` 后，如 `showASWHit` 为 true，调用：

```swift
showAntiSubmarineHit(at: target.node.position, faction: attacker.faction)
```

不得改变 damage、armor、kill / XP 或爆炸顺序。

3. 新增可见性 guard：

```swift
private func shouldShowAntiSubmarineHitFeedback(for target: GameEntity) -> Bool {
    guard target.kind == .submarine else { return false }
    return target.faction == .player || isKnownToFaction(target, observer: .player)
}
```

4. 新增 `showAntiSubmarineHit(at:faction:)`：
   - 使用 `SKShapeNode` 椭圆环 / 冲击圈和 `SKLabelNode(fontNamed: "Menlo-Bold")`。
   - label 文案为 `ASW HIT`。
   - 使用 `effectsLayer.addChild(...)`。
   - 使用 `SKAction.sequence` / `group` 淡出、缩放、移动后 `removeFromParent()`。
   - 不修改实体状态、迷雾、HUD message 或 minimap。

5. 更新文档：
   - `README.md` 当前功能中补充 direct-fire 命中玩家已知 / 可见潜艇时显示 ASW HIT / underwater shock-ring feedback。
   - `md/flow/flow.md` 的移动与战斗 / 迷雾章节同步：反馈 visual-only，只在玩家潜艇或玩家已知敌潜艇上显示，不改变潜艇检测、暴露或伤害。
   - `md/flow/flowchart.md` 的 Combat 节点同步短描述。
   - 不更新 `md/test/test.md` 或 `update_log.md`。

## 验收标准

- 玩家潜艇被敌方 direct-fire 命中时，玩家可见 `ASW HIT` / 水下冲击圈反馈。
- 玩家 direct-fire 命中已满足 `isKnownToFaction(... observer: .player)` 的敌方潜艇时，玩家可见反馈。
- 命中对玩家未知 / 不可见的敌方潜艇时，不显示任何会泄露位置的反馈。
- `AIRS` / `BARR` 命中潜艇不会新增 `ASW HIT` 反馈。
- `fire` 的 damage、`armorMultiplier`、kill / XP、`canAttack`、sonar、`revealedUntil`、AI、迷雾和单位数值没有行为改动。
- 特效使用 `effectsLayer`，短生命周期自动移除，不保存新持久状态。
- README、flow、flowchart 与代码真实行为一致。
- 最新 `origin/main` GitHub Actions artifact 通过，且 Agent C 核对 run / commit / manifest 一致。

- 风险：在可见性判断前或无 guard 地显示 target 位置，会泄露隐藏敌方潜艇。
- 风险：把 `ASW HIT` 接入 `applySupportDamage(...)` 会扩大到支援技能，超出本轮范围。
- 风险：误改 `armorMultiplier`、`revealedUntil` 或 `isSubmarineDetected(...)` 会改变反潜平衡。
- 禁止修改伤害公式、攻击合法性、声呐能力、声呐范围、潜艇检测、支援暴露、AI 逻辑或单位数值。
