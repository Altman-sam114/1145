# v3.1（反潜攻击HUD信息）

## 版本号

v3.1

- v3.0 已显示 active sonar range 和多选 sonar summary；v3.1 继续做同一阶段的小型 HUD-only 可读性修正。
- 本轮目标是区分“能反潜攻击”和“有 sonar sensor”，不改变反潜规则或数值。

## 背景

当前代码中 `EntityKind.canAttack(.submarine)` 允许 Submarine、Battleship、Helicopter、Fighter、Carrier 攻击潜艇；`EntityKind.hasSonar` 只包含 Battleship、Submarine、Carrier、Helicopter、Sonar Buoy，不包含 Fighter。因此 Fighter 能反潜攻击但没有 sonar，需要其他 sonar、`SCAN` 或临时暴露链路才能看见潜艇。v3.0 HUD 已显示 sonar range，但还没有把 ASW attack capability 与 sonar sensor 明确区分。

## 目标

1. 单选 HUD 明确显示 ASW attack capability。
   - Fighter 显示 `ASW attack  No sonar` 或等价短文案。
   - Battleship / Submarine / Carrier / Helicopter 同时显示 `ASW attack` 和 `Sonar <range>`。
   - Sonar Buoy 只显示 sonar detector 信息，不显示 `ASW attack`。
2. 多选 HUD 在 combat summary 行显示 ASW count 和 sonar count / max range。
   - 推荐格式：`ASW <count>  Sonar <count>/<maxRange>`。
   - Fighter 计入 ASW count，不计入 sonar count。
   - Battleship / Submarine / Carrier / Helicopter 同时计入 ASW count 和 sonar count。
   - active Sonar Buoy 计入 sonar count，不计入 ASW count。
3. 保持选择面板 4 行内容，不覆盖 hold / attack-move / veterancy 行。
4. 本轮只改 HUD 文案、helper 和必要文档。

## 非目标

- 不改变 `canAttack(_:)`、`hasSonar`、`sonarRange(for:)`、`isSubmarineDetected(...)` 或 `revealedUntil`。
- 不改变 `SCAN`、`AIRS`、`BARR`、AI、战斗、迷雾、目标搜索、伤害、冷却、单位数值或建筑规则。
- 不新增按钮、图标、外部素材、测试 target、debug HUD 或新玩法。
- 不修改 Xcode project、CI workflow 或 `md/test/test.md`。
- 按人工最新要求，本轮不得运行本地测试、本地构建或本地静态检查作为验证。

   - active ASW attacker 显示 `ASW attack`。

4. 调整多选 HUD：
   - combat summary 行追加 `ASW <count>  Sonar <count>/<maxRange>`。
   - 无 ASW 且无 active sonar 时保持原文案。
