## 版本分配依据

## 背景

项目当前已有 Agent A / Agent B / Agent C 三角色流程：Agent A 写版本化提示词，Agent B 在 `main` 上实现并 push，Agent C 下载 GitHub Actions artifact 并复核 manifest、JUnit、日志和结果包。

后续需要新增 Agent X 作为主控调度角色。Agent X 接收人工总目标 X，将其拆分为多个小轮次，并在每轮完成 Agent A -> Agent B -> Agent C 后判断继续、退回、暂停或完成。

## 目标

1. 在 `AGENTS.md` 增加 Agent X 的召唤规则、职责、循环边界、停止条件、禁止项和最终回复格式。
2. 在 `md/flow/flow.md` 增加 Agent X 主控循环说明。
3. 在 `md/flow/flowchart.md` 更新 Mermaid 图，表达人工总目标、Agent X 拆分、Agent A 提示词、Agent B 实现 push、GitHub Actions artifact、Agent C 验收和 Agent X 判断。
4. 在 `md/test/test.md` 增加 Agent X 循环下的验证规则和小数据 artifact 下载容量限制。
5. 在 `md/prompt/README.md` 增加 Agent X 轮次提示词管理规则。
6. 在 `README.md` 简短补充 `agentx:` 协作入口。
7. 在 `update_log.md` 增加 v0.5 记录。

## 非目标

## 验收标准

- `agentx` / `x:` / `X:` 召唤规则清楚。
- Agent X 职责明确为主控调度，不替代 A/B/C。
- Agent X 循环必须等待 Agent C artifact 验收。
- 停止条件覆盖总目标完成、连续阻塞、连续无有效 diff、同因 CI 连续失败、权限/账号/密钥/付费服务/人工决策、冲突归属不明和用户停止。
- Mermaid 图能读出人工总目标 -> Agent X -> Agent A -> Agent B -> GitHub Actions -> artifact -> Agent C -> Agent X 判断。
- `git diff --check` 通过。
