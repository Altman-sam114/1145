# v1.3（双击同类选择）

## 版本号

v1.3

## 版本分配依据

- `update_log.md` 当前最新核心玩法版本是 v1.2，Guard Tower 已由 Agent C 通过 `origin/main` 最新 GitHub Actions artifact 验收闭环。
- 本轮是在 v1.2 后的小型操作体验增量，不新增单位、建筑、科技树、AI 战术或大 UI。
- `md/flow/flow.md` 的未来扩展点列出“操作体验：双击同类选择、控制编队、命令按钮高亮、非法命令反馈”。本轮只实现其中最小可验收切片：双击同类选择。

## 背景

`Desert Frontline` 当前已经覆盖经济、建造、Radar Outpost、Guard Tower、陆海空单位、海军、航母、AI、地图、战争迷雾、老兵经验、支援技能、HUD、任务阶段和 HQ 胜负条件。核心输入、选择、命令和反馈都集中在 `DesertFrontline/GameScene.swift`。

现有玩家操作包含单击选择、拖框选择、`ARMY` 全军选择、普通移动、攻击、`HOLD`、`AMOV`、集结点、建筑放置、支援技能、小地图和双指平移缩放。玩家已经能框选多个单位，但缺少 RTS 常见的“双击一个单位，选择屏幕内同类单位”的快速操作。本轮只补这个输入体验，不改命令系统和 HUD 布局。

## 目标

1. 玩家双击一个己方机动单位时，选择当前摄像机视野内同 `kind` 的所有己方存活机动单位。
2. “机动单位”定义沿用现有边界：`faction == .player`、`isAlive == true`、`!kind.isStructure`。
3. 双击只对玩家己方机动单位生效：
   - 不选择建筑。
   - 不选择敌方或中立实体。
   - 不选择已死亡实体。
   - 不绕过战争迷雾或敌方可见性规则。
4. 当前摄像机视野应按 `cameraRig.position`、`cameraRig.xScale/yScale` 和 `size` 计算世界空间可见矩形；可适当增加少量边缘容差，但不能退化成全地图同类选择。
5. 如果屏幕内没有其他同类单位，也保持选中当前单位，并显示简短反馈。
6. 不影响现有行为：
   - 单击选择己方实体。
   - 框选玩家机动单位。
   - 点击可见敌方下达攻击命令。
   - 点击地面移动。
   - HUD 按钮。
   - 小地图点击/拖动。
   - 双指平移缩放。
   - pending 建筑、支援技能、集结点、`AMOV` 模式。
7. 优先使用 `UITouch.tapCount` 或现有触摸状态做最小改动；不要新增复杂输入系统。
8. 反馈文案要短，类似：
   - `Selected 3 Tanks.`
   - `Selected 1 Tank.`
   - 或 `No other Tanks in view.`
9. 更新 `README.md` Controls，更新 `md/flow/flow.md` 输入流程；必要时更新 `md/flow/flowchart.md` 的玩家输入图。
10. 不更新 `update_log.md`。正式 v1.3 记录由 Agent C 在最新 artifact 验收通过后追加。

关键源码事实：

- SwiftUI 宿主很薄：`DesertFrontlineApp` 创建 `GameView`，`GameView` 用 `SpriteView` 承载 `SceneHolder.scene`，实际输入处理在 `GameScene`。
- `SceneHolder` 创建固定尺寸 `GameScene(size: 1366x1024)`，`scaleMode = .aspectFill`。
- `GameScene` 使用 `cameraRig` 作为 `SKCameraNode`，HUD 挂在 `cameraRig` 下，世界内容挂在 `worldNode` 下。
- 当前选择状态是 `selectedIDs: Set<Int>`。
- `refreshSelection()` 依据 `selectedIDs` 控制实体选择圈和 rally 标记显示。
- `updateHUD()` 每帧根据 `selectedIDs` 汇总 `selectedLabel` 和选择信息面板。
- `selectedMobilePlayerUnits()` 已统一返回玩家己方存活非结构单位，是移动、攻击、`HOLD`、`AMOV` 的主要选择边界。
- `entity(at:)` 会跳过死亡实体；敌方实体必须满足 `isKnownToFaction(..., observer: .player)` 才能被玩家点到；己方和中立实体没有迷雾过滤。
- 触摸入口：
  - `touchesBegan` 先处理双指、HUD、小地图，再记录 `touchStartScene`、`touchStartWorld`、`touchStartedOnEntity`、`touchStartedOnPlayerEntity`。
  - `touchesMoved` 处理小地图、双指平移/缩放、建筑预览、框选和相机拖动。
  - `touchesEnded` 处理小地图、框选、pending 建筑、相机拖动和短距离点击，最后调用 `handleWorldTap(at:)`。
- 当前 `handleWorldTap(at:)` 的优先级是 pending 支援 -> pending 建筑 -> pending 集结点 -> pending `AMOV` -> 世界实体点击 -> 地面移动/清空选择。
- 当前点击己方实体分支是：
  - `if tapped.faction == .player { selectedIDs = [tapped.id]; refreshSelection(); showMessage(...) }`
  - 该分支目前会选择玩家建筑和玩家机动单位。
- 框选 `selectUnits(in:)` 已只选择玩家存活非结构单位，但没有按摄像机视野或同 kind 过滤。
- 小地图和敌方可见性不应被本轮改动。
- `restartSkirmish()` 会清空 `selectedIDs` 和 pending 模式；本轮不应新增需要额外重置的长期状态。若为兼容 `tapCount` 引入短暂触摸字段，也必须在触摸结束或 `SKRM` 中无残留。

## 实现步骤

### 1. 确定双击信号传递方式

优先采用 `UITouch.tapCount` 做最小改动。

建议实现：

- 在 `touchesEnded(_:,with:)` 读取当前触摸的 `tapCount`。
- 对通过短距离点击校验、且不属于小地图/拖框/平移/建筑拖动的普通世界点击，把 `tapCount >= 2` 传给世界点击处理。
- 将 `handleWorldTap(at:)` 改为带默认参数，例如：

```swift
private func handleWorldTap(at point: CGPoint, tapCount: Int = 1)
```

或：

```swift
private func handleWorldTap(at point: CGPoint, isDoubleTap: Bool = false)
```

要求：

- pending 支援、pending 建筑、pending 集结点、pending `AMOV` 分支不应因为双击改变语义；这些模式仍按当前点击处理。
- `touchesBegan` 不要立即根据 `tapCount` 执行选择，避免绕开现有 HUD、小地图、拖框和相机判断。
- 如果 `UITouch.tapCount` 在某些设备上不可靠，可用现有触摸状态加短时间/短距离辅助判断，但必须保持实现小而局部，不新增复杂输入管理器。

### 2. 新增摄像机可见世界矩形 helper

在 `GameScene.swift` 内新增私有 helper，用当前相机和场景尺寸计算世界可见范围。

建议形态：

```swift
private func visibleWorldRect(padding: CGFloat = 0) -> CGRect {
    let width = size.width * cameraRig.xScale + padding * 2
    let height = size.height * cameraRig.yScale + padding * 2
    return CGRect(
        x: cameraRig.position.x - width / 2,
        y: cameraRig.position.y - height / 2,
        width: width,
        height: height
    )
}
```

说明：

- 当前 `updateMinimap()` 已用 `size.width * cameraRig.xScale` / `size.height * cameraRig.yScale` 估算可见宽高，可以沿用这个模型。
- 可给 `padding` 一个小值，例如 12 到 32，避免单位脚边在屏幕边缘时被排除。
- 不要用全地图 `worldBounds` 代替摄像机视野。

### 3. 新增同类选择 helper

新增一个聚焦 helper，避免把过滤逻辑堆在 `handleWorldTap` 里。

建议形态：

```swift
private func selectVisiblePlayerUnits(matching tapped: GameEntity)
```

过滤条件必须包含：

- `entity.faction == .player`
- `entity.isAlive`
- `!entity.kind.isStructure`
- `entity.kind == tapped.kind`
- `visibleWorldRect(padding: ...)` 包含 `entity.node.position`

结果处理：

- `selectedIDs = Set(matches.map(\.id))`
- 若 matches 意外为空，至少选中 tapped：`selectedIDs = [tapped.id]`
- 调用 `refreshSelection()` 和 `updateHUD()`。
- 显示短反馈：
  - `matches.count > 1`：`Selected \(matches.count) \(plural display name).`
  - `matches.count == 1`：`Selected \(tapped.kind.displayName).`
  - 不要求复杂英文复数；可用 `\(tapped.kind.displayName)s` 或避免复数：`Selected \(matches.count) \(tapped.kind.shortCode).`

注意：

- 不要复用 `selectedMobilePlayerUnits()` 作为候选源，因为它只从当前已选集合取单位；本轮需要从 `entities.values` 中查找屏幕内同类。
- 不要选中建筑，即使玩家双击的是 HQ、Factory、Radar Outpost 或 Guard Tower。建筑双击应退化为原来的单击选择。
- 不要把敌人或中立实体纳入同类选择。

### 4. 接入己方实体点击分支

在 `handleWorldTap(at:tapCount:)` 的 `if let tapped = entity(at: point)` 分支中处理：

- 如果 `tapped.faction == .player && !tapped.kind.isStructure && tapCount >= 2`：
  - 调用同类选择 helper。
  - `return`。
- 否则保持原有单击选择：
  - `selectedIDs = [tapped.id]`
  - `refreshSelection()`
  - `showMessage("Selected \(tapped.kind.displayName).", color: .white)`
  - `return`

必须保持：

- 双击玩家建筑仍只选中该建筑，不触发同类建筑选择。
- 双击敌人不触发同类选择；如果已有可攻击单位选中，仍按现有攻击命令处理。
- 双击地面不触发同类选择；如果已有移动单位选中，仍按现有移动处理。若 `tapCount >= 2` 传入地面分支，不应有额外副作用。
- pending 模式在世界实体分支之前处理，因此双击不应取消或改变 pending 语义。

### 5. 保持 HUD、选择框和命令链路

- `refreshSelection()` 应足够更新选择圈；如同类选择后 HUD 不立即刷新，可显式 `updateHUD()`。
- 多选同类后，选择信息面板应自然走 `groupSelectionInfo(for:)`，显示数量、HP、总伤害、射程和老兵分布。
- 同类选择后，后续移动、攻击、`HOLD`、`AMOV` 应自然使用 `selectedMobilePlayerUnits()`，不需要新增命令路径。
- 框选 `selectUnits(in:)` 不应被改造。
- `ARMY` 全军按钮不应被改造。

### 6. 文档同步

实现完成后更新：

- `README.md`
  - Controls 增加一条当前真实操作，例如：`Double-tap a player mobile unit to select visible units of the same type in the current camera view.`
  - 不写控制编队、热键、命令按钮高亮或未实现能力。
- `md/flow/flow.md`
  - 在“输入与命令”或“用户入口”中记录双击同类选择的当前真实流程。
  - 明确它发生在普通世界点击选择链路中，只对玩家机动单位生效，并保持 pending 模式优先级。
- `md/flow/flowchart.md`
  - 如果玩家输入图仍准确，可不改。
  - 若修改，建议只在 `WorldTap -- 己方实体 --> Select` 附近补充“双击己方机动单位 -> 选择视野内同 kind 单位”，不要重画全图。
- `md/test/test.md`
  - 本轮不新增测试方式，通常不需要修改；只有实际调整验证流程时才更新。
- `update_log.md`
  - Agent B 不要预写正式 v1.3 记录。Agent C 在 artifact 验收通过后追加。

不要修改 `md/prompt/README.md`，除非发现提示词管理规则本身必须修正；本轮正常不需要。

## 验收标准

功能标准：

- 单击己方机动单位仍只选择该单位。
- 单击己方建筑仍选择该建筑。
- 双击己方机动单位会选择当前摄像机视野内所有同 `kind` 的己方存活机动单位。
- 如果当前摄像机视野内只有被双击单位本身，仍保持选中该单位，并给出简短反馈。
- 双击己方建筑不选择同类建筑，不影响建筑选择、生产来源、集结点和施工信息。
- 双击敌方或中立实体不触发同类选择；已有可攻击单位选中时，敌方点击仍按现有攻击命令。
- 双击地面不触发同类选择；已有机动单位选中时仍按现有移动命令。
- 框选、`ARMY`、`HOLD`、`AMOV`、普通移动、攻击命令、建筑放置、支援技能目标、集结点、小地图和双指平移缩放不回归。

- 同类选择不选择敌方、迷雾中不可交互对象、中立油井、建筑或死亡实体。
- `SKRM` 重开后选择状态、pending 模式和任何新增短暂触摸状态无残留。

架构标准：

- 风险：`tapCount` 如果在 `touchesBegan` 提前处理，可能绕过 HUD、小地图、拖框和 pending 模式。应在 `touchesEnded` 通过现有点击门槛后处理。
