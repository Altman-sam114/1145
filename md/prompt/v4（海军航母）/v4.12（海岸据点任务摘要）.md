# v4.12（海岸据点任务摘要）

## 版本号

v4.12

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.11「航母甲板状态面板」。
- v4 阶段持续围绕海军、航母、海岸目标、任务面板和 HUD 可读性做小步增强。
- 本轮是一个范围很小的 HUD / mission detail 增量：增强 `Secure Coast` 任务详情，让玩家能看出 2 个 operational coastal assets 的构成，而不是只看到总数。

## 背景

当前 `Secure Coast` 任务阶段已经存在：

- `MissionStage.secureCoast` 标题为 `Secure Coast`。
- `missionDetail(for:)` 对 `.secureCoast` 返回：`Hold 2 coastal assets. X/2 secured. +$600`。
- `isMissionStageComplete(_:)` 对 `.secureCoast` 使用 `coastalAssetCount(for: .player) >= 2`。
- `coastalAssetCount(for:)` 统计己方存活且 `isOperational` 的 `.shipyard`、`.sonarBuoy` 和 `.coastalBattery`。
- 未完工结构不计入任务进度，这符合项目规则：建筑未 operational 前不得生产、赚钱、提供视野、支援资产、基地覆盖或贡献任务资产。

问题是当前任务详情只显示总数，玩家无法从任务面板直接看出自己已经完成的是 Shipyard、Sonar Buoy 还是 Coastal Battery。人工指定本轮目标：在任务面板中显示玩家已完工 Shipyard / Sonar Buoy / Coastal Battery 计数摘要，例如 `SY1 SON0 CB1`，同时保持完成条件仍是任意 2 个 operational coastal assets。

## 目标

1. 增强 `Secure Coast` 任务详情文案。
   - 当当前任务阶段是 `.secureCoast` 时，任务面板 detail 必须显示玩家 operational coastal assets 总进度。
   - 同一行必须显示分项计数摘要，建议格式：

```text
Hold 2 coastal assets. 2/2 secured. SY1 SON0 CB1 +$600
```

   - `SY` 表示 operational player Shipyard 数。
   - `SON` 表示 operational player Sonar Buoy 数。
   - `CB` 表示 operational player Coastal Battery 数。
   - 计数只统计玩家阵营、存活、已完工 operational 的对应建筑。
2. 保持任务完成条件不变。
   - `.secureCoast` 完成条件仍必须是 `coastalAssetCount(for: .player) >= 2`。
   - 仍然允许任意组合完成，例如 `SY2 SON0 CB0`、`SY1 SON1 CB0`、`SY0 SON1 CB1`、`SY0 SON0 CB2`。
   - 不要求三种建筑各一个，不新增科技门槛、地形门槛、奖励门槛或任务顺序变化。
3. 保持任务奖励不变。
   - `missionReward(for: .secureCoast)` 仍为 `$600`。
   - `completedMissionStages` 防止重复发奖的逻辑不变。
4. 保持面板紧凑。
   - 不改 mission panel 尺寸、字体、布局、行数或 HUD 节点结构，除非发现现有文案无法容纳且必须做最小调整。
   - 推荐短文案，不引入长解释。
5. 文档同步。
   - `README.md`：把 mission objective panel 说明同步为 Secure Coast 显示 operational coastal asset 总进度和 `SY/SON/CB` 分项摘要。
   - `md/flow/flow.md`：同步 `MissionStage` 或 `HUD / 小地图` 中 Secure Coast 详情的真实行为。
   - `md/flow/flowchart.md`：如当前 Mission 或 HUD 节点提到海岸任务，可用短语同步；如果图已经足够概括且不改，Agent B 必须说明原因。
6. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不改变 `coastalAssetCount(for:)` 的语义，除非为了复用分项统计做最小等价重构。
- 不改变 Shipyard、Sonar Buoy、Coastal Battery 的建造规则、海岸放置规则、费用、建造时间、HP、视野、声呐、武器、生产或 AI 使用。
- 不改变 `missionReward(for:)`、`updateMissionProgress()`、`completedMissionStages`、胜负条件或任务阶段顺序。
- 不改变 `MissionStage` 枚举 case 顺序、标题或总任务数量。
- 不新增 HUD 控件、弹窗、面板行、图标、小地图标记、地图 ping 或教程文本。
- 不改变 AI、战斗、移动、潜艇隐身、声呐检测、支援技能、生产队列、rally、AMOV、老兵系统或地图生成。
- 不新增外部依赖、素材、XCTest target、workflow 或 Xcode project 配置。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。

2. 修改 `DesertFrontline/GameScene.swift`。

推荐实现方向：

- 新增一个小型分项统计 helper，避免在 `missionDetail(for:)` 里堆复杂 filter。例如：

```swift
private func coastalAssetBreakdown(for faction: Faction) -> (shipyards: Int, sonarBuoys: Int, coastalBatteries: Int)
```

- helper 必须只统计对应 faction、存活、已完工 operational 的三类 coastal assets。
- `.secureCoast` 详情中复用该 helper 生成短摘要：

```swift
let assets = coastalAssetBreakdown(for: .player)
let total = assets.shipyards + assets.sonarBuoys + assets.coastalBatteries
return "Hold 2 coastal assets. \(min(total, 2))/2 secured. SY\(assets.shipyards) SON\(assets.sonarBuoys) CB\(assets.coastalBatteries) +$600"
```

- `coastalAssetCount(for:)` 可以继续保留原实现，也可以改为复用 `coastalAssetBreakdown(for:)` 并返回三项和。若改为复用，必须保证语义完全一致。
- `isMissionStageComplete(.secureCoast)` 仍调用 `coastalAssetCount(for: .player) >= 2`，或等价调用同一总数 helper；不得改成要求具体种类。

## 验收标准

- `Secure Coast` 任务详情显示总进度，例如 `0/2`、`1/2`、`2/2`，并保留 `+$600` 奖励提示。
- 同一任务详情显示 operational player Shipyard / Sonar Buoy / Coastal Battery 分项摘要，例如 `SY1 SON0 CB1`。
- 分项计数只统计玩家阵营、存活、已完工 operational 的 `.shipyard`、`.sonarBuoy`、`.coastalBattery`。
- 未完工、死亡、敌方或其他类型建筑不计入分项摘要或总数。
- `.secureCoast` 完成条件仍是任意 2 个 operational coastal assets；没有要求三类各一个。
- `missionReward(for: .secureCoast)` 仍为 `$600`，且重复发奖保护不变。
- 其他任务阶段详情、胜负、AI、生产、建造、战斗、迷雾、支援技能、rally、AMOV、Carrier 面板、Xcode project 和 workflow 不发生无关改动。

- 本轮提交已 push 到 `origin/main` 并触发 GitHub Actions。
- Agent C 能下载最新 run artifact 并核对 manifest / JUnit / xcodebuild.log / failure summary / xcresult。

## 风险和禁止项

- 风险：mission detail label 只有一行，文案过长可能在较窄 HUD 下不够清晰。优先使用短摘要 `SYx SONy CBz`，不要加长解释。
- 风险：如果 `coastalAssetCount(for:)` 被重构，可能误把未完工建筑、敌方建筑或死亡建筑算入任务；必须保持 `isAlive && isOperational && faction` 过滤。
- 风险：把分项摘要误解为完成条件可能导致玩家必须造三类建筑；严禁修改完成条件为三类各一个。
- 风险：任务详情文案变化属于玩家可见行为，必须同步 README 和 flow。

禁止：

- 禁止改变 `.secureCoast` 的完成条件、奖励、阶段顺序或完成消息语义。
- 禁止新增或删除任务阶段。
- 禁止修改建筑数值、AI 建造策略、海岸放置规则、声呐/岸防/船坞功能。
- 禁止让任务面板泄露敌方建筑信息；本轮只显示玩家自己的 operational asset 计数。
- 禁止用本地测试、本地 build、本地静态检查或 `git diff --check` 作为验收。
- 禁止伪造 GitHub Actions、artifact、manifest、JUnit 或 xcodebuild 日志结果。
- 禁止使用非 `Altman-sam114` 的 GitHub 账号伪装完成 push、CI 或 artifact 验收。
