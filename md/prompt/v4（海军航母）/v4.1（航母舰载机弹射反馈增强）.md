# v4.1（航母舰载机弹射反馈增强）

## 版本号

v4.1

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新玩法版本为 v4.0「声呐浮标专职反潜再平衡」。
- v4 阶段已经开启海军 / 航母深化，本轮继续推进航母存在感，但只做一个小而可验证的反馈增量。
- 当前遗留方向点名“航母舰载机巡逻 / launch 反馈”；完整巡逻、截击或舰载机 AI 会牵动战斗、AI 和目标选择，范围过大，因此 v4.1 只增强现有 launch visual。

## 背景

当前 Carrier 已经是移动空军生产平台，可生产 Helicopter / Fighter、设置空军 rally point，并在自身攻击时显示短暂舰载机 / 弹道视觉。生产完成时，若来源是 Carrier 且产物为空军，会调用 `showCarrierLaunch(...)`，但反馈较轻，玩家不容易从战场上读出“航母甲板正在放飞舰载机”。

本轮目标是让现有航母生产空军和航母开火更像舰载机从甲板起飞，不新增真正巡逻 AI、不新增持久子单位，也不改变任何战斗或生产数值。

## 目标

1. 增强 Carrier 生产 Helicopter / Fighter 完成时的 launch 反馈。
   - 继续只在 `source.kind == .carrier && order.kind.domain == .air` 时触发。
   - 将产物 `order.kind` 传入 launch visual，使 Fighter / Helicopter 可显示不同短暂机体 cue 或不同 label。
   - 保留现有 ready pulse；新增效果只补充甲板起飞可读性。
2. 增强 Carrier 自身开火时的 wing strike 反馈。
   - `fire(attacker:target:)` 中 Carrier 仍走既有直接火力结算。
   - 可让 `launchCarrierWing(...)` 接收 faction，并显示短暂甲板闪光 / 舰载机掠过 cue。
   - 不改变 damage、cooldown、range、target acquisition、XP、ASW、迷雾或胜负。
3. 新增的视觉节点必须是短生命周期 `effectsLayer` 节点。
   - 动画结束后 removeFromParent。
   - 不保存到 `GameEntity`，不影响选择、碰撞、迷雾、目标合法性、AI 认知或单位数量。
4. 文档同步。
   - `README.md`：Current Features 中说明 Carrier deck launch / strike cues 已存在。
   - `md/flow/flow.md`：生产与战斗小节同步 Carrier 生产空军和开火的短暂视觉反馈。
   - `md/flow/flowchart.md`：Build / Combat 节点简短同步 carrier deck launch / wing strike feedback。
5. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不新增舰载机巡逻、自动截击、CAP、甲板容量、弹药、燃油、返航、重新挂载或 carrier aura。
- 不新增持久舰载机子实体，不改变 Helicopter / Fighter 实体生产结果。
- 不改变 `EntityKind.carrier`、Helicopter、Fighter 的 cost、buildTime、HP、speed、attackRange、damage、cooldown、visionTiles 或 canAttack。
- 不改变 `queueBuild(...)`、`productionSource(...)`、`spawnPoint(...)`、rally point、AI build pattern、AI assault wave、动态 AA / ASW、支援技能、潜艇侦测、迷雾合法性或胜负。
- 不新增素材、外部依赖、Xcode project 配置或 GitHub Actions workflow。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。

## 验收标准

- Carrier 生产 Helicopter / Fighter 完成时，除通用 ready pulse 外，还显示更明确的 deck launch visual。

## 风险和禁止项

- 风险：把短暂视觉 cue 写成持久单位会污染实体、AI、选择和胜负统计，禁止。
- 风险：文档误写“巡逻 / 截击 / CAP”会夸大功能，禁止。
- 风险：Carrier 开火频繁时视觉太重；新增 label / pulse 应短暂、轻量。
- 风险：误改生产、spawn、rally 或 AI 会扩大本轮范围，禁止。
- 禁止使用非 `Altman-sam114` 的 GitHub 账号伪装 push、CI 或 artifact 验收。
