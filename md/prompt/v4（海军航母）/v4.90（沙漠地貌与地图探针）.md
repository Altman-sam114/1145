# v4.90 沙漠地貌与地图探针

## 版本号

`v4.90`

## 版本分配依据

- `update_log.md` 与 `md/prompt/v4（海军航母）/` 的最新正式版本是 `v4.89`。
- 本轮是在现有陆海空战场表达上继续做单一、可视化、可云端验收的地图画面增量，因此递增为 `v4.90`，不另开阶段。

## 背景

- 人工确认参考对象是百度百科 lemma id `4042982` 对应、Noble Master Games 于 2012 年发布的《沙漠风暴 Desert Stormfront》，不是其他同名 Desert Storm 游戏。
- Noble Master 官方 press kit `https://www.noblemaster.com/press/sheet.php?p=desert_stormfront` 和官方 trailer `https://www.youtube.com/watch?v=7oZC6DpIUhQ` 展示了清晰的等距沙漠战场：沙地存在明暗层次与风蚀纹理，道路有明确边缘和连续走向，岩柱 / 山脊有阴影与亮暗面，海岸有浅水过渡，地貌细节丰富但不会压过单位和命令反馈。
- 项目当前已经有 `sand / road / water / oil / ridge` 五类 `Terrain`、海岸浅水 / 冲洗带 / 泡沫、稀疏水面波纹以及村庄 / 绿洲 / 农田 / 仓库 / 残骸，但沙地、道路、山脊和油田仍偏平。
- 本轮只借鉴官方作品的可读性、层次和触屏战场密度，不复制其图片、模型、图标、HUD 排版或其他受保护素材。

## 目标

1. 在不改变地形语义的前提下，为沙地增加确定性色差、低透明风蚀沙丘等高线和少量风纹，使大面积沙地不再是一块纯色平面。
2. 根据正交相邻 road tile 的真实连接关系绘制道路肩、内层路床和方向一致的中心标记，使直路、转角和交叉口都能连续阅读。
3. 为 ridge 增加确定性岩体尺寸 / 偏移、落影、亮暗岩面与碎石，形成更清楚的障碍轮廓。
4. 为 oil tile 增加克制的油污环和裂纹，继续保持油田可辨识且不遮挡建筑。
5. 将 GitHub Actions simulator launch probe 从九次扩展为十次；新增 `simulator-map-terrain.png`，使用 `DESERT_CI_CAMERA_FOCUS=coast` 和 tactical 页，专门核对沙地、道路、山脊、油田、海岸、水面、海军航迹与 HUD 层级。

## 非目标

- 不新增或修改 `Terrain` case，不改变 `buildTerrain()`、skirmish 地图种子、油田坐标或海岸坐标。
- 不改变寻路、移动速度、道路加速、油地减速、建造合法性、基地覆盖、海军路径、战争迷雾、小地图地形语义或任务 / 胜负条件。
- 不改单位 / 建筑数值、AI、战斗、经济、生产、支援能力、玩家触摸控制或 HUD 命令结构。
- 不新增外部依赖，不下载 52 MB 官方截图包，不把官方图片或视频作为 App 资源。
- 不运行本地 Xcode build、本地 simulator 或本地玩法探针。

## 当前架构依据

- `DesertFrontline/GameScene.swift` 的 `buildTerrain()` 是地形数据来源；`drawMap()`、`color(for:)`、`addTerrainDetail(...)`、`shorelineSegments(...)` 和 `addScenery(...)` 是本轮允许调整的渲染层。
- `terrain(at:)`、`isValid(...)` 和 `tileCenter(...)` 可为道路连接绘制提供只读邻格信息；不得通过绘制 helper 回写 terrain。
- `initialCameraPosition()` 已支持 `DESERT_CI_CAMERA_FOCUS=coast`，`prepareCICaptureScene()` 的 coast/default 分支会保留玩家海军、方向航迹与水面命中样本，无需另建 capture-only 玩法系统。
- `.github/workflows/ci-results.yml` 当前生成九张截图并在 `projectSpecificReports` 中逐项列出；本轮在同一 helper 中增加第十次独立启动、PID 存活检查和截图归档。

## 实现步骤

1. Agent B 开始前确认当前分支为 `main`、远端为 `origin/main`，执行 `git fetch origin` 与 `git pull --ff-only origin main`；保留工作区中人工的 `DesertFrontline.xcodeproj/project.pbxproj` `DEVELOPMENT_TEAM` 改动，不暂存、不覆盖。
2. 让 tile 基础色可读取 `TileCoord`，用 row / col / `skirmishSeed` 的纯整数散列为 sand 选择小幅亮暗变化；同一个 skirmish 必须每次一致，不使用随机数或按帧变化。
3. 在 `addTerrainDetail(...)` 中增加低密度沙丘弧线 / 风纹；保持低透明、低 zPosition，避免抢夺单位、建筑、选择圈和命令反馈的对比度。
4. 新增只读道路连接 helper，将四个正交 road 邻格映射到等距菱形边中点；沿连接绘制深色道路肩、较浅路床和短中心标记，使端点、直路、转角、T 形与十字路口都由真实邻格决定。
5. 增强 ridge 的确定性岩体：先绘制偏移落影，再绘制不规则深色主体、暖色亮面和少量碎石；所有节点在 `drawMap()` 时一次创建，不进入 update loop。
6. 增加克制的 oil 环与裂纹，保留现有中心污渍，不改变 oil 的结构放置或移动减速语义。
7. 在 workflow 清理列表、启动序列和 manifest `projectSpecificReports` 中加入 `simulator-map-terrain.png`；调用 `launch_hud_page tactical ... none coast`，继续逐次解析 PID、等待、截图并检查存活。
8. 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `md/test/test.md`，只描述已经实现的当前行为和十张截图基线。

## 风险和禁止项

- 避免每个 sand tile 堆叠大量节点；以确定性低密度规则限制节点数量，优先保证移动端清晰度和启动成本。
- 道路连接只能读取合法正交邻格，不能把 ridge 的越界默认值误当成道路。
- 地貌细节 zPosition 必须低于实体、建筑、雾层和主要反馈，不能遮挡单位或泄露战争迷雾信息。
- 禁止扩大为地形规则重做、地图生成器重写、HUD 重构或战斗数值调整。
- 禁止使用旧 run、旧 artifact 或本地截图冒充本轮云端验收。
