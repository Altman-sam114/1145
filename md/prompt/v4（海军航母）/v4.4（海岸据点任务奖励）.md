# v4.4（海岸据点任务奖励）

## 版本号

v4.4

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.3「建立海岸据点任务阶段」。
- v4 阶段聚焦海军、航母和海岸争夺。v4.2 强化 Red 对玩家海岸基础设施的目标权重，v4.3 引导玩家建立海岸据点。
- 本轮继续沿 v4.3 的任务链推进，但只做一个小型关卡反馈增量：完成 `Secure Coast` 时给玩家一次性资源奖励，不新增科技树、升级系统、HUD 按钮或新 AI 状态。

## 背景

当前任务阶段完成后只显示 `Objective complete: ...`，没有关卡式奖励反馈。旗点占领已有一次性占领奖励，说明共享 `changeMoney(for:by:)` 已能安全处理资源奖励。

`Secure Coast` 要求玩家拥有 2 个 operational 海岸资产，通常意味着玩家已经投入资金建设 Sonar Buoy 或 Coastal Battery。本轮给该阶段增加一次性资源奖励，让海岸据点目标更像 RTS 关卡中的经济推进节点，同时不扩大到完整任务奖励系统。

## 目标

1. 为 `Secure Coast` 增加一次性玩家资源奖励。
   - 奖励金额固定为 `$600`。
   - 只在 `.secureCoast` 首次完成并写入 `completedMissionStages` 时发放。
   - 使用现有 `changeMoney(for: .player, by:)`。
2. 增加任务奖励 helper。
   - 建议新增 `private func missionReward(for stage: MissionStage) -> Int`。
   - `.secureCoast` 返回 `600`，其他阶段返回 `0`。
   - 不新增新的持久状态；`completedMissionStages` 已能避免重复发奖。
3. 更新任务完成反馈。
   - `updateMissionProgress()` 在完成阶段时累计本轮完成阶段的奖励。
   - 若本次完成的阶段奖励大于 0，完成消息应包含 `+$600` 或等价短文案。
   - 无奖励阶段保持原有完成消息语义。
4. 更新任务 HUD 文案。
   - `missionDetail(for: .secureCoast)` 应简短显示该阶段有 `$600` 奖励，例如 `Hold 2 coastal assets. X/2 secured. +$600`。
   - 文案必须短，避免挤压 HUD。
5. 文档同步。
   - `README.md`：任务面板描述中说明 Secure Coast 有一次性资源奖励。
   - `md/flow/flow.md`：任务系统说明中同步 `missionReward(for:)` 与 `$600` 奖励边界。
   - `md/flow/flowchart.md`：可简短同步任务节点含海岸据点奖励。
6. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不为所有任务阶段设计完整奖励系统，不新增奖励 UI 面板或任务奖励列表。
- 不新增海岸旗点、地图目标、科技树、升级系统、HUD 按钮、教程弹窗或新地图。
- 不新增单位、建筑、外部素材或依赖。
- 不改变 Shipyard、Sonar Buoy、Coastal Battery 或任何单位的 cost、HP、damage、range、cooldown、vision、sonar、buildTime、canAttack、footprint 或建造规则。
- 不改变生产队列、AI build pattern、AI 海岸目标权重、动态 AA / ASW、support powers、潜艇侦测、迷雾合法性、路径、damage / XP 结算或胜负条件。
- 不让未完工建筑完成任务或触发奖励。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。

关键源码事实：

- `MissionStage` 当前顺序为 `secureOil`、`captureFrontline`、`secureCoast`、`combinedArms`、`breakRedProduction`、`destroyHQ`。
- `missionDetail(for:)` 负责当前任务 HUD 详情。
- `activeMissionStage()` 返回第一个未完成阶段。
- `updateMissionProgress()` 会 while 连续完成已满足条件的阶段，并在完成后显示最后一个完成阶段消息。
- `completedMissionStages.insert(stage)` 是阶段首次完成边界；该集合在 `restartSkirmish()` 中清空。
- `coastalAssetCount(for:)` 只统计 matching faction、alive、operational 的 Shipyard、Sonar Buoy、Coastal Battery。
- `changeMoney(for:by:)` 是玩家 / AI 资金变化的共享入口，旗点占领奖励和生产/建造/支援消费已经使用它。

建议改动：

- 新增常量或 helper：
  - `private func missionReward(for stage: MissionStage) -> Int`
  - `.secureCoast` 返回 `600`
  - `default` / 其他阶段返回 `0`
- 修改 `missionDetail(for: .secureCoast)`：
  - 建议文案：`Hold 2 coastal assets. \(min(coastalAssetCount(for: .player), 2))/2 secured. +$600`

  - 在 `while let stage = activeMissionStage(), isMissionStageComplete(stage)` 内，插入完成阶段后读取 `missionReward(for:)`。
  - 若 reward > 0，调用 `changeMoney(for: .player, by: reward)`。
  - 累计本轮完成阶段的 reward，用于完成消息。
  - 若 `completedStage` 有奖励，显示 `Objective complete: Secure Coast +$600` 或等价短文案。
  - 无奖励阶段仍显示 `Objective complete: <title>`。

必须保持：

- 不新增独立奖励状态；不要新增 `awardedMissionStages`。
- 不改变 `completedMissionStages` 的清空和 stage 完成边界。
- 不改变 `isMissionStageComplete(_:)` 的 Secure Coast 条件。
- 不改变金钱下限逻辑、收入循环、旗点奖金、生产/建造/支援消费。
- 不改海岸建筑放置、AI、战斗、迷雾、生产、支援技能和单位数值。

## 验收标准

- `Secure Coast` 仍要求玩家拥有 2 个 alive、operational、player-owned 的 Shipyard / Sonar Buoy / Coastal Battery；未完工不计入。
- `.secureCoast` 首次完成时玩家资金通过 `changeMoney(for: .player, by: 600)` 增加 `$600`。
- 奖励不会重复发放；`completedMissionStages` 已包含 `.secureCoast` 后不再触发。
- `SKRM` 重开后 `completedMissionStages` 清空，新局可重新按新局实体完成并发放奖励。
- `Secure Coast` HUD detail 简短显示 `+$600` 奖励；完成消息也显示奖励反馈。
- 其他任务阶段没有新增奖励。
- README、flow、flowchart 与真实行为一致，不宣称新增完整战役奖励系统、海岸旗点、科技升级、AI 新能力或新地图。
- 本轮提交已 push 到 `origin/main` 并触发 GitHub Actions。
- Agent C 能下载最新 run artifact 并核对 manifest / JUnit / xcodebuild.log / failure summary / xcresult。

## 风险和禁止项

- 风险：如果新增独立奖励状态，`SKRM` 和 future save/load 会多一个重置面；本轮禁止新增，直接依赖 `completedMissionStages`。
- 风险：如果在 `missionDetail` 或每帧 HUD 刷新里发钱，会造成重复奖励；奖励必须只发生在 `updateMissionProgress()` 的阶段首次完成路径。
- 风险：任务文案过长会挤压 HUD；保持短句。
- 风险：奖励过高可能跳过生产节奏；本轮固定 `$600`，只补偿海岸建设，不足以直接购买大型舰艇。
- 禁止使用非 `Altman-sam114` 的 GitHub 账号伪装 push、CI 或 artifact 验收。
- 禁止下载大体积历史 artifact、模型、测试数据或无关产物。
