# v4.46（航母警戒翼队锚点范围圈）

## 版本号

v4.46

- v4.45 已让被 Carrier guard anchor 绑定的 HEL/JET 在选择面板显示 `CV GUARD Dn`，但玩家仍需要在战场上自行定位 anchor Carrier。

## 背景

当前选择派生视觉已有三类：

- 玩家选中 active sonar sensor 时显示 `sonarCoverageNode`。
- 玩家选中 Carrier / Battleship 时显示 `escortCoverageNode`。
- 玩家选中 Mechanic 时显示 `repairCoverageNode`。

这些覆盖圈都挂在实体自己的 `node` 下，由 `refreshSelection()` 每次选择刷新时根据 `selectedIDs` 显隐，位于 fog 语义之下，不写入迷雾或目标状态。

当前 Carrier guard wing 链路已有：

- `guardAnchorCarrierID` 记录 HEL/JET 的 anchor Carrier。
- `carrierGuardAnchor(for:)` 判断 anchor Carrier 是否仍有效。
- `carrierGuardThreatRadius` 是 Carrier guard 近域威胁半径。
- `carrierGuardWingStatusLine(for:)` 和 `groupSelectedCarrierGuardWingSummary(for:)` 已显示 `CV GUARD` 与 anchor 距离。

## 目标

当玩家选中已被有效 Carrier guard anchor 绑定的 HEL/JET 时，在其 anchor Carrier 上显示只读 carrier guard anchor 范围圈：

- 范围圈只显示在有效 anchor Carrier 上，不显示在 HEL/JET 自己身上。
- 范围半径使用 `carrierGuardThreatRadius`，避免复用 `highValueNavalEscortRadius` 的 620 护航半径造成语义混淆。
- 单选一个绑定 HEL/JET 时显示该 wing 的 anchor Carrier 范围圈。
- 多选多个绑定 HEL/JET 时，所有有效 anchor Carrier 都显示范围圈。
- 选中未绑定 HEL/JET、普通 HOLD 单位、敌方单位、无效 / 死亡 / 非 HOLD Carrier anchor 时，不显示额外范围圈。
- 选中 Carrier / Battleship 时既有 escort ring 继续按原规则显示；本轮不改变现有 sonar / escort / repair ring 的语义。
- 该范围圈只读，不改变 guard wing 绑定、站位、目标选择、交战、AI、迷雾、生产、支援、任务或胜负。

## 非目标

- 不新增 CAP、combat air patrol、巡逻、截击、拦截、新按钮、新 HUD action、新命令状态机或新 AI 策略。
- 不画跨实体 anchor line，不新增 per-frame line overlay，不新增 scene-level persistent line 节点。
- 不复用 `escortCoverageNode` 的 620 护航范围冒充 guard anchor 范围。
- 不改变 `assignCarrierGuardWing(for:)`、`clearCarrierGuardAnchor(for:)`、`updateCarrierGuardStation(for:)`、`carrierGuardStationPoint(...)`。
- 不改变 `carrierGuardAnchor(for:)` 的有效性条件。
- 不改变 `isCarrierGuardContact(...)`、`carrierGuardPriorityTarget(for:)`、`carrierGuardTargetPriority(...)`、`updateCombat(dt:)`、`fire(attacker:target:)`。
- 不写入 `visibleTiles`、`exploredTiles`、`supportRevealTiles`、`revealedUntil` 或任何迷雾 / 潜艇侦测状态。
- 不新增外部依赖、素材、测试 target 或大体积 artifact。

## 当前架构依据

关键源码边界在 `DesertFrontline/GameScene.swift`：

- `GameEntity`：已有 `sonarCoverageNode`、`escortCoverageNode`、`repairCoverageNode`，可按同类模式新增一个 `carrierGuardAnchorCoverageNode`。
- `configureEntityNode(_:)`：实体创建时集中配置并挂载 coverage nodes。
- `refreshSelection()`：每次选择刷新时统一显隐 selection-derived visuals。
- `shouldShowSonarCoverage(for:)`、`shouldShowHighValueNavalEscortCoverage(for:)`、`shouldShowMechanicRepairCoverage(for:)`：现有只读 coverage 判定模式。
- `carrierGuardAnchor(for:)`：必须复用它判断选中 HEL/JET 的 anchor 是否有效。
- `carrierGuardThreatRadius`：本轮 anchor 范围圈半径来源。

推荐实现：

1. 在 `GameEntity` 增加 `let carrierGuardAnchorCoverageNode = SKShapeNode()`。
2. 在 `configureEntityNode(_:)` 中仿照 sonar / escort / repair：
   - 调用 `configureCarrierGuardAnchorCoverageNode(for:)`。
   - 将 `entity.carrierGuardAnchorCoverageNode` 加到 `entity.node`。
3. 新增 `configureCarrierGuardAnchorCoverageNode(for entity:)`：
   - 仅 `entity.kind == .carrier` 时配置。
   - 半径使用 `carrierGuardThreatRadius`。
   - 使用低 alpha、与现有覆盖圈区分但不刺眼的颜色。
   - 默认 `isHidden = true`。
4. 新增 `shouldShowCarrierGuardAnchorCoverage(for carrier:) -> Bool`：
   - `carrier` 必须是玩家 Carrier、存活。
   - 当前 `selectedIDs` 中至少有一个玩家 HEL/JET，且 `carrierGuardAnchor(for: wing)?.id == carrier.id`。
   - 不直接信任裸 `guardAnchorCarrierID`。
5. 在 `refreshSelection()` 中设置：
   - `entity.carrierGuardAnchorCoverageNode.isHidden = !shouldShowCarrierGuardAnchorCoverage(for: entity)`
6. 不改变任何 command、combat、fog、AI 或 production 函数。
