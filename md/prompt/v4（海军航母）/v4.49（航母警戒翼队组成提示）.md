# v4.49（航母警戒翼队组成提示）

## 版本号

v4.49

## 版本分配依据

- 当前 `md/prompt/v4（海军航母）/` 已存在到 `v4.48（AI航母警戒翼队）.md`，本轮顺延为 v4.49。
- v4.34-v4.48 已为 Carrier guard wing 建立玩家 HOLD 绑定、站位保持、近域威胁优先、HUD 接触提示、选择派生范围圈和 Red AI 空闲航母警戒翼队。
- 当前 Carrier 单选 `GW` 行和多选 `CV GW` 摘要只显示绑定翼队总数，不显示 Helicopter / Fighter 组成；玩家难以判断警戒翼队偏反潜/对空/对地的实际构成。
- 本轮继续提升 guard wing 可读性，只增加 HUD 只读组成提示，不改变命令、AI、战斗、迷雾或生产逻辑。

## 背景

现有 `carrierAirWingLine(for:)` 在 Carrier 单选面板中显示：

- 非 HOLD 时：`Wing x/2 OK` 或 `Wing x/2 Need n`，统计附近 HEL/JET readiness。
- HOLD 时：`GW x/2 OK C0`、`GW x/2 Need n C0` 或带接触类型、优先目标、交战数的紧凑 guard wing 行。

现有 `groupCarrierGuardWingSummary(for:)` 在多选 HOLD Carrier 时显示：

- `CV GW x/y OK/Need Cn Eng n`

这些统计已经复用 `boundCarrierGuardWing(for:)`，只计入仍 HOLD、`guardAnchorCarrierID` 指向该 Carrier 且位于当前 guard wing 口径内的 Helicopter / Fighter。

## 目标

在现有 Carrier guard wing HUD 行中追加绑定翼队组成：

- 单选 HOLD Carrier 的 `GW` 行显示 HEL/JET 组成，例如：
  - `GW 2/2 OK H1 J1 C0`
  - `GW 1/2 Need 1 H1 C2 Sea Tgt SUB Eng 1`
  - `GW 2/2 OK J2 C1 Air Tgt JET Eng 1`
- 多选 HOLD Carrier 的 `CV GW` 摘要也显示聚合组成，例如：
  - `CV GW 3/4 Need 1 H2 J1 C2 Eng 1`
- 组成只统计有效 `boundCarrierGuardWing(for:)` 返回的 wing。
- Helicopter 使用 `Hn`，Fighter 使用 `Jn`。
- 某一类型为 0 时可省略该类型，避免 HUD 过长；两类都为 0 时不追加组成。
- 组成提示只读，不改变 wing readiness、guard anchor、站位保持、接触数、目标选择、交战数、AI、迷雾或战斗。

## 非目标

- 不新增 CAP、combat air patrol、巡逻、截击、新按钮、新 HUD action 或新命令状态机。
- 不改变玩家 Carrier HOLD 绑定条件、Red AI guard wing 绑定条件或任何单位数值。
- 不改变 `assignCarrierGuardWing(for:)`、`clearCarrierGuardAnchor(for:)`、`updateCarrierGuardStation(for:)`、`carrierGuardAnchor(for:)`。
- 不改变 `isCarrierGuardContact(...)`、`carrierGuardPriorityTarget(for:)`、`carrierGuardTargetPriority(...)`、`updateCombat(dt:)`、`fire(attacker:target:)`。
- 不写入 `visibleTiles`、`exploredTiles`、`supportRevealTiles`、`revealedUntil` 或任何迷雾 / 潜艇侦测状态。
- 不新增外部依赖、素材、测试 target 或大体积 artifact。

## 当前架构依据

关键源码边界在 `DesertFrontline/GameScene.swift`：

- `carrierAirWingLine(for:)`：单选 Carrier 的 `Wing` / `GW` 行组装点。
- `groupCarrierGuardWingSummary(for:)`：多选 HOLD Carrier 的 `CV GW` 行组装点。
- `boundCarrierGuardWing(for:)`：当前 guard wing 统计口径，必须直接复用，不能改成 `nearbyCarrierAirWing(for:)`。
- `carrierGuardContactSummary(for:guardWing:)`、`carrierGuardEngagedCount(for:guardWing:)`：现有接触数、类型、目标和交战数口径，本轮不得改动。

推荐实现：

1. 新增 `carrierGuardWingCompositionSuffix(for guardWing: [GameEntity]) -> String`。
2. helper 内分别统计 `.helicopter` 与 `.fighter`：
   - `H1 J1`
   - `H2`
   - `J2`
   - 空翼队返回空字符串。
3. 在 `carrierAirWingLine(for:)` 的 `isGuarding` 分支中，用 `guardWing` 计算 `compositionSuffix`，插入 readiness 后、`C...` 接触信息前。
4. 在 `groupCarrierGuardWingSummary(for:)` 中聚合所有选中 HOLD Carrier 的 `guardWing`，用同一 helper 追加到 readiness 后、`C...` 前。
5. 不改非 HOLD `Wing` 行；本轮只给 `GW` / `CV GW` guard wing 行显示组成。
