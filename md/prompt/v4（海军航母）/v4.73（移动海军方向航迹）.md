# v4.73 / 移动海军方向航迹

## 版本号与依据

v4.73。v4.72 已由最新 `origin/main` run `29120614461` artifact 验收通过，海岸已有浅滩、冲洗带和泡沫线；下一项限定为移动海军的低开销方向航迹。

## 背景与目标

现有 Battleship、Submarine、Carrier 船体有细节，但移动时水面缺少速度和方向反馈。本轮给每个海军实体预创建航迹子节点，仅在实际位移时按方向显示双侧冲洗、细泡沫和中心扰流，停止时隐藏。航迹必须跟随实体迷雾可见性，不得留下独立节点泄露隐藏敌军位置。

## 非目标

- 不改速度、路径、编队、碰撞、攻击、AI、经济、任务、胜负、声呐或潜艇暴露。
- 不做粒子系统、持久轨迹缓存、海浪物理或外部素材。
- 不新增依赖或 XCTest target。
- 不运行任何本地测试、build、模拟器或探针。
- 不提交用户的 `project.pbxproj` 团队号改动。

## 架构依据

- `GameEntity` 已集中持有实体 SpriteKit 节点。
- `updateMovement(dt:)` 计算实际 `direction` 和 step，是刷新航迹的唯一事实入口。
- `animateIdle(...)` 处理无 destination 状态，可统一隐藏航迹。
- 敌方迷雾通过隐藏实体根节点执行；航迹作为实体子节点可自动遵守同一边界。
- v4.72 的 CI capture mode 已稳定冻结初始海岸画面，可用确定性海军移动意图展示航迹。

## 实现步骤

1. 在 `GameEntity` 增加预创建的 `navalWakeNode`。
2. 配置海军节点时创建按舰种缩放的冲洗线、泡沫线和中心扰流，默认隐藏且位于船体下方。
3. 海军实际移动后按世界方向和实体镜像换算局部航迹角度，显示并轻微调制透明度。
4. idle 时隐藏航迹，不在每帧创建新节点。
5. CI capture scene 只在 `DESERT_CI_CAPTURE_MODE=1` 时给初始玩家海军设置确定性移动意图并刷新航迹；普通启动不受影响。
6. 更新 README、flow、flowchart、test 文档。
7. commit `v4.73: 增加移动海军方向航迹` 并直推 `origin/main`，只做云端验证。

## 验收标准

- Battleship、Submarine、Carrier 移动时出现与实际反方向一致的舰艉航迹，停止时隐藏。
- 航迹随舰种大小变化，位于船体下方，不遮挡 HUD、选择圈或生命条。
- 敌军实体被迷雾隐藏时航迹同步隐藏；无独立残留节点泄露位置。
- 移动、路径、战斗、声呐和潜艇规则无变化，节点数量不按帧增长。
- 最新 main run 的 manifest SHA 一致，static checks、lint、generic iOS build、simulator launch 成功，JUnit 0 失败。
- 云端 coast screenshot 显示玩家 Battleship / Carrier 方向航迹和真实 HUD，不是黑屏或白屏。
