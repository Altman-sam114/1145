# v4.96 机动防空模型与双弹拦截

## 版本号

`v4.96`

## 版本分配依据

- `update_log.md` 与 `md/prompt/v4（海军航母）/` 的最新正式版本为 `v4.95`。
- 人工要求优先改善 UI 与战斗细节；当前 AA Truck 仍是两轮简化车身和单枚通用导弹，本轮只细化该单位模型、开火反馈和云端取证，因此递增为 `v4.96`。

## 背景

- 参考对象固定为百度百科 lemma id `4042982` 对应的 Noble Master Games《沙漠风暴 Desert Stormfront》，不是其他同名作品。
- 已核对 Noble Master 官方 press kit `https://www.noblemaster.com/press/sheet.php?p=desert_stormfront`、官方 trailer `https://www.youtube.com/watch?v=7oZC6DpIUhQ` 和官方截图。官方作品以小尺寸高对比军车轮廓、明确雷达 / 发射设备、可读弹体、烟迹和短促爆点维持俯视战场辨识度；本项目只借鉴这些原则，不复制素材或 UI。
- 当前 AA Truck 已共享移动、目标合法性、AI、防空威胁圈、迷雾和统一 `fire(attacker:target:)` 伤害链路，但模型只有简单车身 / 两个轮子，攻击与 Fighter / SAM 共用单弹直线视觉。

## 目标

1. 在既有 footprint 内将 AA Truck 重绘为六轮装甲底盘、分层车身和驾驶舱，增加阵营色风挡 / 光学件、雷达盘、双轨发射架、四枚导弹、灯具、保险杠、设备舱和天线。
2. 将 AA Truck 开火细化为两枚导弹错开发射，包含双发射点、烟迹、发光弹体、尾焰 / 闪光和短促雷达跟踪 cue；普通效果应快速完整清理。
3. 每次 `fire` 仍只扣一次 HP；双弹与雷达 helper 只负责视觉，不新增第二次伤害、碰撞或 splash。
4. 未知 Red AA Truck 不得通过双弹、发射闪光或雷达 cue 泄露位置；既有目标可见性和空中命中反馈保持。
5. 新增 capture-only `mobile-aa` 场景：单选 Blue AA Truck 锁定约 46% HP Red Fighter，冻结双弹、雷达跟踪、命中环和伤害飘字，同时保留空军投影、目标 HP / 距离 / reload、小地图和单排命令条。
6. GitHub Actions 从十五次扩展为十六次独立启动，新增 `simulator-mobile-aa.png`。

## 非目标

- 不改变 AA Truck damage、range、cooldown、cost、build time、HP、speed、vision、生产来源、AI、target acquisition 或 `canAttack(...)`。
- 不改变 SAM Site、Fighter、Helicopter 的模型、数值或既有单弹视觉。
- 不新增弹药库存、装填状态、范围伤害、拦截概率、雷达 gameplay state 或外部素材。
- 不改变防空威胁圈、迷雾、声呐、潜艇隐身、经济、建造、任务或胜负。
- 不运行本地 Xcode build、本地 simulator 或本地玩法探针。

## 当前架构依据

- `addLandUnitBody(for:to:)` 是 AA Truck 既有模型入口，应继续复用 `GameEntity` 节点、碰撞、选择、血条和 entity zPosition。
- `fire(attacker:target:)` 是玩家与 AI 共用的唯一伤害结算入口；双弹 helper 只能派发视觉，`target.hp` 仍只在统一位置写一次。
- `showProjectile(...)` 已按 `EntityKind` 分流，AA Truck 可在此进入专属齐射 helper；Fighter / SAM 保持 `showGuidedMissileTrail(...)`。
- `prepareCICaptureScene()` 只在 `DESERT_CI_CAPTURE_MODE=1` 调用；新 marker 必须在 camera focus 早退前分流，避免落入通用 land capture。

## 实现步骤

1. 确认 `main`、`HEAD == origin/main`，保留人工未暂存的 `DesertFrontline.xcodeproj/project.pbxproj` `DEVELOPMENT_TEAM` 改动。
2. 在 `addLandUnitBody(...)` 内重绘 AA Truck，不改变 footprint、物理或数据属性。
3. 为 `showGuidedMissileTrail(...)` 增加兼容默认值的 delay 支持；新增 `showMobileAASalvo(...)` 组合两个发射点、两条轨迹、错开发射闪光和雷达跟踪 cue。
4. `showProjectile(...)` 仅对 `.aaTruck` 分流到专属 helper，其他种类保持现状；`fire(...)` 只在 AA Truck 来源对玩家已知时派发该视觉。
5. 新增 `prepareCIMobileAACaptureScene()`，写入 AA / Fighter 位置、目标、46% HP、reload、selection、fog 和 persistent 视觉。
6. workflow 清理列表、launch 列表和 manifest 增加 `simulator-mobile-aa.png`，总计十六次 PID / PNG 检查。
7. 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 与 `md/test/test.md`；Agent C 验收通过后再更新 `update_log.md`。
