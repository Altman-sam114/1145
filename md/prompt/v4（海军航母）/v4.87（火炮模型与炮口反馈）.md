# v4.87 / 火炮模型与炮口反馈

## 版本号与分配依据

版本号为 `v4.87`。v4.86 已完成 Tank 几何细化和独立 land 云端截图，本轮沿同一综合战场表现阶段递增一个小版本，只补齐 Artillery 的模型与开火可读性。

## 背景

Desert Stormfront 官方 trailer、TouchGameplay 15 分钟实机视频及官方公开说明都把 Artillery 列为核心地面兵种。参考画面强调俯视小比例下的长炮管、履带底盘和开火闪光。当前 Desert Frontline 的 Artillery 只有矩形车体和单根炮管，视觉层级明显落后于 v4.86 Tank；现有 land capture 已能提供确定性地面截图，可继续强化而无需增加额外启动次数。

## 目标

1. 在 `addLandUnitBody(...)` 内细化 Blue / Red Artillery：履带 / 负重轮、稳定铲、战斗室、炮座、炮盾、分段炮管、制退器和阵营化光学件。
2. Artillery 通过既有 `fire(...)` 开火时，在玩家单位或玩家已知敌方炮位显示短促炮口焰、烟团和贴地尘环；隐藏敌方炮位不得被特效泄露。
3. `showProjectile(...)` 增加默认关闭的 `persistent` 参数，普通行为不变；land CI capture 冻结 Artillery 炮口反馈与炮线样本。
4. land capture 加入双方 Artillery，与 v4.86 Tank 对峙和爆炸样本同屏；同步 README、flow、flowchart 和 test。

## 非目标

- 不修改 Artillery HP、速度、射程、伤害、冷却、结构伤害倍率、生产价格或 veterancy。
- 不新增抛物线弹道、炮塔独立旋转、溅射伤害、压制状态或音频系统。
- 不修改 AI、寻路、碰撞、迷雾集合、经济、任务、胜负或 HUD 页结构。
- 不新增外部依赖或素材，不运行本地 Xcode build / simulator / 玩法探针。
- 不修改或提交用户已有的 `project.pbxproj` DEVELOPMENT_TEAM 改动。

## 当前架构依据

- `GameScene.swift` 是实体渲染、`fire(...)`、弹丸特效和 CI capture 的事实核心。
- Artillery 继续使用 `GameEntity` / `EntityKind.artillery`；新 shape 必须留在原 `base` 子树。
- `fire(...)` 继续唯一负责伤害、目标、击杀、飘字和水面命中；本轮只在合法开火后补视觉反馈。
- 玩家认知边界继续复用 `isKnownToFaction(..., observer: .player)`。
- `DESERT_CI_CAPTURE_MODE=1` 冻结实时循环，`DESERT_CI_CAMERA_FOCUS=land` 只用于云端截图。

## 实现步骤

1. 重构 Artillery shape 组合，保持 footprint、标签、队旗和生命条布局不变。
2. 新增 `showArtilleryMuzzleBlast(...)`，组合炮口闪光、亮芯、烟团和尘环；普通模式短动画后清理，CI `persistent` 模式冻结。
3. `fire(...)` 仅当 Artillery 炮位对玩家已知时调用炮口反馈，再走原 `showProjectile(...)` 和伤害结算。
4. `showProjectile(...)` 增加默认参数 `persistent=false`，旧调用无需改动；CI land scene 显式冻结炮线。
5. 在 land capture 中布置双方 Artillery，并把玩家火炮加入选择集合。
