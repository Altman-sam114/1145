# v4.20（多选高价值海军护航摘要）

## 版本号

v4.20

## 版本依据

- `update_log.md` 当前最新正式玩法版本为 v4.19「高价值海军护航半径圈」。
- v4.18 已为玩家单选 Carrier / Battleship 显示只读 `Escort x/y` 护航状态。
- v4.19 已为玩家选中的 Carrier / Battleship 显示独立暖色 escort radius ring，且多选多个高价值海军时可显示多个半径圈。
- 本轮是在 v4.18 / v4.19 基础上的小 HUD 信息增量：当玩家多选或框选编队中包含 Carrier 或 Battleship 时，在选择信息面板显示高价值海军护航满足情况摘要。
- 人工指定本轮版本为 v4.20，且要求只做 Agent B 实现提示词，不在本轮实现代码。

## 背景

当前 `DesertFrontline/GameScene.swift` 已有以下事实：

- `highValueNavalEscortRadius = 620` 是护航统计和 v4.19 护航半径圈的共同范围。
- `highValueNavalEscortRequirement(for:)` 对 Battleship 返回 1，对 Carrier 返回 2，其它单位返回 `nil`。
- `highValueNavalEscortLine(for:)` 和 `nearbyNavalEscortCount(for:)` 已用于单选面板显示 `Escort x/y`。
- `nearbyNavalEscortCount(for:)` 只读统计同阵营、存活、operational、非结构、可攻击、非当前实体、非 Battleship / Carrier、距离不超过 `highValueNavalEscortRadius` 的附近单位。
- `groupSelectionInfo(for:)` 当前显示多选编队的 land / air / sea、HP、总伤害 / 最大射程 / ASW、以及 HOLD / AMOV / 老兵分布，但没有汇总高价值海军的 escort 满足情况。
- `refreshSelection()` 已在多选时显示多个选中玩家 Carrier / Battleship 的 `escortCoverageNode`，但选择信息面板没有给出多选摘要。

玩家多选含 Carrier 或 Battleship 的编队时，当前只能看地图上的多个半径圈或逐个单选检查，缺少一个稳定的编队级摘要。

## 目标

1. 当玩家多选 / 框选的编队中包含 Carrier 或 Battleship 时，在选择信息面板显示只读护航摘要。
2. 摘要必须统计被选中的高价值海军单位的 escort 满足情况，而不是统计整个编队的总 escort 数。
3. 推荐文案短小，适合当前 4 行 selection panel，例如：
   - `HV Navy 2/3 escorted`
   - 或 `Escort OK 2/3  Need 1`
4. 满足口径：
   - 只统计当前多选中的高价值海军：`selected.filter { highValueNavalEscortRequirement(for: $0.kind) != nil }`。
   - 对每个高价值海军，用现有 `nearbyNavalEscortCount(for:)` 与 `highValueNavalEscortRequirement(for:)` 判断是否满足。
   - `satisfied = nearby >= requirement`。
   - `total = selected high-value naval count`。
   - 可选显示 `Need n`，其中 `n` 是未满足单位的缺口总和：`max(0, requirement - nearby)` 的求和。
5. 多选摘要只在 `selected.count > 1` 且高价值海军数量大于 0 时显示；单选 Carrier / Battleship 继续使用 v4.18 的 `Escort x/y`。
6. 不改变 v4.18 单选 `Escort x/y` 文案和 v4.19 护航半径圈显示规则。
7. 不新增玩家命令、自动护航、CAP、巡逻、follow 行为、战斗加成、AI 行为变化、生产规则变化、支援规则变化或迷雾变化。

## 非目标

- 不实现自动护航、护航命令、CAP、舰载机巡逻、巡航路径、跟随行为或编队锁定。
- 不改变 Red AI 的高价值海军护航门槛、routine assault wave、reservation、撤退回修、支援评分或目标选择。
- 不改变玩家 `HOLD`、`AMOV`、普通移动、攻击、集结点、生产、Carrier deck launch、支援技能或控制组语义。
- 不改变护航统计半径 620，不改变 `nearbyNavalEscortCount(for:)` 的现有口径，除非只是为了复用 helper 做只读摘要。
- 不改变 `escortCoverageNode`、`sonarCoverageNode`、战争迷雾、潜艇侦测、目标合法性或可见性判断。
- 不新增外部依赖、素材、测试 target、HUD 按钮或持久化状态。
- 不更新 `update_log.md`；Agent C 在 artifact 验收通过后负责追加 v4.20 正式记录。

3. 在 `GameScene.swift` 中新增一个只读 helper，推荐放在 `nearbyNavalEscortCount(for:)` 或 `groupSelectionInfo(for:)` 附近，例如：
   ```swift
   private func groupHighValueNavalEscortSummary(for selected: [GameEntity]) -> String? {
       let highValueNaval = selected.compactMap { entity -> (nearby: Int, required: Int)? in
           guard let required = highValueNavalEscortRequirement(for: entity.kind) else { return nil }
           return (nearbyNavalEscortCount(for: entity), required)
       }
       guard !highValueNaval.isEmpty else { return nil }

       let satisfied = highValueNaval.filter { $0.nearby >= $0.required }.count
       let missing = highValueNaval.reduce(0) { total, item in
           total + max(0, item.required - item.nearby)
       }
       if missing > 0 {
           return "HV Navy \(satisfied)/\(highValueNaval.count) escorted  Need \(missing)"
       }
       return "HV Navy \(satisfied)/\(highValueNaval.count) escorted"
   }
   ```
   这是参考写法，不要求逐字照抄；但必须复用现有 requirement 和 nearby escort helper，避免复制一套新口径。

## 验收标准

- 多选 / 框选包含 Carrier 或 Battleship 时，selection info panel 显示只读高价值海军护航满足摘要。
- 摘要按被选中的每个 Carrier / Battleship 单独判断是否满足自身 escort 需求：Carrier 需求 2，Battleship 需求 1。
- 摘要复用现有 `nearbyNavalEscortCount(for:)` 和 `highValueNavalEscortRequirement(for:)`，没有复制或分叉 escort 口径。
- 单选 Carrier / Battleship 仍显示 v4.18 `Escort x/y`，Carrier deck / rally / queue 信息不回归。
- 多选护航半径圈仍按 v4.19 行为显示，声呐覆盖圈仍按 active sonar sensor 行为显示。
- HOLD / AMOV 多选状态行优先级保持，不被护航摘要破坏。
- README、flow、flowchart 同步当前真实行为，未夸大为自动护航、CAP、巡逻、战斗加成、AI 行为变化或新命令。
- 最新 `origin/main` 的 v4.20 GitHub Actions artifact 经 Agent C 下载核对通过。

## 风险与禁止项

- 风险：selection panel 只有 4 行，新增摘要可能挤掉老兵分布；必须保持 HOLD / AMOV 优先，文案要短。
- 风险：Carrier 和 Battleship 同时也是 sonar sensor / ASW attacker，摘要不能混淆护航与 sonar / ASW 语义。
- 风险：如果用全队总 escort 数除以全队总需求，会掩盖某艘高价值海军孤立的问题；必须逐艘判断满足情况。
- 禁止修改 AI escort 函数、attack wave 过滤、reservation、撤退回修、支援评分或战略目标权重。
- 禁止新增自动护航、CAP、巡逻、follow 行为、新 HUD 按钮、新命令或战斗加成。
