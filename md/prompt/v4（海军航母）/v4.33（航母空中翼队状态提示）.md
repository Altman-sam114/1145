# v4.33（航母空中翼队状态提示）

## 版本号

v4.33

## 版本分配依据

- 当前 `md/prompt/v4（海军航母）/` 已存在到 v4.32，本轮继续在 v4 海军 / 航母 / HUD 可读性阶段追加小范围增量。
- v4.31-v4.32 已完善 Carrier / Battleship 护航数量与缺口类型提示。
- 后续目标包含航母 CAP / 巡逻；本轮先做最小可验证的前置 HUD 增量：让玩家单选 Carrier 时看到附近友方 HEL/JET 空中翼队数量。

## 背景

当前 `DesertFrontline/GameScene.swift` 中：

- Carrier 可以生产 Helicopter / Fighter，可设置 air rally。
- 单选 Carrier 时，`singleSelectionInfo(for:)` 用 `carrierDeckCapabilityLine()`、`highValueNavalEscortLine(for:)` 和 `carrierDeckQueueAndRallyLine(for:)` 组装 deck、escort、rally、queue 状态。
- 目前 Carrier 单选面板没有显示附近友方空军 wing / cover 数量，玩家难以判断 Carrier 周围是否有可用于空中掩护、未来 CAP / 巡逻的基础空中力量。

本轮只做 HUD readiness 提示，不实现自动 CAP、巡逻、拦截或 follow。

## 目标

单选玩家 Carrier 时，在选择信息面板显示附近友方空中翼队状态：

- 统计同阵营、存活、operational、非结构、kind 为 Helicopter 或 Fighter、距离 Carrier 不超过 `highValueNavalEscortRadius` 的单位。
- 建议需求值为 2。
- 若数量达标，显示 `Wing x/2 OK`。
- 若不足，显示 `Wing x/2 Need n`。
- 推荐追加到 Carrier 的 rally / queue 行末尾，例如：
  - `Rally unset  Queue idle  Wing 1/2 Need 1`
  - `Rally set  Queue JET 6s  Wing 2/2 OK`
- 只在 Carrier 单选分支显示；Battleship、非 Carrier、多选摘要不受影响。

## 非目标

- 不实现 CAP、巡逻、自动护航、follow、拦截、自动起飞、命令队列或新 HUD action。
- 不改变 Carrier 生产、deck launch、rally、air source hint、BuildOrder 或 queueBuild。
- 不改变 escort requirement、escort radius ring、high-value naval escort 统计、AI 护航门槛或 attack wave。
- 不改变 Helicopter / Fighter 的 cost、HP、速度、攻击、射程、视野、damage、cooldown 或 canAttack。
- 不改变玩家移动、攻击、HOLD、AMOV、支援、迷雾、声呐、任务、胜负或 `SKRM` 重置。
- 不新增外部依赖、素材、测试框架、截图、探针或 Xcode project 配置。
