# v4.92 来袭威胁态势 UI

## 版本号

`v4.92`

## 版本分配依据

- `update_log.md` 与 `md/prompt/v4（海军航母）/` 的最新正式版本为 `v4.91`。
- 人工要求继续优先提升 UI 与战斗细节。本轮延续 v4.91 的只读交战态势，在不改变战斗规则的前提下补齐“选中部队正在被谁锁定、威胁来自哪个方向”的反馈，因此递增为 `v4.92`。

## 背景

- 参考对象继续固定为百度百科 lemma id `4042982` 对应的 Noble Master Games《沙漠风暴 Desert Stormfront》，不是其他同名作品。
- 本轮重新核对 Noble Master 官方 press kit `https://www.noblemaster.com/press/sheet.php?p=desert_stormfront`、官方 trailer `https://www.youtube.com/watch?v=7oZC6DpIUhQ`、官方海战图 `screen_dsf_v02_10.png` 和 iOS 图 `screen_dsf_v02_ios_c02_1024x768.png`。官方说明强调面向 casual player 的简化触屏操作；截图以战场为主、用紧贴单位的高对比生命条和直接弹道表达战况。本项目只借鉴信息优先级，不复制原素材或 UI 布局。
- v4.91 已能显示玩家选中部队的输出侧状态：Engaged、Ready、PRIMARY、目标 HP 和 FOCUS。但玩家仍无法快速知道选中部队中有多少单位正被已知敌军锁定、哪一单位承受最多威胁、威胁方向和小地图上的受压位置。
- 敌方实体已有共享 `attackTarget`，战争迷雾已有 `isKnownToFaction(...)`；本轮应只读这些现有状态，不能另建来袭战斗状态或泄露未知敌军。

## 目标

1. 每次 HUD 刷新只扫描一次当前存活、已完工、玩家已知且真实 `attackTarget` 指向选中玩家实体的敌方作战单位，建立当前选择的只读来袭威胁快照。
2. 单选单位存在来袭威胁时，选择面板优先显示 `INCOMING <n> <code> D<n>`；若该单位同时攻击目标，则在 Weapon ready / Reload 行紧凑追加 `IN<n> <code> D<n>`。
3. 多选编队标题增加 `IN <n>`，最后一行增加来袭敌军数量；保留 v4.91 的 Engaged、Ready、PRIMARY、HP、Wounded、Critical 和距离信息。
4. 为每个实体预创建来袭方向标记节点。仅对当前选中、存活的玩家实体显示：目标边缘双重红橙方向箭头，以及目标下方 `IN n` 标签；箭头朝向最近的合法已知来袭敌军，并正确抵消单位 `xScale=-1` 镜像。
5. 小地图上当前选中且存在已知来袭威胁的玩家实体增加红橙告警外圈；未知敌军不得通过世界标记、面板或小地图泄露。
6. 新增 capture-only `incoming-ui` 陆战场景：至少三个已知 Red 作战单位分别锁定两个 Blue 选中单位，其中一个 Blue Tank 处于 Critical HP；冻结一条敌方来袭炮线。GitHub Actions 增加第十二次独立启动并生成 `simulator-incoming-ui.png`。

## 非目标

- 不改变伤害、射程、冷却、护甲倍率、自动索敌、AI、移动、经济、生产、任务、胜负或战争迷雾。
- 不显示未被玩家知晓的攻击者，不以玩家单位正在掉血为理由绕过 `isKnownToFaction(...)`。
- 不新增常驻大面板，不改变命令按钮、触控区域、HUD 页结构或字号。
- 不改单位模型，不新增外部依赖、图片素材或音频。
- 不运行本地 Xcode build、本地 simulator 或本地游戏探针。

## 当前架构依据

- `updateHUD()` 每帧先得到 `selected`，再刷新世界选择反馈、选择信息面板与小地图；可在这里先构建一次来袭威胁快照，供后续 UI 复用。
- `GameEntity.attackTarget` 是玩家与 AI 共用的实时目标来源；`isKnownToFaction(attacker, observer: .player)` 是敌方是否可用于玩家 UI 的唯一可见性边界。
- `singleSelectionInfo(...)` 与 `groupSelectionInfo(...)` 是右侧选择面板的既有文字入口；`updateSelectionInfoPanel(...)` 可按快照改变标题颜色，但不能写回玩法状态。
- `configureEntityNode(...)` 已为 selection、command intent、air-defense 等反馈预创建实体子节点；新增来袭标记也应在实体创建时一次性配置，刷新时只改 hidden、文字、镜像和方向。
- `minimapBlipNode(...)` 已按当前选择重建玩家选中外圈，可基于同一只读快照增加来袭外圈。
- `prepareCILandCaptureScene()` 已能冻结陆战模型、维修、炮线和 v4.91 combat-ui；`DESERT_CI_COMMAND_MARKER=incoming-ui` 只允许在 capture mode 中写入临时攻击目标与 HP。

## 实现步骤

1. 开始前确认当前分支为 `main`、`HEAD == origin/main`，保留人工未暂存的 `DesertFrontline.xcodeproj/project.pbxproj` `DEVELOPMENT_TEAM` 改动。
2. 为 `GameEntity` 增加预创建的来袭 marker、方向节点和计数 label；在 `configureEntityNode(...)` 中调用 `configureIncomingThreatMarkerNode(...)`。
3. 新增 HUD-only `incomingThreatsByTargetID` 快照。`refreshIncomingThreatVisuals(for:)` 每帧先清空并单次扫描敌方实体，只接受存活、已完工、伤害大于 0、玩家已知、合法攻击选中玩家目标的实体；按距离和 id 确定性排序。
4. 对每个受威胁的选中玩家实体显示 marker，方向指向最近攻击者；对没有威胁或未选中的实体隐藏。通过 counter-scale 保证单位左右镜像后文字与箭头方向仍正确。
5. 新增 `incomingThreatSummary(for:)`，为单选 / 多选面板提供攻击者总数、受威胁选中目标数、最近攻击者类型与距离。选择面板标题在有来袭威胁时使用红橙高亮，无威胁时恢复既有青色。
6. `minimapBlipNode(...)` 对选中且存在合法来袭快照的玩家实体增加红橙告警圈。
7. `incoming-ui` land capture 将两个 Red Tank / Artillery / Humvee 中至少三个设置为两个 Blue 选中实体的真实目标，把一个 Blue Tank 冻结到约 31% HP，并增加一条 persistent 敌方炮线；普通启动与其他 capture marker 不进入该分支。
8. workflow 清理列表、启动列表和 manifest 增加 `simulator-incoming-ui.png`，总计十二次独立 launch / PID 存活检查。
9. 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 与 `md/test/test.md`，只记录已完成行为。

## 风险和禁止项

- 禁止为每个选中实体分别全量扫描敌军；每次 HUD 刷新只构建一次快照并复用。
- 禁止 marker 每帧 remove/add child 或创建新 SpriteKit 节点；实体创建时预配置，刷新只改属性。
- 禁止 `IN` 数量统计未知、死亡、未完工、无伤害、友方或非法攻击者。
- 禁止来袭箭头因单位 `xScale=-1` 指向相反方向，或让 `IN n` 文字镜像。
- 禁止多选文案溢出 compact 面板、覆盖目标血条或缩小既有字号。
- 禁止用旧 artifact、本地截图或文字汇报代替最新云端验收。
