# v4.9（恢复 AMOV 按钮入口）

## 版本号

v4.9

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.8「终局 HQ 攻击指引」。
- v4.8 已补强终局 Red HQ 已知时的 `AMOV` 指引，但当前源码中 `HudAction.attackMove`、`handleHudAction(.attackMove)` 和 `issueAttackMoveOrder(...)` 已存在，`layoutHUD()` 的底部 `actions` 数组却没有 `.attackMove`，导致玩家可能无法从 HUD 触发该能力。
- 本轮作为 v4 阶段的小修复，只恢复已有 RTS 基础命令入口，不新增新系统。

## 背景

Attack-move 是 RTS 的基础操作。项目已有完整的 pending attack-move 状态、按钮高亮、终局 HQ 指引、地图点击下令、沿途交战和选择面板文案，但 HUD 按钮数组漏掉 `.attackMove`，使 README 和 flow 里记录的 `AMOV` 控制入口与实际 HUD 不一致。

本轮目标是把已有 `AMOV` 按钮恢复到命令条，使 v4.8 的终局 HQ 指引和长期 attack-move 操作真正可达。

## 目标

1. 恢复底部 HUD 的 `AMOV` 按钮。
   - 在 `layoutHUD()` 的 `actions: [HudAction]` 中加入 `.attackMove`。
   - 建议位置放在 `.holdPosition` 后、生产按钮前，使 `HOLD` / `AMOV` 作为基础军队命令相邻。
2. 保持已有 attack-move 行为不变。
   - 点击 `AMOV` 仍进入 `isSettingAttackMove` pending。
   - 没有作战单位时仍提示 `Select combat units for attack move.`。
   - 玩家点击地图后仍由 `issueAttackMoveOrder(...)` / `issueFormationMove(... attackMove: true)` 下达命令。
   - v4.8 的终局 Red HQ 已知提示和短暂 HQ 标记继续只在 `isKnownToFaction(..., observer: .player)` 成立时出现。
3. 文档同步。
   - `README.md`：明确 `AMOV` 是底部 HUD 命令按钮。
   - `md/flow/flow.md`：HUD / 输入流程同步 `AMOV` 实际暴露在命令条。
   - `md/flow/flowchart.md`：如需，简短同步 `HOLD` / `AMOV` 命令入口。
4. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不改变 attack-move 的寻路、编队、目标搜索、警戒半径、目标合法性或沿途交战语义。
- 不改变 `HudAction.attackMove` 标题、subtitle、颜色、高亮、pending 清理或消息文案，除非必须修复按钮可见性。
- 不重做 HUD 系统，不新增第三行按钮、滚动条、分页、教程弹窗或新输入模式。
- 不改变 AI、生产队列、单位数值、战斗、胜负、任务完成条件、迷雾、潜艇侦测或支援技能。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。
