# v4.63 / AI 主攻波已感知摘要措辞

## 版本号

v4.63

## 版本分配依据

v4.62 已完成多选 HOLD Carrier 警戒接触类型和优先目标摘要，并通过 `origin/main` 云端 artifact 验收。`update_log.md` 的 v4.62 遗留事项明确后续可微调 AI wave HUD 的“已感知子集”措辞。本轮继续 v4 海军 / AI / HUD 可读性小步增量，版本递增为 v4.63。

## 背景

当前 Red routine assault wave 成功下发后，`updateEnemyAssaultWaveSummary(for:)` 会先用 `isKnownToFaction(..., observer: .player)` 过滤玩家已知子集，再把该子集交给 `enemyAssaultWaveSummary(for:)`。HUD 状态行约 12 秒显示 `R# F# Wave n Lx/Ax/Nx ...`，然后恢复 `AI <difficulty>`。

该摘要的数量 `n` 并不是完整 Red wave 的总规模，而是玩家当前已感知 / 已知的 wave 子集规模。`Wave n` 容易被误读为完整波次规模，本轮只调整玩家可见 label，让它表达“已看见的推进单位数量”。

## 目标

- 将 HUD Red 状态行中 AI 主攻波摘要的首段从 `Wave n` 调整为更明确的 `Seen n`。
- 保持后续组成格式不变：`Lx/Ax/Nx`，非零时追加 `CVc`、`Hh`、`Jj`。
- 保持摘要来源不变：只来自 `knownWave = wave.filter { isKnownToFaction($0, observer: .player) }`。
- 保持摘要过期不变：约 12 秒后恢复 `AI <difficulty>`。
- 同步 README 和 `md/flow/flow.md`，写清这是最近玩家可感知 wave 子集的 `Seen n` 摘要。

## 非目标

- 不改变 AI 主攻波选择、目标排序、护航门槛、低血撤退、Carrier guard wing、生产、支援或移动。
- 不改变 `knownWave` 过滤条件，不扩大玩家可见信息，不显示隐藏单位、目标坐标、目标位置、完整波次数量或隐藏组成。
- 不新增 HUD 行、按钮、状态机、事件队列、音效、地图 ping 或顶部消息。
- 不改变 `enemyAssaultWaveSummaryDuration`、`lastEnemyAssaultWaveSummary` 保存 / 清理逻辑或 `SKRM` 重置逻辑。
- 不更新 `md/test/test.md`，本轮不改变测试流程。
