# v4.17（空军生产按钮来源提示）

## 版本号

v4.17

## 版本分配依据

- `update_log.md` 当前最新正式玩法版本为 v4.16「支援目标面板资产状态」。
- `md/prompt/v4（海军航母）/` 已连续围绕海军、航母、支援、任务和 HUD 可读性做小步增强。
- v4.16 遗留事项中已把「航母生产来源提示」列为后续可推进方向。

## 背景

当前生产相关事实：

- `EntityKind.canProduce(_:)` 允许 Airfield 和 Carrier 生产 Helicopter / Fighter。
- `queueBuild(kind:faction:showFeedback:)` 通过 `productionSource(for:faction:)` 选择生产来源，缺 air source 时给玩家提示 `Requires Airfield or Carrier.`。
- `productionSource(for:faction:)` 对玩家优先使用当前选中的可生产来源；若未选中可用来源，则选择队列最短的可用来源，队列相同按实体 ID。
- Carrier 是移动空军生产平台，能生产 Helicopter / Fighter，完成 air build order 时会显示 deck launch 反馈，并使用 Carrier rally point。
- 单选 Carrier 的 selection info 已显示 `Deck builds Helicopter/Fighter` 和 `Rally set/unset  Queue ...`。
- 当前 HUD 生产按钮 subtitle 对所有 build action 都只显示 `$<cost>`，所以玩家在 `HELI` / `JET` 按钮上看不出当前会从 Airfield 还是 Carrier 出机，也看不出没有 air source 时按钮虽然可点但会被拒绝。
- 人工要求不得用本地测试、本地 build、本地 static check 或 `git diff --check` 作为验收，只认 GitHub Actions artifact。

## 目标

1. 增强 `HELI` / `JET` 按钮 subtitle 的生产来源可读性。
   - 当 `productionSource(for: .helicopter, faction: .player)` 返回 Airfield 时，`HELI` subtitle 显示类似 `AF $1200`。
   - 当返回 Carrier 时，显示类似 `CV $1200`。
   - `JET` 同理，显示类似 `AF $1500` 或 `CV $1500`。
   - 当没有 operational Airfield 或 Carrier 可生产对应空军时，显示 `need AF/CV` 或等价短文案。
2. 必须复用现有 `productionSource(for:faction:)` 口径。
   - 不要在 HUD helper 中重写一套 air source 搜索规则。
   - 不要把未完工、死亡、敌方、中立或不能生产该单位的实体计入来源。
   - 保持选中 Airfield / Carrier 时的玩家优先来源逻辑与实际 `queueBuild` 一致。
3. 保持其他生产按钮 subtitle 不变。
   - `HMV`、`AA`、`TANK`、`ART`、`MECH`、`SHIP`、`SUB`、`CV` 仍显示 `$<cost>`。
   - 本轮不新增 land / naval source hint，避免扩大范围。
4. 保持生产执行链路完全不变。
   - 不改变 `EntityKind.canProduce(_:)`、`requiredFactory`、费用、建造时间或来源条件。
   - 不改变 `queueBuild(kind:faction:showFeedback:)`、`productionSource(for:faction:)`、`canQueueBuild(kind:faction:)`、`updateBuildOrders(dt:)`、`spawnPoint(for:from:)`、`showCarrierLaunch(...)`。
   - 不改变玩家或 AI 的生产行为、资金扣除、队列排序、rally point、航母出机反馈、AI build pattern 或 ASW / anti-air 评估。
5. HUD 布局保持不变。
   - 不新增按钮、行、图标或控件。
   - 不改变按钮尺寸、字体、位置、颜色或高亮规则。
   - 文案要短，适合现有 subtitle 行宽。

2. 修改 `DesertFrontline/GameScene.swift`。

推荐实现方向：

- 在 `subtitle(for:)` 的 buildKind 分支中调用新的只读 helper，例如：

```swift
if let kind = action.buildKind {
    return productionButtonSubtitle(for: kind)
}
```

- 新增 helper，例如：

```swift
private func productionButtonSubtitle(for kind: EntityKind) -> String {
    switch kind {
    case .helicopter, .fighter:
        guard let source = productionSource(for: kind, faction: .player) else {
            return "need AF/CV"
        }
        return "\(source.kind.shortCode) $\(kind.cost)"
    default:
        return "$\(kind.cost)"
    }
}
```

- 上面只是示例，不要求逐字照抄；但必须保持 air source hint 来自 `productionSource(for:faction:)`。
- 不要在 helper 中检查资金。资金不足但 source 存在时，仍显示来源和价格，点击后仍由 `queueBuild` 给出资金不足提示。
