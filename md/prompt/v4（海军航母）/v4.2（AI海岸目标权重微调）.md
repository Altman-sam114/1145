# v4.2（AI海岸目标权重微调）

## 版本号

v4.2

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新玩法版本为 v4.1「航母舰载机弹射反馈增强」。
- v4 阶段聚焦海军 / 航母深化。v4.0 强化 Sonar Buoy 专职反潜，v4.1 强化 Carrier launch 反馈。
- 本轮继续推进海岸争夺，但只做 AI 目标评分微调：当 Red 已具备可攻击结构的海军压力时，更重视玩家海岸基础设施。

## 背景

当前 AI 已经能生产海军、补建 Shipyard / Sonar Buoy / Coastal Battery，也会把玩家结构作为 routine assault 目标。`enemyStrategicAssaultScore(_:)` 对 Sonar Buoy、Coastal Battery 等结构已有固定加分，`targetScore(for:target:)` 也会让 Battleship 偏向结构、Carrier 偏向 Shipyard / Airfield / Carrier。

但海岸争夺还不够明确：当 Red 已经拥有 Battleship / Carrier 等可攻击结构的 operational 海军时，AI 没有额外把玩家 Shipyard、Sonar Buoy、Coastal Battery 视为海岸战役目标。本轮只通过评分让既有 AI 更倾向压制玩家海岸生产、声呐与岸防。

## 目标

1. 新增小型判定 / 评分 helper。
   - 判断目标是否为海岸基础设施：`Shipyard`、`Sonar Buoy`、`Coastal Battery`。
   - 判断某阵营是否有可攻击结构的 active naval pressure：存活、operational、非结构、domain 为 naval、damage > 0，且可攻击 Shipyard / Sonar Buoy / Coastal Battery 至少一种的海军单位。
2. 调整 Red routine assault 的战略结构目标评分。
   - 在 `enemyStrategicAssaultScore(_:)` 中，当 target 是玩家海岸基础设施且 Red 有可攻击结构的 active naval pressure 时，给额外加分。
   - 该加分只影响 Red 选择已知玩家结构目标，不绕过 `isKnownToFaction(..., observer: .enemy)`。
3. 调整具体单位的 tactical target 评分。
   - 在 `targetScore(for:target:)` 中，当 attacker 是 Red naval combat unit 且 target 是玩家海岸基础设施时，给额外加分。
   - Battleship / Carrier / Submarine 等仍必须满足 `attacker.kind.canAttack(target.kind)` 和 `isKnownToFaction(target, observer: attacker.faction)`。
   - 不让 Coastal Battery 攻击潜艇，不改变任何 `canAttack` 规则。
4. 文档同步。
   - `README.md`：Enemy AI 描述中加入会在具备海军压力时优先压制玩家海岸基础设施。
   - `md/flow/flow.md`：AI 小节同步目标评分行为。
   - `md/flow/flowchart.md`：AI 节点简短同步海岸目标权重。
5. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

关键源码事实：

- `enemyWaveObjective()` 会优先旗点、油井，再用 `enemyStrategicAssaultTarget()` 找玩家已知结构。
- `enemyStrategicAssaultTarget()` 已过滤 `target.faction == .player`、`target.isAlive`、`target.kind.isStructure` 和 `isKnownToFaction(target, observer: .enemy)`。

## 验收标准

- 当 Red 有可攻击结构的 active naval pressure 时，Red 已知玩家 Shipyard / Sonar Buoy / Coastal Battery 的 strategic assault score 提升。
- Red 可攻击结构的 naval combat unit 对已知且可攻击的玩家 coastal infrastructure tactical score 提升。
- AI 仍不能选择未知玩家结构；不绕过 `isKnownToFaction`。
- 不改变 `canAttack`、单位数值、生产、AI build pattern、动态 AA / ASW、支援技能、潜艇侦测、建造海岸规则、路径、伤害、XP、胜负或 Xcode/workflow。
- README、flow、flowchart 与真实行为一致，不宣称新增地图目标、旗点或海岸任务。
- 本轮提交已 push 到 `origin/main` 并触发 GitHub Actions。
- Agent C 能下载最新 run artifact 并核对 manifest / JUnit / xcodebuild.log / failure summary / xcresult。

## 风险和禁止项

- 风险：加分过大可能让 AI 忽略旗点和 HQ；加分只应影响结构目标阶段，旗点 / 油井优先级仍由 `enemyWaveObjective()` 保持。
- 风险：误改 `strategicValue(of:)` 会影响 HUD value、repair score、所有目标评分；本轮默认用局部加分。
- 风险：绕过 `isKnownToFaction` 会造成 AI 全图作弊，禁止。
- 风险：改 `canAttack` 会破坏岸防 / 潜艇 / 海军克制边界，禁止。
- 禁止使用非 `Altman-sam114` 的 GitHub 账号伪装 push、CI 或 artifact 验收。
- 禁止下载大体积历史 artifact、模型、测试数据或无关产物。
