# v4.6（摧毁生产任务奖励）

## 版本号

v4.6

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.5「混合军种任务奖励」。
- v4.3 到 v4.5 已经把任务链推进到海岸据点、混合军种和对应一次性资源奖励。
- 本轮继续沿同一任务奖励表扩展，只给 `Break Red Production` 增加一个首次完成奖励，不引入完整战役奖励系统、支援冷却、免费技能、科技树、AI 新能力或新地图。

## 背景

`Break Red Production` 位于 `Deploy Combined Arms` 之后、`Destroy Red HQ` 之前，要求玩家摧毁 Red 的 War Factory、Airfield、Shipyard 和 Carrier 等生产来源。它是主攻阶段的关键里程碑，完成后玩家通常需要补兵、维修或组织最后一波 HQ 攻势。

v4.4 和 v4.5 已经通过 `missionReward(for:)`、`updateMissionProgress()` 和 `completedMissionStages` 建立了一条可复用的一次性奖励链路。本轮只扩展奖励表和 HUD 文案，让摧毁生产后的终局推进更清晰。

## 目标

1. 为 `Break Red Production` 增加一次性玩家资源奖励。
   - 奖励金额固定为 `$900`。
   - 只在 `.breakRedProduction` 首次完成并写入 `completedMissionStages` 时发放。
   - 使用现有 `changeMoney(for: .player, by:)`。
2. 扩展 `missionReward(for:)`。
   - `.secureCoast` 保持 `600`。
   - `.combinedArms` 保持 `800`。
   - `.breakRedProduction` 返回 `900`。
   - 其他阶段仍返回 `0`。
   - 不新增新的持久状态；仍依赖 `completedMissionStages` 防止重复发奖。
3. 更新任务 HUD 文案。
   - `missionDetail(for: .breakRedProduction)` 在未完成时保留剩余生产来源数量，并追加简短 `+$900`。
   - 完成态文案也可简短包含 `+$900`，但不得变成长段说明。
4. 保持任务完成消息逻辑。
   - `updateMissionProgress()` 继续以最后完成阶段为标题并附本轮累计奖励。
   - 如果同一帧连续完成多个阶段，奖励仍在循环内按对应阶段发放，消息显示最后完成阶段标题和本轮累计奖励。
5. 文档同步。
   - `README.md`：任务面板描述中说明 Secure Coast、Deploy Combined Arms 和 Break Red Production 有一次性资源奖励。
   - `md/flow/flow.md`：同步 `missionReward(for:)` 当前奖励表。
   - `md/flow/flowchart.md`：任务节点简短同步海岸 / 混编 / 破生产奖励。
6. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不为 `Secure Oil`、`Capture Frontline` 或 `Destroy Red HQ` 增加奖励。
- 不新增完整战役奖励 UI、奖励列表、教程弹窗、支援冷却缩短、免费支援、免费单位、生产加速、科技树、升级系统、HUD 按钮或新地图。
- 不新增单位、建筑、外部素材或依赖。
- 不改变 `Break Red Production` 的完成条件或 `enemyProductionCount()` 的统计范围。
- 不改变任何单位/建筑的 cost、HP、damage、range、cooldown、vision、sonar、buildTime、canAttack、footprint 或建造规则。
- 不改变生产队列、AI build pattern、AI 海岸目标权重、动态 AA / ASW、support powers、潜艇侦测、迷雾合法性、路径、damage / XP 结算或胜负条件。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。

关键源码事实：

- `MissionStage` 当前顺序为 `secureOil`、`captureFrontline`、`secureCoast`、`combinedArms`、`breakRedProduction`、`destroyHQ`。
- `missionDetail(for:)` 负责当前任务 HUD 详情。
- `enemyProductionCount()` 统计 Red 剩余生产来源，用于 `Break Red Production` 完成条件和 HUD 文案。
- `isMissionStageComplete(.breakRedProduction)` 当前由 `enemyProductionCount() == 0` 判定。
- `updateMissionProgress()` 会 while 连续完成已满足条件的阶段，完成时通过 `missionReward(for:)` 发钱，并显示最后完成阶段标题加本轮累计奖励。
- `completedMissionStages.insert(stage)` 是阶段首次完成边界；该集合在 `restartSkirmish()` 中清空。
- `changeMoney(for:by:)` 是玩家 / AI 资金变化的共享入口。

## 风险和禁止项

- 风险：如果在 `missionDetail` 或每帧 HUD 刷新里发钱，会造成重复奖励；奖励必须只发生在 `updateMissionProgress()` 的阶段首次完成路径。
- 风险：如果改 `enemyProductionCount()` 或完成条件，会改变任务目标范围；本轮禁止。
- 风险：奖励过高可能放大终局滚雪球；本轮固定 `$900`，仅作为摧毁生产后的 HQ 推进补给。
- 风险：任务文案过长会挤压 HUD；保持短句。
- 禁止使用非 `Altman-sam114` 的 GitHub 账号伪装 push、CI 或 artifact 验收。
- 禁止下载大体积历史 artifact、模型、测试数据或无关产物。
