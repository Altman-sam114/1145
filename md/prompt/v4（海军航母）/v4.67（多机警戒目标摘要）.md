# v4.67 / 多机警戒目标摘要

## 版本号

v4.67

## 版本分配依据

v4.66 已完成单选 bound HEL/JET 的 `CV GUARD` 合法目标状态，并通过 `origin/main` 最新云端 artifact 验收。v4.66 遗留事项仍建议继续细化 Carrier guard wing 命令 UI。本轮选择多选 bound HEL/JET 的只读摘要作为最小可验证增量，版本递增为 v4.67。

## 背景

当前单选已绑定 Carrier guard anchor 的 HEL/JET 可以显示 `CV GUARD Dn Tgt XXX Type  Guard ...`。但多选这些 bound HEL/JET 且未选中 HOLD Carrier 时，摘要仍只有 `CV GUARD n Hn/Jn Dm`，不能提示这组 guard aircraft 当前是否有合法近域接触。为保持单选 / 多选 HUD 口径一致，本轮给多选摘要追加只读优先目标短码和类型。

## 目标

- 多选玩家 bound Carrier guard HEL/JET 且未选中 HOLD Carrier 时，如果当前存在合法 Carrier guard contact，在 `CV GUARD n Hn/Jn Dm` 摘要后追加 `Tgt <shortCode> <type>`。
- 目标选择必须复用现有合法性：
  - `carrierGuardAnchor(for:)`
  - `carrierGuardPriorityTarget(for:)`
  - `carrierGuardTargetPriority(for:target:)`
  - `carrierGuardContactType(for:)`
- 没有合法 contact 时保持现有 `CV GUARD n Hn/Jn Dm` 文案。
- 该状态行只读展示，不写入 `attackTarget`、命令、迷雾或 AI 状态。
- 同步 README 和 `md/flow/flow.md` 当前事实说明。

## 非目标

- 不新增 CAP、巡逻、截击、护航命令或状态机。
- 不改变 `isCarrierGuardContact(...)`、`carrierGuardPriorityTarget(for:)`、`carrierGuardTargetPriority(...)` 的合法性和排序。
- 不改变单选 bound HEL/JET 文案、Carrier 单选 / 多选 `GW` 摘要、`Eng n`、release feedback、guard ring、AI、战斗、生产、支援、任务或胜负。
- 不更新 `md/test/test.md`，本轮不改变测试流程。

## 当前架构依据

- `groupSelectedCarrierGuardWingSummary(for:)` 是多选 bound HEL/JET 且未选中 HOLD Carrier 的摘要入口。
- `carrierGuardPriorityTarget(for:)` 已复用 `isCarrierGuardContact(...)`，保证目标满足可攻击、Carrier 近域、HOLD 站位警戒半径和 `isKnownToFaction(...)`。
- `carrierGuardTargetPriority(for:target:)` 和距 anchor Carrier 距离可用于多个 wing 之间稳定选出优先目标。
- `carrierGuardContactType(for:)` 已统一生成 Air / Sea / Sub / Ground 类型。

## 实现步骤

1. 同步 main：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - 确认工作区没有无关改动。
2. 修改 `DesertFrontline/GameScene.swift`：
   - 在 `groupSelectedCarrierGuardWingSummary(for:)` 保留现有选中条件、距离和 `Hn/Jn` 组成。
   - 新增一个只读 helper，例如 `carrierGuardWingTargetSuffix(for guardWings: [GameEntity]) -> String`：
     - 遍历每个 wing，读取 `carrierGuardAnchor(for:)` 和 `carrierGuardPriorityTarget(for:)`。
     - 对候选 target 复用 `carrierGuardTargetPriority(for:target:)`，再按 target 到 anchor Carrier 距离、target id 稳定打平。
     - 找到候选时返回 ` Tgt \(target.kind.shortCode) \(carrierGuardContactType(for: target))`。
     - 没有候选时返回空字符串。
   - `groupSelectedCarrierGuardWingSummary(for:)` 返回 `CV GUARD n Hn/Jn Dm\(targetSuffix)`。
   - 不写入任何实体、命令、攻击目标或迷雾字段。
3. 修改 `README.md`：
   - 在 Carrier guard / selection panel 描述中补充：多选 bound guard aircraft 若有合法 guard contact，会显示 `Tgt XXX Air/Sea/Sub/Ground`。
4. 修改 `md/flow/flow.md`：
   - 在 HUD 选择信息段补充：多选 bound HEL/JET 的 `CV GUARD n Hn/Jn Dm` 摘要只读追加当前合法优先 contact。
5. 不需要修改 `md/flow/flowchart.md`，除非实际图中需要同步该短文案。
