# v4.86 / 坦克模型与地面战云端探针

## 版本号与分配依据

版本号为 `v4.86`。最新已验收版本为 v4.85；本轮继续在 v4 海军航母与综合战场表现阶段递增一个小版本，只交付一个可独立验收的地面模型增量。

## 背景

Desert Stormfront 官方 trailer、TouchGameplay 长玩法视频和 Noble Master 公开截图都强调俯视小比例单位的轮廓辨识：主战坦克即使尺寸紧凑，仍能从履带、炮塔、长炮管和顶部明暗层次快速识别。当前 Desert Frontline 的 HEL/JET 已有细化几何和专属云端截图，但 Tank 仍只有车体、炮塔、炮管三层，且现有八张 simulator 截图全部使用 air focus，缺少地面模型专属证据。

## 目标

1. 在现有 `addLandUnitBody(...)` 内细化 Blue / Red Tank，增加履带、负重轮、侧裙/前装甲、炮塔环、舱盖、阵营化光学件、炮管根部与制退器等俯视细节。
2. 新增只在 `DESERT_CI_CAPTURE_MODE=1` 且 `DESERT_CI_CAMERA_FOCUS=land` 时运行的确定性地面捕获编排，集中展示双方坦克并保持普通开局不变。
3. GitHub Actions 增加第九次 simulator launch，生成 `simulator-land-combat.png`，继续检查 PID 存活并把文件写入 artifact manifest。
4. 同步 README、flow、flowchart 和测试规范。

## 非目标

- 不修改坦克 HP、速度、攻击、射程、冷却、生产价格或 veterancy。
- 不修改 AI、寻路、碰撞、编队、迷雾、经济、任务或胜负。
- 不细化其他陆军、海军或空军模型，不新增外部依赖或素材。
- 不运行本地 Xcode build、本地 simulator 或本地探针。
- 不修改或提交工作区已有的 `project.pbxproj` DEVELOPMENT_TEAM 改动。

## 当前架构依据

- `GameScene.swift` 是实体、渲染、CI capture 和战斗循环事实核心。
- Tank 继续由 `GameEntity` / `EntityKind.tank` 管理；新几何必须留在现有 entity base 子树，自动继承移动、镜像、选择、迷雾和销毁。
- CI capture 只通过环境变量进入，`isCICaptureMode` 会冻结实时循环；普通玩家启动不得设置这些变量。
- 云端验证由 `.github/workflows/ci-results.yml` 构建、启动、截图并上传未加密 artifact。

## 实现步骤

1. 重构 Tank 的 SpriteKit shape 组合，保持现有 footprint 与朝向语义。
2. 为 `initialCameraPosition()` 增加 `land` focus，并在 `prepareCICaptureScene()` 分流到 `prepareCILandCaptureScene()`。
3. 地面 capture 使用现有/临时 Tank 组成双方对峙样例，刷新 zPosition、选择和迷雾；不得进入普通启动。
4. workflow 的 launch helper 接受 camera focus 参数，保留原八次 air launch，再追加一次 land launch 与截图。
5. artifact 清单、测试文档和玩家可见模型事实同步更新。

## 文档更新要求

## 验收标准

- 普通开局、Tank gameplay 属性与所有旧命令保持不变。
- 新节点不超出可接受的原 footprint 视觉范围，不遮挡 label、队旗或生命条。
- `DESERT_CI_CAMERA_FOCUS=land` 只影响 CI capture；旧 air/HUD/command 截图继续生成。
- 最新 `origin/main` commit、Actions run 和 artifact manifest SHA 完全一致。
