# v4.64 / HOLD 按钮航母警戒提示

## 版本号

v4.64

## 版本分配依据

v4.63 已完成 AI 主攻波已感知摘要措辞，并通过 `origin/main` 云端 artifact 验收。v4.63 遗留事项建议继续细化 Carrier guard wing 命令 UI。本轮选择最小可验证的 HUD 操作提示增量：让 `HOLD` 按钮 subtitle 根据当前玩家选择提示 Carrier guard wing 语义，版本递增为 v4.64。

## 背景

当前 `HOLD` 按钮标题固定为 `HOLD`，subtitle 固定为 `guard`。实际同一个按钮存在三种玩家可见语义：

- 普通移动单位：进入 HOLD guard。
- 选中玩家 Carrier：Carrier HOLD 并尝试绑定附近 HEL/JET 为 `CV GUARD` wing。
- 选中已绑定 Carrier guard 的 HEL/JET 且未选中 Carrier：ordinary HOLD 会释放旧 Carrier guard anchor。

选择面板和顶部消息已经能解释结果，但按钮本身在点击前没有区分这些语义。本轮只补短 subtitle，降低移动端误操作成本。

## 目标

- 在 `HOLD` 按钮 subtitle 中显示上下文短提示：
  - 当前选择包含玩家 Carrier 时显示 `CV GW`。
  - 当前选择包含已绑定有效 Carrier guard anchor 的玩家 HEL/JET，且没有选中 Carrier 时显示 `CV rel`。
  - 其他情况仍显示 `guard`。
- 只读复用当前选择集合和 `carrierGuardAnchor(for:)`。
- 不改变 `HOLD` 按钮标题、颜色、布局、点击处理、命令执行或反馈消息。
- 同步 README 和 `md/flow/flow.md` 当前事实说明。

## 非目标

- 不新增按钮、HUD action、命令模式、CAP、巡逻、截击、护航命令或状态机。
- 不改变 `issueHoldPositionOrder(units:)`、`assignCarrierGuardWing(for:)`、`clearCarrierGuardAnchor(for:)`、`carrierGuardAnchor(for:)`、`isCarrierGuardContact(...)`。
- 不改变移动、攻击、AMOV、HOLD、Carrier guard wing 绑定 / 释放、AI Carrier guard、战斗、迷雾、生产、支援、任务、胜负或 `SKRM` 重置。
- 不更新 `md/test/test.md`，本轮不改变测试流程。

## 当前架构依据

- `subtitle(for:)` 是 HUD button subtitle 的集中入口，`holdPosition` 当前固定返回 `guard`。
- `updateHUD()` 会刷新 `hudButtonSubtitleLabels`，因此只改 subtitle helper 就能随选择变化更新。
- `selectedMobilePlayerUnits()` 是 HOLD 实际执行使用的玩家存活移动单位集合，适合作为 subtitle 的同口径读取入口。
- `carrierGuardAnchor(for:)` 已能判断 HEL/JET 是否仍有有效 Carrier guard anchor，且不写入状态。
- `issueHoldPositionOrder(units:)` 才是命令语义核心；本轮不得修改。

## 实现步骤

1. 同步 main：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - 确认工作区没有无关改动。
2. 修改 `DesertFrontline/GameScene.swift`：
   - 在 `subtitle(for:)` 的 `.holdPosition` 分支中改为调用 `holdButtonSubtitle()`。
   - 新增 `private func holdButtonSubtitle() -> String`：
     - 读取 `selectedMobilePlayerUnits()`。
     - 若其中存在 `kind == .carrier`，返回 `CV GW`。
     - 否则若存在玩家 HEL/JET 且 `carrierGuardAnchor(for:) != nil`，返回 `CV rel`。
     - 否则返回 `guard`。
   - helper 只读，不写任何实体字段。
3. 修改 `README.md`：
   - 在控制说明或功能清单中补充：`HOLD` subtitle 会在选中 Carrier 时显示 `CV GW`，在选中已绑定 guard aircraft 时显示 `CV rel`，只是点击前提示。
4. 修改 `md/flow/flow.md`：
   - 在输入 / 命令或 HUD 段补充：`HOLD` subtitle 根据当前选择显示 `guard` / `CV GW` / `CV rel`，只读提示不改变命令语义。
5. 不需要修改 `md/flow/flowchart.md`，除非实际图中需要同步提到按钮 subtitle。
