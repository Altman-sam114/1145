# v4.68 / AMOV 按钮航母警戒释放提示

## 版本号

v4.68

## 版本分配依据

v4.67 已完成多选 bound HEL/JET 的 `CV GUARD` 合法目标摘要，并通过 `origin/main` 最新云端 artifact 验收。v4.64 已让 `HOLD` 按钮在 Carrier guard 相关选择下显示 `CV GW` / `CV rel` 预告，但 `AMOV` 仍固定显示 `push`。本轮选择 `AMOV` 按钮的只读释放预告作为小范围命令 UI 补齐，版本递增为 v4.68。

## 背景

当前玩家对已绑定 Carrier guard anchor 的 HEL/JET 下达 attack-move 时，执行路径会清理这些 wing 的 `guardAnchorCarrierID`，并通过现有 `carrierGuardReleaseSuffix(for:)` 给出释放反馈。选中 HOLD Carrier 后下达 AMOV 也会让 Carrier 退出 HOLD，导致其 guard anchor 对 bound wing 不再有效。按钮点击前仍只显示 `push`，缺少与 `HOLD` 按钮一致的 `CV rel` 预告。

## 目标

- 当当前玩家选择会因为 `AMOV` 而释放 Carrier guard 关系时，`AMOV` 按钮 subtitle 从 `push` 改为 `CV rel`。
- 只读判断范围：
  - 选中玩家可战斗移动单位中存在已绑定有效 Carrier guard anchor 的 HEL/JET。
  - 或选中玩家 HOLD Carrier，且该 Carrier 当前有 bound guard wing。
- 其他选择仍显示 `push`。
- 本轮只改变按钮 subtitle，不改变 AMOV 执行、release feedback、guard anchor 写入 / 清理、AI、战斗、迷雾或生产。

## 非目标

- 不新增 CAP、巡逻、截击、自动携带 wing、自动释放未选中 wing 或新命令模式。
- 不改变 `issueAttackMoveOrder(...)`、`issueFormationMove(...)`、`carrierGuardReleaseWing(for:)`、`carrierGuardReleaseSuffix(for:)`、`clearCarrierGuardAnchor(for:)` 的行为。
- 不改变 `HOLD` 按钮 subtitle、Carrier `GW` 摘要、bound HEL/JET `CV GUARD` 摘要、AI Carrier guard wing、战斗、支援、任务或胜负。
- 不更新 `md/test/test.md`，本轮不改变测试流程。

## 当前架构依据

- `subtitle(for:)` 的 `.attackMove` 分支当前固定返回 `push`。
- `holdButtonSubtitle()` 已按当前选择只读判断 Carrier guard 相关语义，适合在附近新增 `attackMoveButtonSubtitle()`。
- `selectedMobilePlayerUnits()` 是按钮 subtitle 的当前玩家选择来源。
- `carrierGuardAnchor(for:)` 能验证 HEL/JET 是否仍有有效 anchor Carrier。
- `boundCarrierGuardWing(for:)` 能判断 HOLD Carrier 是否当前有 bound guard wing。
- `issueFormationMove(... attackMove: true)` 已在执行 AMOV 时对参与单位调用 `clearCarrierGuardAnchor(for:)` 并保留 release feedback。
