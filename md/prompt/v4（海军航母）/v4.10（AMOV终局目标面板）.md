# v4.10（AMOV 终局目标面板）

## 版本号

v4.10

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.9「恢复 AMOV 按钮入口」。
- v4.7 到 v4.9 已围绕终局 Red HQ 情报、`AMOV` 目标提示和 HUD 按钮入口形成一条操作闭环。
- 本轮继续做一个 HUD-only 小增强：当玩家已经 armed `AMOV`，并且终局 Red HQ 已被玩家合法认知时，选择信息面板显示 Red HQ HP 与最近作战单位到 HQ 的近似距离，帮助玩家组织最终推进。

## 背景

当前 `AMOV` armed 时，选择信息面板固定显示通用 attack-move 文案：

- `Combat units N`
- `Seek radius ...`
- `Engage visible enemies en route`
- `Tap map to push formation`

v4.8 已在顶部消息和战场短暂标记中提示已知 Red HQ，但选择面板仍不区分终局 HQ 已知场景。由于选择面板会在 pending 模式期间持续显示，本轮让它在合法条件下提供更稳定的终局目标摘要。

## 目标

1. 增强 `AMOV` armed 状态下的选择信息面板。
   - 条件：`isSettingAttackMove == true`，`activeMissionStage() == .destroyHQ`，且 `playerKnownEnemyHQ()` 非空。
   - 显示 Red HQ 当前 / 最大 HP。
   - 显示选中玩家存活机动作战单位到 Red HQ 的最近近似距离，例如 `Nearest approx 820`。
   - 仍显示 `Tap map to push formation`，明确玩家仍需点击地图下令。
2. 未知 HQ 或非终局阶段保持原有通用 `AMOV` 面板。
   - 不显示隐藏 HQ HP、位置、距离或提示。
   - 不泄露未被 `isKnownToFaction(..., observer: .player)` 认可的 Red HQ 信息。
3. 保持命令语义不变。
   - 不自动选择 Red HQ。
   - 不自动 attack-move 到 HQ。
   - 不改变 `handleHudAction(.attackMove)`、`issueAttackMoveOrder(...)`、`issueFormationMove(...)`、目标合法性、寻路、AI 或战斗。
4. 文档同步。
   - `README.md`：任务 / 控制说明同步 `AMOV` armed 时终局已知 HQ 会在选择面板显示目标摘要。
   - `md/flow/flow.md`：HUD / 输入说明同步该面板信息复用 `playerKnownEnemyHQ()`。
   - `md/flow/flowchart.md`：如需，简短同步 HUD 已知 HQ AMOV 目标面板。
5. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不改变 `AMOV` pending、按钮高亮、顶部消息、短暂 HQ 标记或地图点击下令语义。
- 不改变 `missionDetail(for:)`、任务完成条件、胜负触发、HQ HP / 数值、单位伤害、射程、冷却或 AI。
- 不显示隐藏 Red HQ 的位置、距离、HP、标记或路线。
- 不把距离说成路径距离；只能标为 approximate / approx 的世界点直线距离。
- 不重做选择信息面板布局，不新增面板行数、滚动、分页或新 HUD 按钮。
- 不改变 `isKnownToFaction(...)`、`visibleTiles`、`exploredTiles`、`supportRevealTiles`、潜艇侦测或支援侦察逻辑。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。

2. 修改 `DesertFrontline/GameScene.swift`。

建议改动：

- 在 `selectionInfoContent(for:)` 的 `isSettingAttackMove` 分支中：
  - 先计算 `combatUnits = selected.filter { $0.faction == .player && $0.isAlive && !$0.kind.isStructure && $0.kind.damage > 0 }`。
  - 若 `activeMissionStage() == .destroyHQ` 且 `let redHQ = playerKnownEnemyHQ()`：
    - 计算最近近似距离：`combatUnits.map { $0.node.position.distance(to: redHQ.node.position) }.min()`。
    - 返回 title `ATTACK MOVE`，4 行内显示：
      - `Combat units N`
      - `Red HQ HP current/max`
      - 有距离时 `Nearest approx X`，无 combat unit 时可显示 `Select combat units`
      - `Tap map to push formation`
  - 否则保留原有通用四行。

可选小 helper：

- `private func attackMoveHQTargetRows(combatUnits:redHQ:) -> [String]`
- 只负责格式化选择面板行，不保存状态、不改变命令。

必须保持：

- 不调用 `enemyHQ()` 直接显示信息；必须通过 `playerKnownEnemyHQ()`。
- 不改 `handleHudAction(.attackMove)` 和 `issueAttackMoveOrder(...)`。
- 不改任何 AI / combat / fog / movement 逻辑。
