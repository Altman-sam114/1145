# v4.31（高价值海军护航缺口类型提示）

## 版本号

v4.31

## 版本分配依据

- 当前 `md/prompt/v4（海军航母）/` 已存在到 v4.30，本轮继续在 v4 海军 / 航母 / HUD 可读性阶段追加小范围增量。
- v4.18-v4.20 已建立 Carrier / Battleship 护航状态、护航半径圈和多选摘要；v4.26 已让单选行显示 `Escort x/y OK` 或 `Escort x/y Need n`。
- 本轮不重复数量缺口，而是在不足时追加一个很短的建议护航类型，帮助玩家理解缺口应优先补 Air、Sea 或 Ground 友军。

## 背景

当前 `DesertFrontline/GameScene.swift` 中：

- `singleSelectionInfo(for:)` 组装单选面板。
- Carrier 行通过 `carrierDeckCapabilityLine()` 拼接 `highValueNavalEscortLine(for:)`。
- Battleship 行在移动 / domain 文案后追加 `highValueNavalEscortLine(for:)`。
- `highValueNavalEscortRequirement(for:)` 定义 Battleship 需要 1 个 escort，Carrier 需要 2 个 escort。
- `nearbyNavalEscortCount(for:)` 统计同阵营、存活、operational、非结构、可攻击、非高价值海军且在 `highValueNavalEscortRadius` 内的 escort。
- `groupHighValueNavalEscortSummary(for:)` 只做多选数量摘要。

现有 `Escort x/y Need n` 能说明数量不足，但不能给玩家一个轻量组成方向。本轮只增加只读文案建议，不改变统计、AI、移动、生产或战斗。

## 目标

在单选 Carrier / Battleship 的护航行中，当护航不足时追加缺口类型提示：

- 若缺口存在且附近没有 air escort，优先显示 `Air`。
- 否则若附近没有 naval escort，显示 `Sea`。
- 否则若附近没有 land escort，显示 `Ground`。
- 若三类都已有但数量仍不足，显示 `Mix`。
- 示例：
  - `Escort 0/2 Need 2 Air`
  - `Escort 1/2 Need 1 Sea`
  - `Escort 1/1 OK`
- 达标时仍只显示 `Escort x/y OK`，不追加类型。
- 类型提示必须复用现有 escort 合法性口径，只能在合法 escort 集合内判断 domain 覆盖。

## 非目标

- 不改变护航需求数量、护航半径、护航合法性或 AI 护航门槛。
- 不改变 `groupHighValueNavalEscortSummary(for:)` 多选摘要。
- 不实现自动护航、follow、CAP、巡逻、编队锁定、战斗加成、侦测加成或新 HUD action。
- 不改变玩家移动、攻击、HOLD、AMOV、生产、Carrier deck launch、rally、支援、迷雾、声呐、任务、胜负或 `SKRM` 重置。
- 不新增外部依赖、素材、测试框架、截图、探针或 Xcode project 配置。
- 不做无关重构、格式化或大范围 HUD 布局调整。
