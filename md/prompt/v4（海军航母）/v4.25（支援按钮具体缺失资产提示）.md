# v4.25（支援按钮具体缺失资产提示）

## 版本号

v4.25

## 版本分配依据

- 人工明确指定本轮版本为 v4.25。
- 当前 `md/prompt/v4（海军航母）/` 已存在到 v4.24，本轮继续在 v4 海军 / 航母 / HUD 可读性阶段追加一个小范围 UI 文案增量。
- 本轮只优化支援按钮 subtitle 的缺资产提示，不改变支援技能规则、费用、冷却、资产需求、AI 或 pending 流。

## 背景

当前支援技能按钮由 `subtitle(for:)` 分发到 `supportButtonSubtitle(for:)`。该函数优先显示玩家支援技能冷却秒数；无冷却时，如果缺少 operational support asset，统一显示 `need asset`；资产满足时显示价格。

同一文件中已经存在 `supportAssetRequirementLabel(for:)`，集中定义各支援技能的资产口径：

- `SCAN` / `.reconSweep`：`HQ/RAD`
- `REPR` / `.fieldRepair`：`HQ/MECH`
- `AIRS` / `.airStrike`：`AF/CV`
- `BARR` / `.navalBarrage`：`BB/CV`

`supportAssetLine(for:faction:)` 已复用该 helper，在支援 pending 目标面板里显示 `Asset ready: ...` 或 `Need ...`。本轮要让底部支援按钮在缺资产时也显示具体缺什么，避免玩家只能看到泛化的 `need asset`。

## 目标

只修改 `DesertFrontline/GameScene.swift` 中支援按钮 subtitle 的缺资产文案：

- 保持冷却优先：支援技能处于冷却时仍显示 `Xs`，不能被缺资产文案覆盖。
- 无冷却且缺少 operational asset 时，把通用 `need asset` 改成具体资产提示。
- 具体文案必须复用或对齐 `supportAssetRequirementLabel(for:)` 的口径：
  - `SCAN` 缺资产显示 `need HQ/RAD`
  - `REPR` 缺资产显示 `need HQ/MECH`
  - `AIRS` 缺资产显示 `need AF/CV`
  - `BARR` 缺资产显示 `need BB/CV`
- 资产满足时仍显示价格，例如 `$300`、`$420` 等现有格式。
- 保持 `supportAssetLine(for:faction:)` 的 pending 面板行为不变。

## 非目标

- 不修改 `supportIssue(for:faction:)` 的任何逻辑或错误文案。
- 不修改 `hasOperationalSupportAsset(for:faction:)` 的资产判定。
- 不修改 `executeSupportPower(...)`、`selectSupportPower(...)`、支援 pending、点击目标、拒绝标记或消息流程。
- 不修改支援技能费用、冷却、半径、伤害、维修量或效果。
- 不修改 AI 使用支援技能、AI 资产判定、AI 目标选择或任何生产 / 战斗逻辑。
- 不新增外部依赖、测试框架、截图、素材或自动化探针。
- 不借机重构 HUD、按钮布局、颜色、字体、命中区域或 `SupportPower` 枚举。
- 不更新 README、flow、flowchart、test 或 update_log，除非实现中发现当前文档与真实代码明显冲突且必须说明给人工决定。

## 当前架构依据

关键文件：`DesertFrontline/GameScene.swift`

当前相关函数和职责：

- `subtitle(for:)`：HUD action subtitle 总分发；支援技能走 `supportButtonSubtitle(for:)`。
- `supportButtonSubtitle(for:)`：支援按钮底部 subtitle 来源。当前顺序是冷却秒数 -> `need asset` -> 价格。
- `supportAssetRequirementLabel(for:)`：支援资产短码口径的唯一 helper，当前返回 `HQ/RAD`、`HQ/MECH`、`AF/CV`、`BB/CV`。
- `supportAssetLine(for:faction:)`：支援 pending 目标面板资产状态，已经复用 `supportAssetRequirementLabel(for:)`。
- `supportIssue(for:faction:)`：真正阻止支援执行的规则链路，按冷却、资金、资产顺序返回问题；本轮不得修改。
- `hasOperationalSupportAsset(for:faction:)`：资产是否满足的共享判定；本轮不得修改。

当前文档事实：

- `md/flow/flow.md` 和 README 已记录支援按钮会显示冷却、缺资产或价格，目标面板显示 `Asset ready` / `Need`。
- 本轮行为变化很小，只把缺资产按钮文案从泛化提示变成具体短码提示；除非 Agent B 认为玩家可见清单必须精确更新，否则不要扩展文档范围。
