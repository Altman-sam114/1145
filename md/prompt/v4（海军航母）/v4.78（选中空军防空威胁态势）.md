# v4.78 / 选中空军防空威胁态势

## 版本号与依据

v4.78。v4.77 已由最终 `origin/main` run `29129764399` artifact 验收通过，空军编队间距和同域避让已完成；当前空军虽有导弹、投影和编队，但玩家选中飞机后不能快速判断已知 SAM / AA 的真实覆盖风险。

## 视觉参考

- Noble Master 官方 Desert Stormfront itch.io 产品页及其 5 张 1280x800 截图：`https://noblemaster.itch.io/desert-stormfront`。
- 官方截图在密集海空军和陆战场景中使用橙色顶标、红色地面目标标记、高对比投影和紧凑左侧命令面板，优先保证战场目标辨识。
- 本轮借鉴高对比、短信息的表达方式，不复制素材、原 UI 布局或美术资产。

## 目标

- 玩家选中至少一架 Helicopter / Fighter 时，识别真实射程覆盖这些选中空军的已知敌方 SAM Site / AA Truck。
- 对每个合法威胁显示低填充红橙射程圈、三重橙色顶标和核心警示点。
- 选择状态行与普通单选/多选信息显示紧凑 `AA THRn Sn Mn Cx/y`（`S`=SAM、`M`=mobile AA、`C`=covered aircraft）；没有合法威胁时显示 `CLEAR`，HOLD / guard / attack-move 的原状态行不得被覆盖。
- 威胁合法性复用 `isKnownToFaction(...)`、`canAttack(...)`、`attackRange`、operational 和存活状态。
- CI air scene 选中 Blue 3 架空军，并在真实覆盖距离内布置 Red SAM 与 AA Truck，云端截图直接验收。

## 非目标

- 不修改 SAM / AA 射程、伤害、冷却、目标获取、AI、移动、防空优先级或生产。
- 不显示未知迷雾内敌军，不扩大玩家视野，不新增雷达侦测或预警系统。
- 不新增规避命令、自动绕飞、航线规划、音效、外部依赖或 XCTest target。
- 不运行本地测试、build、模拟器、探针或本地静态检查。
- 不提交用户 `project.pbxproj` 团队号改动。

## 实现步骤

1. 为 `GameEntity` 增加预创建的防空威胁覆盖节点和顶标节点，只对 SAM / AA 配置几何。
2. 增加已知防空威胁收集和真实覆盖判断 helper；只接受 enemy、alive、operational、known、SAM / AA 和 `canAttack` 合法目标。
3. `updateHUD()` 根据当前选中玩家空军刷新威胁节点，避免持久残留或迷雾泄露。
4. 单选和多选信息复用同一威胁集合生成紧凑摘要。
5. CI air capture 选择 Blue HEL/JET，并增加 Red SAM / AA 作为确定性截图证据。
6. 更新 README、flow、flowchart、test 和提示词。
7. commit `v4.78: 增强空军防空威胁态势`，直推 `origin/main`，只做云端验证。
