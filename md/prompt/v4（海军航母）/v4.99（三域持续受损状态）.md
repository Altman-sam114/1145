# v4.99 三域持续受损状态

## 版本号与依据

`v4.99`。当前正式版本为 `v4.98`；人工要求继续优先改善 UI 和战斗细节，本轮以一个统一、可云端截图验收的增量补齐地面、空中和水面单位的持续受损反馈，因此顺序递增。

## 背景

- 参考对象固定为百度百科 lemma id `4042982` 对应的 Noble Master Games《沙漠风暴 Desert Stormfront》。已重新核对 Noble Master 官方 press kit、trailer、海战图 `screen_dsf_v02_10.png` / `screen_dsf_v02_11.png` 与陆战图 `screen_dsf_v02_13.png`；官方画面保持舰船 / 飞机轮廓和弹道清楚，并以短黑烟与小火点表达受损地面资产。本项目只借鉴信息层级与反馈原则，不复制素材或 UI。
- 当前已有瞬时爆炸、伤害飘字、红色生命条和多选 Wounded / Critical 统计，但单位在两次命中之间没有贴身持续状态，玩家必须反复读取生命条才能快速识别高风险单位。

## 目标

1. 为非结构、非 Submarine 实体预创建并挂载 `damageStateNode`，不按帧或每次受击创建常驻节点。
2. HP `<= 62%` 显示低遮挡受损状态：短暗烟、局部焦痕；HP `<= 35%` 升级为危急状态：更浓的第三段烟、紧凑橙红火点和更强焦痕。
3. 地面、空中、水面使用不同挂点和尺度：地面靠发动机舱，空中靠尾部，水面靠舰体偏舷；节点随实体移动和镜像，不遮挡模型主体、生命条、旗帜或命令 UI。
4. 敌方状态必须随实体父节点共同经过战争迷雾；Submarine 不显示烟火，保持声呐检测、`revealedUntil` 和隐藏链路。
5. `updateHealthBar(...)` 统一刷新生命条和受损状态，真实攻击、支援伤害、Mechanic / Field Repair 恢复都共享同一阈值和恢复隐藏行为。
6. 单选 HP 行在两档阈值追加 `DMG` / `CRIT`，不新增面板、不改变四行布局；多选沿用既有 Wounded / Critical 统计。
7. 新增 capture-only `damage-state` 场景：同屏显示 29% Tank、32% Fighter 和 56% Battleship，验证三域挂点、两档强度、模型轮廓、生命条、选择圈、多选摘要和小地图。
8. GitHub Actions 扩展为十九次独立启动，新增 `simulator-damage-state.png`。

## 非目标

- 不改 HP、damage、armor、range、cooldown、repair rate、AI 撤退阈值、目标选择、经济、生产、任务或胜负。
- 不新增持续伤害、燃烧 DOT、移动减速、命中位置模拟、物理粒子、声音或外部素材。
- 不给 Submarine 增加水面烟火，不绕过迷雾显示未知敌方损伤。
- 不运行本地 Xcode build、本地 simulator 或本地玩法探针。

## 架构与实现步骤

1. 保留人工 `project.pbxproj` Team ID 改动，不纳入提交。
2. 在 `GameEntity` 增加实体自有的 damage root、烟节点集合、火点和焦痕节点；`addEntity(...)` 内按 domain 一次性配置。
3. `configureDamageStateNode(...)` 只负责形状、挂点、层级和初始隐藏；结构与 Submarine 早退。
4. `updateDamageStateVisual(...)` 只读 HP ratio，统一切换隐藏、透明度、第三段烟和危急火点；由 `updateHealthBar(...)` 调用，修复超过阈值时自动降级或隐藏。
5. `singleSelectionInfo(...)` 的 HP 行复用相同 62% / 35% 阈值追加 `DMG` / `CRIT`。
6. 在 camera focus 早退前分流 `damage-state`，编排三域 Blue 单位、HP、位置、投影 / 航迹、选择和 fog；普通 App 不进入该写入分支。
7. workflow 的清理、launch 与 manifest 加入第十九张截图。
8. 同步 README、flow、flowchart 与 test；Agent C 云端 artifact 验收通过后再更新 `update_log.md`。

## 风险与禁止项

- 禁止每帧创建烟节点或在每次受击后遗留 world-level 常驻效果。
- 禁止用大面积烟火遮住模型、血条、旗帜、目标标记或 HUD；受损反馈必须贴身且明显小于死亡爆炸。
- 禁止未知敌方或未侦测 Submarine 通过新状态泄露位置。
- 禁止新增 DOT、二次扣血、减速或改变现有战斗平衡。
- 禁止把 capture-only 位置与 HP 写入普通启动。
- 禁止用旧 artifact、本地截图或文字汇报代替最新 `origin/main` 云端结果包。
