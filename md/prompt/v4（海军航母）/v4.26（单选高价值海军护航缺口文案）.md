## 背景

当前 `DesertFrontline/GameScene.swift` 中，单选信息由 `singleSelectionInfo(for:)` 组装：

- Carrier 单选时，`rows[2]` 组合 `carrierDeckCapabilityLine()` 和 `highValueNavalEscortLine(for:)`，当前显示类似 `Deck HEL/JET  Escort 1/2`。
- Battleship 单选时，`highValueNavalEscortLine(for:)` 追加到移动 / domain 行，当前显示类似 `Move 70  Naval  Escort 0/1`。
- `highValueNavalEscortRequirement(for:)` 定义需求：Battleship 需要 1 个 escort，Carrier 需要 2 个 escort。
- `nearbyNavalEscortCount(for:)` 统计同阵营、存活、operational、非结构、可攻击、非当前实体、非 Battleship / Carrier、距离不超过 `highValueNavalEscortRadius` 的附近单位。
- `groupHighValueNavalEscortSummary(for:)` 已经为多选摘要计算缺口，满足时显示 `HV Navy x/y escorted`，不足时追加 `Need n`。

当前单选 `Escort x/y` 能表达数量，但不能直接告诉玩家是否达标或还缺几个 escort。本轮目标是复用现有 requirement 和 nearby count 口径，在单选高价值海军行追加 `OK` 或 `Need n`。

## 目标

只增强单选 Carrier / Battleship 的选择面板护航行：

- 当附近 escort 数量满足需求时，显示 `Escort x/y OK`。
- 当附近 escort 数量不足时，显示 `Escort x/y Need n`。
- `n` 必须是 `max(0, requirement - nearby)`。
- Carrier 示例：
  - 满足：`Deck HEL/JET  Escort 2/2 OK`
  - 不足：`Deck HEL/JET  Escort 1/2 Need 1`
- Battleship 示例：
  - 满足：`Move 70  Naval  Escort 1/1 OK`
  - 不足：`Move 70  Naval  Escort 0/1 Need 1`
- 必须复用现有 `highValueNavalEscortRequirement(for:)` 和 `nearbyNavalEscortCount(for:)`，不能复制、分叉或重新定义 escort 统计口径。
- 保持单选 Carrier 的 deck capability、rally set/unset、queue 状态和其他选择信息不变。
- 保持单选 Battleship 的 HP、攻击 / 射程 / 视野、移动 / domain、老兵、HOLD / AMOV / repair 等状态优先级不变。

## 非目标

- 不改变 `groupHighValueNavalEscortSummary(for:)` 的多选摘要文案、优先级或统计方式。
- 不改变 escort radius ring 显示规则、颜色、半径、节点层级或刷新链路。
- 不改变 Red AI 高价值海军护航门槛、routine attack-move wave、provisional wave、reservation、混编主攻或低血保护逻辑。
- 不新增自动护航、follow 命令、CAP / 巡逻、战斗加成、侦测加成或新 HUD action。
- 不改变玩家移动、攻击、HOLD、AMOV、生产、Carrier deck launch、rally point、支援技能、迷雾、声呐、任务、胜负或 `SKRM` 重置。
- 不新增外部依赖、素材、测试框架、截图、探针或 Xcode project 配置。
- 不做无关重构、格式化或大范围 HUD 布局调整。

## 当前架构依据

当前文档事实：

- `README.md` 已写明 Carrier / Battleship 单选面板会报告附近 escort status，多选会显示 high-value naval escort summaries，选中玩家高价值海军会显示 escort radius ring。
