## 背景

当前 `DesertFrontline/GameScene.swift` 中已有以下事实：

- `groupSelectionInfo(for:)` 会把 `groupAntiSubmarineSummary(for:)` 的结果追加到多选战斗摘要行，当前格式类似 `Dmg 90  Max rng 250  ASW 2  Sonar 1/340`。
- `groupAntiSubmarineSummary(for:)` 当前统计 active ASW attackers 和 active sonar sensors，并显示 sonar sensor 数量及最大 sonar range。
- `isActiveSonarSensor(_:)` 判断 active sonar sensor：`kind.hasSonar`、实体存活，且结构必须 `isOperational`。
- `sonarContactCount(for:)` 已为单个玩家 active sonar sensor 统计接触数：敌方存活 Submarine、`isKnownToFaction(contact, observer: .player)` 为 true，且处于该 sensor 的 `sonarRange` 内。
- `sonarInfoLine(for:)` 当前对玩家 active sonar sensor 显示 `Sonar <range> Ctc <count>`，对非玩家 sensor 不显示玩家 contact count。
- `isKnownToFaction(_:, observer:)` 是玩家已知敌方实体的核心边界；敌方潜艇还会进入 `isSubmarineDetected(_:, by:)`，依赖 `revealedUntil` 或 active sonar sensor 范围。
- `refreshSelection()` / `shouldShowSonarCoverage(for:)` 已只为玩家选中的 active sonar sensor 显示声呐覆盖圈，该视觉反馈不改变 fog 或 detection。

当前缺口是：玩家多选含多个 sonar sensor 时，面板能看到 sensor 数量和最大 range，却看不到编队当前合计接触了多少个玩家已知敌方潜艇；如果多个 sensor 覆盖同一潜艇，也需要避免重复计数。

## 目标

在多选选择面板的现有 ASW / sonar 摘要后追加去重 contact count：

- 当多选中包含至少一个玩家 active sonar sensor 时，在现有 `ASW n  Sonar n/maxRange` 后追加 `Ctc n`。
- 示例：`Dmg 120  Max rng 250  ASW 3  Sonar 2/340  Ctc 1`。
- `Ctc n` 必须是编队级去重统计：
  - contact 必须是敌方存活 `Submarine`。
  - contact 必须满足 `isKnownToFaction(contact, observer: .player)`。
  - contact 必须处于至少一个已选玩家 active sonar sensor 的 `sonarRange(for: sensor.kind)` 内。
  - 多个 sensor 覆盖同一潜艇时必须按实体 `id` 去重，只计 1 次。
- 只有玩家 active sonar sensor 参与 contact 统计。
- 只选 ASW attacker 但没有玩家 active sonar sensor 时，不显示 `Ctc`。
- 如果多选中有 sonar-capable 但非玩家或非 active sensor，不应触发玩家 `Ctc`。
- 保持当前 `ASW n  Sonar n/maxRange` 摘要位置、行数和多选面板结构；优先在现有 `groupAntiSubmarineSummary(for:)` 链路内完成。

- 如果 helper 修改 `revealedUntil` 或 fog 状态，会把只读 HUD 变成玩法逻辑变更，必须避免。
- 选择面板行较短，文案过长可能溢出；使用 `Ctc n`。
