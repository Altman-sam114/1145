# v4.52 / 航母警戒翼队多选组成提示

## 版本号

v4.52

## 版本分配依据

上一轮已完成并云端验收 v4.51“航母翼队组成提示”，单选 Carrier 普通翼队与 HOLD guard wing、以及多选 HOLD Carrier `CV GW` 摘要都已经显示 `Hn` / `Jn` 组成。本轮继续同一海军航母 HUD 细化方向，只补齐“多选已绑定 Carrier guard 的 HEL/JET”摘要中的飞机组成。

## 背景

当前玩家多选被有效 Carrier guard anchor 绑定的 Helicopter / Fighter，且没有选中 HOLD Carrier 时，HOLD 状态行显示：

```text
CV GUARD n Dm
```

其中 `n` 是有效绑定飞机数量，`Dm` 是最近 anchor Carrier 距离。该摘要能确认 guard 归属，但不能快速分辨当前多选的是 Helicopter、Fighter 还是混合翼队。

## 目标

- 将多选绑定 guard 飞机摘要从 `CV GUARD n Dm` 扩展为 `CV GUARD n Hx Jy Dm`。
- `Hn` / `Jn` 口径复用现有 `carrierAirWingCompositionSuffix(for:)`：
  - 只统计有效绑定到 Carrier guard anchor 的玩家 Helicopter / Fighter。
  - Helicopter 数量大于 0 时显示 `Hn`。
  - Fighter 数量大于 0 时显示 `Jn`。
  - 数量为 0 的类型省略。
- 保留最近距离 `D`，示例：`CV GUARD 2 H1 J1 D180`、`CV GUARD 2 H2 D140`、`CV GUARD 1 J1 D95`。

## 非目标

- 不改变单选 bound HEL/JET 的 `CV GUARD Dn  Guard ...` 状态行。
- 不改变多选 HOLD Carrier 的 `CV GW x/y OK/Need Hn/Jn Cn Eng n` 摘要优先级。
- 不新增巡逻、CAP、截击、战斗加成、AI 作弊侦察或自动目标状态机。
- 不改变 Carrier guard anchor 分配、脱离、站位保持、目标搜索、迷雾合法性、生产来源或集结点行为。

## 当前架构依据

- `GameScene.swift` 集中管理选择信息和 HUD 文案。
- `groupSelectedCarrierGuardWingSummary(for:)` 当前负责多选 bound HEL/JET 摘要。
- `carrierAirWingCompositionSuffix(for:)` 已为 Carrier wing / guard wing 输出一致的 `Hn` / `Jn` 后缀，应复用以避免新口径。
- `groupSelectionInfo(...)` 中 `carrierGuardWingSummary ?? selectedCarrierGuardWingSummary ?? "Guard ..."` 保证多选 HOLD Carrier 摘要优先于多选 bound HEL/JET 摘要，本轮必须保持。

## 实现步骤

1. 在 `groupSelectedCarrierGuardWingSummary(for:)` 中先收集有效 selected guard wing 实体，而不是只收集距离。
2. 有效条件保持不变：玩家阵营、`helicopter` 或 `fighter`、`carrierGuardAnchor(for:)` 成功。
3. 用有效实体计算最近 anchor Carrier 距离。
4. 用 `carrierAirWingCompositionSuffix(for:)` 生成组成后缀。
5. 返回 `CV GUARD \(count)\(compositionSuffix) D\(nearestDistance)`。
6. 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 中关于多选 guard 飞机摘要的当前事实。
