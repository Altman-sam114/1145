# v4.88 / 悍马模型与沙地扬尘

## 版本号与分配依据

版本号为 `v4.88`。v4.87 已完成 Artillery 模型和炮口反馈；本轮继续沿百度百科 lemma id `4042982` 所确认的 Noble Master Games《沙漠风暴 Desert Stormfront》核心单位表，聚焦 Humvee 和沙漠地面行进表现。

## 背景

该游戏的官方资料与玩家实机画面把 Humvee 作为高机动侦察单位，并通过沙漠底色、轮式轮廓和行进尘迹维持俯视辨识。当前 Desert Frontline 的 Humvee 只有车体、座舱、枪管和两个轮子；地面移动循环已有 Naval wake 与 Air shadow 的预创建节点模式，却没有 Land dust，导致车辆在沙地上缺少速度感和地图接触感。

## 目标

1. 在现有 footprint 内细化 Blue / Red Humvee：四轮、前后保险杠、引擎盖、装甲座舱、阵营化风挡、车顶枪座 / 枪管、前灯和天线。
2. 为所有机动陆军实体增加预创建 `landDustNode`，包含双胎迹 / 履带印和确定性尘团；沙地 / 油地移动时显示，道路、停止和非陆军实体隐藏。
3. `landDustNode` 作为实体子节点自动继承移动、镜像、迷雾和销毁，不按帧新增 effectsLayer 节点。
4. land CI capture 加入双方 Humvee，并强制冻结其方向化扬尘证据；同步 README、flow、flowchart 和 test。

## 非目标

- 不修改 Humvee 或其他陆军的 HP、速度、视野、伤害、射程、冷却、生产价格或 veterancy。
- 不新增真实粒子系统、物理轮胎、永久地面贴花、路面抓地、漂移或碰撞。
- 不修改 AI、寻路、地形速度倍率、迷雾集合、经济、任务、胜负或 HUD 页面。
- 不新增外部依赖 / 素材，不运行本地 Xcode build、simulator 或玩法探针。
- 不修改或提交用户已有的 `project.pbxproj` DEVELOPMENT_TEAM 改动。

## 当前架构依据

- `GameEntity` 已持有 `navalWakeNode` 与 `airShadowNode`，是本轮 `landDustNode` 的所有权范式。
- `configureEntityNode(...)` 负责预建实体视觉子树；`updateMovement(dt:)` 统一更新玩家与 AI 移动。
- `animateIdle(...)` 已隐藏 Naval wake，可同时隐藏 Land dust。
- 敌方实体节点由现有迷雾链路整体隐藏，子节点不得脱离实体进入独立残留层。
- `DESERT_CI_CAMERA_FOCUS=land` 只在 capture mode 编排确定性截图。

## 实现步骤

1. 给 `GameEntity` 增加 `landDustNode`。
2. `configureLandDustNode(...)` 为 land domain 预建双轨迹和三组尘团，默认隐藏并挂到实体节点。
3. `updateLandDust(...)` 按实际移动方向、镜像、地形与实体 id 更新旋转和透明度；普通调用只允许 sand / oil，CI 可显式 `forceVisible`。
4. 在 `updateMovement(...)` 调用，并在 `animateIdle(...)` 隐藏。
5. 重构 Humvee shape 组合，保持碰撞和 gameplay footprint 不变。
6. land capture 布置 Blue / Red Humvee、设置方向并强制显示 dust，保留 Tank / Artillery / 炮口 / 爆炸证据。
