# v4.32（多选高价值海军护航缺口类型摘要）

## 版本号

v4.32

## 版本分配依据

- 当前 `md/prompt/v4（海军航母）/` 已存在到 v4.31，本轮继续在 v4 海军 / 航母 / HUD 可读性阶段追加小范围增量。
- v4.31 已让单选 Carrier / Battleship 在护航不足时显示 `Need n Air/Sea/Ground/Mix`。
- 多选摘要当前仍只显示 `HV Navy x/y escorted  Need n`，玩家框选多个高价值海军时看不到缺口类型。本轮把同一只读提示扩展到多选摘要。

## 背景

当前 `DesertFrontline/GameScene.swift` 中：

- `highValueNavalEscortLine(for:)` 用于单选 Carrier / Battleship，已显示 `Escort x/y OK` 或 `Escort x/y Need n Type`。
- `nearbyNavalEscorts(for:)` 是 v4.31 新增的合法 escort 集合 helper。
- `highValueNavalEscortNeedType(for:)` 根据合法 escort 集合返回 `Air`、`Sea`、`Ground` 或 `Mix`。
- `groupHighValueNavalEscortSummary(for:)` 用于多选摘要，当前只统计满足数量和总缺口数量，不显示缺口类型。
- `groupSelectionInfo(for:)` 会在没有 HOLD / AMOV 状态时，把高价值海军多选摘要放在第四行。

本轮只增强多选 HUD 文案，复用 v4.31 的合法 escort 集合和缺口类型 helper，不改变护航规则或 AI。

## 目标

增强多选高价值海军摘要：

- 多选包含玩家 Carrier / Battleship 且至少一个高价值海军护航不足时，摘要显示 `HV Navy x/y escorted  Need n Type`。
- `n` 仍是所有被选中玩家高价值海军的总缺口数。
- `Type` 由护航不足的高价值海军缺口类型汇总得出：
  - 如果所有不足项的类型一致，显示该类型，例如 `Air`。
  - 如果存在多种类型，显示 `Mix`。
- 所有高价值海军均达标时，仍显示 `HV Navy x/y escorted`，不追加缺口类型。
- 类型判断必须复用 `nearbyNavalEscorts(for:)` 和 `highValueNavalEscortNeedType(for:)` 或等价共享口径，不能复制第二套 escort 合法性判断。

## 非目标

- 不改变单选 `Escort x/y OK` / `Need n Type` 文案。
- 不改变多选摘要的显示优先级：HOLD / AMOV 仍优先于高价值海军摘要。
- 不改变 escort requirement、escort radius、escort coverage ring、AI 高价值海军护航门槛或 attack wave。
- 不实现自动护航、follow、CAP、巡逻、编队锁定、战斗加成、侦测加成或新 HUD action。
- 不改变玩家移动、攻击、HOLD、AMOV、生产、Carrier deck launch、rally、支援、迷雾、声呐、任务、胜负或 `SKRM` 重置。
- 不新增外部依赖、素材、测试框架、截图、探针或 Xcode project 配置。
