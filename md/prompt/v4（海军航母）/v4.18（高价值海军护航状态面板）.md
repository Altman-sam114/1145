# v4.18（高价值海军护航状态面板）

## 版本号

v4.18

## 版本分配依据

- `update_log.md` 当前最新正式玩法版本为 v4.17「空军生产按钮来源提示」。
- `md/prompt/v4（海军航母）/` 已连续围绕海军、航母、支援、任务和 HUD 可读性做小步增强。
- v4.17 遗留事项中已把「航母 CAP / 巡逻、航母护航行为」列为后续可推进方向；当前 AI 已有 Battleship / Carrier 高价值海军护航门槛，但玩家 HUD 还没有对应可读信息。
- 本轮是一个很小的 selection info 文案增量：玩家单选 Carrier 或 Battleship 时显示附近友方护航数量和需求，例如 `Escort 1/2`，不改变 AI、玩家命令、移动、战斗、生产、支援、迷雾或护航判定的执行效果。

## 背景

当前高价值海军和选择面板相关事实：

- `enemyAssaultWave(from:)` 会调用 `canJoinEnemyAssaultWave(_:provisionalWave:)`。
- `canJoinEnemyAssaultWave` 对 Red 的 Battleship / Carrier 使用高价值海军门槛：
  - Battleship 需要 1 个 escort。
  - Carrier 需要 2 个 escort。
  - 若 provisional wave 足够大则放行，否则用 `enemyEscortCount(near:in:)` 统计 620 范围内的 escort。
- `isEnemyAssaultEscort(_:)` 当前只把 Red 存活、非结构、可攻击、operational、非占点 reservation、非高价值海军的单位视为 escort。
- 玩家单选 Carrier 时，`singleSelectionInfo(for:)` 目前用：
  - row 2：`Deck builds Helicopter/Fighter`
  - row 3：`Rally set/unset  Queue ...`
- 玩家单选 Battleship 时仍走普通移动单位信息，显示 HP、攻击 / 射程 / 视野、移动 / domain、老兵 XP。
- 当前玩家看不到 Carrier / Battleship 附近是否已有足够护航单位，只能从地图视觉判断。
- 人工要求不得用本地测试、本地 build、本地 static check 或 `git diff --check` 作为验收，只认 GitHub Actions artifact。

## 目标

1. 增强单选 Carrier 的选择信息面板。
   - row 2 或等价位置显示 deck 能力和护航状态，例如 `Deck HEL/JET  Escort 1/2`。
   - 护航需求为 2。
   - row 3 继续显示 rally set/unset 和 queue 状态。
2. 增强单选 Battleship 的选择信息面板。
   - 显示护航状态，例如 `Escort 0/1` 或等价短文案。
   - 护航需求为 1。
   - 不删除 HP、攻击 / 射程 / 视野、移动 / domain 和老兵信息的核心价值；如需压缩，只压缩 HUD 文案，不改变数据来源。
3. 护航统计必须是只读 HUD helper。
   - 统计同阵营、存活、非结构、可攻击、operational、非当前实体、非 Battleship / Carrier 的附近单位。
   - 推荐半径沿用现有 AI escort 语义：620。
   - 玩家单选 Blue Battleship / Carrier 时统计 Blue 护航；如果未来代码路径显示其他 faction，也应统计同 faction，不能硬编码敌我混用。
   - 不要求使用 `isEnemyAssaultEscort(_:)`，因为该函数含 Red 占点 reservation 语义；可以新增通用只读 helper，但逻辑必须与本轮描述一致。
4. 保持所有执行链路不变。
   - 不改变 `enemyAssaultWave`、`canJoinEnemyAssaultWave`、`isEnemyAssaultEscort`、`enemyEscortCount` 或任何 AI 决策。
   - 不改变玩家移动、attack-move、hold、rally、生产、支援、战斗、迷雾、潜艇侦测、任务或胜负。
   - 不改变 Carrier launch、Carrier wing strike、BuildOrder、productionSource 或 queueBuild。
5. 文案要短、稳定、适合现有 selection panel 行宽。
   - 推荐文案：`Escort 0/1`、`Escort 2/2`、`Deck HEL/JET  Escort 1/2`。
   - 可显示真实计数大于需求，例如 `Escort 3/2`，也可 clamp 到需求上限；实现必须在提示词输出中说明选择。

2. 修改 `DesertFrontline/GameScene.swift`。

推荐实现方向：

- 新增常量：

```swift
private let highValueNavalEscortRadius: CGFloat = 620
```

- 新增只读 helper，例如：

```swift
private func highValueNavalEscortRequirement(for kind: EntityKind) -> Int? {
    switch kind {
    case .battleship:
        return 1
    case .carrier:
        return 2
    default:
        return nil
    }
}

private func highValueNavalEscortLine(for entity: GameEntity) -> String? {
    guard let requirement = highValueNavalEscortRequirement(for: entity.kind) else { return nil }
    let count = nearbyNavalEscortCount(for: entity)
    return "Escort \(count)/\(requirement)"
}

private func nearbyNavalEscortCount(for entity: GameEntity) -> Int {
    entities.values.filter { escort in
        escort.faction == entity.faction &&
            escort.isAlive &&
            escort.id != entity.id &&
            !escort.kind.isStructure &&
            escort.kind.damage > 0 &&
            escort.isOperational &&
            highValueNavalEscortRequirement(for: escort.kind) == nil &&
            escort.node.position.distance(to: entity.node.position) <= highValueNavalEscortRadius
    }.count
}
```

- 上面只是示例，不要求逐字照抄；但必须保持 helper 只读，不改变实体状态。
- Carrier 分支建议把 row 2 改成短文案，例如：

```swift
rows[2] = "\(carrierDeckCapabilityLine())  \(highValueNavalEscortLine(for: entity) ?? "")"
```

并可把 `carrierDeckCapabilityLine()` 从 `Deck builds Helicopter/Fighter` 压缩为 `Deck HEL/JET`，以容纳 `Escort x/2`。

- Battleship 可以在普通移动单位行加入护航状态，例如：

```swift
if let escortLine = highValueNavalEscortLine(for: entity) {
    rows[2] = "\(rows[2])  \(escortLine)"
}
```

- 不要调用或修改 AI 的 `enemyEscortCount`，避免把 Red reservation 语义泄漏到玩家 HUD helper。
