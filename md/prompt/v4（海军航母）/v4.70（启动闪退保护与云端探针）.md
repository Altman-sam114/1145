# v4.70 / 启动闪退保护与云端探针

## 版本号

v4.70

## 版本分配依据

v4.69 已修复启动视口空白并通过实现提交的云端 artifact 验收，但用户反馈启动后仍会闪退。该问题阻断可玩性，优先级高于继续扩展 RTS 玩法。本轮作为启动热修和验证体系增强，版本递增为 v4.70。

## 背景

v4.69 将 `GameScene.scaleMode` 改为 `.resizeFill` 并让 SwiftUI 容器同步真实窗口尺寸。SpriteKit 可能在 `GameScene.didMove(to:)` 完成地形初始化前触发 `didChangeSize(_:)`。当前 `didChangeSize(_:)` 会直接 `layoutHUD()`，而 `layoutHUD()` 中的小地图会读取 `terrain[row][col]`。若 `terrain` 仍为空，会出现数组越界并导致启动闪退。现有 CI 只验证 generic iOS build，无法捕捉启动崩溃，因此本轮同时增加云端 iOS Simulator 启动探针。

## 目标

- 修复 `didChangeSize(_:)` 在 `terrain` 尚未初始化时提前布局 HUD 导致的启动闪退风险。
- GitHub Actions 增加 iOS Simulator 启动探针：
  - 构建 simulator app。
  - 安装到可用 iPhone simulator。
  - 启动 `DesertFrontline`。
  - 等待短时间后截图、抓取 App 日志。
  - 确认 `DesertFrontline` 进程仍在运行。
- CI artifact、manifest、JUnit 和 failure summary 必须记录 simulator launch probe 结果。
- 同步 `md/test/test.md` 和 `md/flow/flow.md` 中的验证流程。

## 非目标

- 不运行本地 build、本地静态检查、本地模拟器或本地探针。
- 不修改 gameplay、地图、单位、AI、战斗、HUD 内容、命令、迷雾、任务、胜负或经济规则。
- 不提交用户本地未提交的 `DesertFrontline.xcodeproj/project.pbxproj` 签名设置。
- 不新增 XCTest target 或第三方依赖。

## 当前架构依据

- `GameScene.didMove(to:)` 顺序是设置节点、构建 terrain、绘制地图、生成实体、`layoutHUD()`。
- `GameScene.didChangeSize(_:)` 可能早于完整初始化被 SpriteKit 调用。
- `layoutHUD()` 会调用 `addMinimap(frame:)`，后者遍历 `rows` / `cols` 并读取 `terrain[row][col]`。
- `.github/workflows/ci-results.yml` 已负责 generic iOS build、manifest/JUnit/failure summary 和 artifact 上传，适合扩展 simulator 启动探针。

## 实现步骤

1. 同步 main：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - 确认工作区已有改动归属；本轮不得覆盖用户的无关 `project.pbxproj` 签名设置。
2. 修改 `DesertFrontline/GameScene.swift`：
   - 在 `didChangeSize(_:)` 中 `super.didChangeSize(oldSize)` 后加入初始化保护。
   - 当 `terrain.isEmpty` 时直接返回，不执行 `clampCamera(...)` 和 `layoutHUD()`。
   - 保持 `didMove(to:)` 初始化完成后的 `layoutHUD()` 和后续尺寸变化行为不变。
3. 修改 `.github/workflows/ci-results.yml`：
   - 在 generic iOS build 后新增 `iOS simulator launch probe` step，`continue-on-error: true`。
   - 用 `iphonesimulator` SDK 构建 app。
   - 从可用 iPhone simulator 中选择一个 UDID，boot、install、launch。
   - 等待约 8 秒，输出 `simulator-launch.log`、`simulator-app.log`，尽量保存 `simulator-screenshot.png`。
   - 用 `ps` 检查 `DesertFrontline` 进程仍在运行，失败时让该 step outcome 为 failure。
   - manifest 增加 `simulatorLaunchOutcome` 和 `simulatorLaunchLogPath`。
   - JUnit 增加 `simulator-launch` testcase。
   - failure summary 增加 simulator launch probe 结果和日志路径。
   - 最终 fail gate 把 `simulatorLaunchOutcome` 纳入失败条件。
4. 修改 `md/test/test.md`：
   - 记录云端 simulator launch probe、artifact 文件和 Agent C 验收要求。
5. 修改 `md/flow/flow.md`：
   - 协作验证数据流补充 simulator launch probe 和 `simulator-launch.log`。

- 必须核对 simulator launch probe 的 `simulatorLaunchOutcome=success`、`simulator-launch.log`、`simulator-app.log` 和截图文件是否存在。

## 验收标准

- `didChangeSize(_:)` 在 `terrain` 尚未初始化时不会调用 `layoutHUD()`。
- `didMove(to:)` 初始化完成后的首次 HUD、迷雾、消息显示仍保持。
- GitHub Actions run 对最新 `origin/main` commit 成功。
- artifact manifest 记录 `buildOutcome=success`、`staticChecksOutcome=success`、`projectLintOutcome=success`、`simulatorLaunchOutcome=success`。
- `junit.xml` 包含 `simulator-launch` testcase 且失败数为 0。
- `xcodebuild.log` 包含 `** BUILD SUCCEEDED **`。
- `simulator-launch.log` 显示 simulator build、install、launch 和进程检查完成。
- `simulator-app.log` 和 `simulator-screenshot.png` 已进入 artifact。

## 风险和禁止项

- 风险：云端 simulator launch probe 能捕捉启动闪退，但不能替代人工手感验证；若用户设备仍闪退，下一轮应下载 probe 日志和用户设备日志对比。
- 风险：macOS runner 的 simulator 可用设备名称可能变化；探针必须从 `simctl list devices available --json` 动态选择 iPhone simulator。
- 禁止运行本地测试、本地静态检查、本机 Xcode build、模拟器或探针。
- 禁止修改玩法逻辑、AI、combat、fog、mission、victory、support、production 或 command 行为。
- 禁止提交用户本地无关 `project.pbxproj` 签名改动。
- 禁止用旧 run、旧 artifact、本地输出或文字汇报冒充本轮云端验证。
