# v4.56 / 航母 HOLD 警戒范围空翼可见

## 版本号

v4.56

## 版本分配依据

上一轮 v4.55 已补齐 Carrier guard wing 脱离反馈的 `Hn` / `Jn` 组成。当前选中 bound Carrier guard aircraft 或选中拥有 bound guard wing 的 HOLD Carrier 时会显示 anchor Carrier 的 guard-radius ring；但一艘已经 HOLD、暂时没有绑定 HEL/JET 的玩家 Carrier 不显示该范围。玩家无法直接看到这艘 HOLD Carrier 的警戒半径和未来 wing 防区。本轮把选中 HOLD Carrier 的 guard anchor 范围圈改为始终可见，属于 v4 海军 / 航母操作可读性小步增量。

## 背景

当前相关事实：

- `carrierGuardAnchorCoverageNode` 是每个实体已有的 SpriteKit 范围圈节点。
- `configureCarrierGuardAnchorCoverageNode(for:)` 只为 Carrier 配置 `carrierGuardThreatRadius` 半径的 cyan guard anchor ring。
- `refreshSelection()` 每次刷新选择时调用 `shouldShowCarrierGuardAnchorCoverage(for:)`。
- 当前 `shouldShowCarrierGuardAnchorCoverage(for:)` 对选中的玩家 Carrier 要求：
  - faction 是 player；
  - kind 是 carrier；
  - alive；
  - selected；
  - `holdPosition != nil`；
  - `!boundCarrierGuardWing(for: entity).isEmpty`。
- 这导致 HOLD Carrier 没有 bound HEL/JET 时不显示自身 guard radius，即使该 Carrier 已经处在 HOLD guard anchor 状态。

## 目标

- 玩家选中一艘存活的 HOLD Carrier 时，总是显示该 Carrier 自身的 guard anchor 范围圈。
- 选中 bound Carrier guard HEL/JET 时，仍显示对应 anchor Carrier 的同一个范围圈。
- 多选时仍按现有 `refreshSelection()` / `selectedIDs` 逻辑显示多个有效 anchor Carrier 范围圈。
- 该范围圈仍是 selection-derived visual，只读显示 `carrierGuardThreatRadius`，不改变游戏规则。

## 非目标

- 不改变 Carrier HOLD 绑定 HEL/JET 的条件、数量、排序或 guard anchor 写入。
- 不改变 bound guard wing 的 HUD `GW` 计数、接触数、类型、目标短码、交战数或释放反馈。
- 不改变 `carrierGuardPriorityTarget(for:)`、`isCarrierGuardContact(...)`、`carrierGuardStationPoint(...)`、station keeping、combat、fog、潜艇侦测、AI、生产、支援、任务、胜负或 SKRM 重置。
- 不让未 HOLD Carrier、敌方 Carrier、未选中 Carrier、非 Carrier、未绑定 aircraft 显示这个 guard anchor 范围圈。
- 不新增 CAP、巡逻、截击、新按钮、新状态机或 AI 策略。

## 当前架构依据

- 范围圈显示唯一入口是 `shouldShowCarrierGuardAnchorCoverage(for:)`。
- 被 bound HEL/JET 选中时，当前函数通过 `selectedIDs.compactMap { entities[$0] }` 查找 `carrierGuardAnchor(for: wing)?.id == entity.id`，这条路径应保持不变。
- 选中 Carrier 自身显示范围圈的逻辑只需要调整 `selectedIDs.contains(entity.id)` 分支，不需要改 node 配置或 HUD。
- `carrierGuardAnchorCoverageNode` 只是 SpriteKit visual；显示/隐藏不会写入 `visibleTiles`、`exploredTiles`、`supportRevealTiles`、攻击目标、guard anchor 或 command 状态。

## 实现步骤

1. 修改 `shouldShowCarrierGuardAnchorCoverage(for:)`：
   - 保持开头 `player + carrier + alive` guard。
   - 将选中 Carrier 自身分支改为：`selectedIDs.contains(entity.id) && entity.holdPosition != nil` 即返回 true。
   - 移除 `!boundCarrierGuardWing(for: entity).isEmpty` 要求。
   - 保留 selected bound HEL/JET 显示 anchor Carrier 范围圈的现有分支。
2. 同步 README：
   - Current Features 中 Carrier 描述改为：选中 HOLD Carrier 会显示自身 guard-radius ring，即使当前没有 bound guard aircraft。
   - Selection overlay 描述删除 “HOLD Carriers without bound guard aircraft 不显示” 的旧事实。
3. 同步 `md/flow/flow.md`：
   - `GameEntity` / Carrier / HUD overlay 相关段落改为当前事实：选中 HOLD Carrier 总是显示 guard anchor 范围圈，选中 bound HEL/JET 也显示其 anchor Carrier 范围圈。
   - 明确这是 selection-derived visual，不改变迷雾、战斗、guard contact 或 AI。
4. `md/flow/flowchart.md` 如果没有具体写 “有绑定翼队才显示”，可不改。
