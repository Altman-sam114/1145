# v4.3（建立海岸据点任务阶段）

## 版本号

v4.3

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.2「AI海岸目标权重微调」。
- v4 阶段聚焦海军、航母和海岸争夺。v4.0 强化 Sonar Buoy 专职反潜，v4.1 强化 Carrier launch 反馈，v4.2 让 Red 在具备海军压力时更重视玩家海岸基础设施。
- 本轮继续推进海岸争夺，但只做任务系统小增量：引导玩家建立海岸据点，不新增单位、按钮、奖励、AI 状态或科技系统。

## 背景

当前任务链已有 `Secure Oil`、`Capture Frontline`、`Deploy Combined Arms`、`Break Red Production`、`Destroy Red HQ`。玩家侧虽然已经能建造 Shipyard、Sonar Buoy 和 Coastal Battery，但任务链没有明确要求玩家巩固海岸阵地。

v4.2 已经让 Red 海军更倾向压制玩家海岸基础设施。本轮用一个可验证的小任务阶段承接该方向：要求玩家拥有 2 个已完工海岸资产，计入 Shipyard、Sonar Buoy、Coastal Battery。初始玩家已有 Shipyard，因此该阶段通常从 1/2 开始，推动玩家再建一个 Sonar Buoy 或 Coastal Battery。

## 目标

1. 新增任务阶段。
   - 在 `MissionStage` 中加入 `secureCoast`。
   - 建议放在 `captureFrontline` 之后、`combinedArms` 之前，让玩家先扩张陆上前线，再建立海岸据点，随后进入三军合成。
   - 标题建议为 `Secure Coast`。
2. 新增玩家海岸资产计数。
   - 新增 `coastalAssetCount(for:)` 或同等小 helper。
   - 只统计 `Shipyard`、`Sonar Buoy`、`Coastal Battery`。
   - 必须要求 `faction` 匹配、`isAlive == true`、`isOperational == true`。
   - 未完工建筑不得计数。
3. 接入任务 HUD 和完成判定。
   - `missionStatusContent()` 的总阶段数应自动来自 `MissionStage.allCases.count`。
   - `missionDetail(for:)` 为 `secureCoast` 显示 `Hold 2 coastal assets. X/2 secured.` 或等价简洁文案。
   - `isMissionStageComplete(_:)` 对 `secureCoast` 使用 `coastalAssetCount(for: .player) >= 2`。
4. 文档同步。
   - `README.md`：玩家可见任务清单加入建立海岸据点阶段。
   - `md/flow/flow.md`：任务系统 / 当前流程同步 `secureCoast` 阶段和计数边界。
   - `md/flow/flowchart.md`：任务流程图同步新增阶段。
5. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不新增海岸旗点、奖励、科技树、升级系统、HUD 按钮、教程弹窗或新地图。
- 不新增单位、建筑、外部素材或依赖。
- 不改变 Shipyard、Sonar Buoy、Coastal Battery 或任何单位的 cost、HP、damage、range、cooldown、vision、sonar、buildTime、canAttack、footprint 或建造规则。
- 不改变生产队列、AI build pattern、AI 动态 AA / ASW、AI 海岸目标权重、support powers、潜艇侦测、迷雾合法性、路径、damage / XP 结算或胜负条件。
- 不让未完工建筑完成任务。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。

关键源码事实：

- `MissionStage` 是任务阶段枚举，`CaseIterable` 顺序决定 HUD 中 `OBJ x/y` 的阶段顺序。
- `missionStatusContent()` 使用 `MissionStage.allCases.count` 作为总阶段数。
- `missionDetail(for:)` 负责每个阶段的 HUD 细节文案。
- `activeMissionStage()` 返回第一个未完成阶段。
- `updateMissionProgress()` 会 while 连续完成已满足条件的阶段，并显示最后一个完成阶段消息。
- `isMissionStageComplete(_:)` 是阶段完成判定入口。
- `restartSkirmish()` 已清空 `completedMissionStages`，新增阶段无需新增独立重置状态。
- `constructionIssue(...)` 已限制 Shipyard、Sonar Buoy、Coastal Battery 必须放在 coast，本轮不改建造规则。
- v4.2 已新增 `isCoastalInfrastructure(_:)`，但它位于 AI / combat 评分区域；本轮可复用该语义或新增任务专用 helper，避免为任务读取 AI 状态。

建议改动：

- 在 `MissionStage` 中加入：
  - `case secureCoast`
  - `title` 返回 `Secure Coast`
- 在 `missionDetail(for:)` 中加入：
  - `let secured = min(coastalAssetCount(for: .player), 2)`

- 在 `isMissionStageComplete(_:)` 中加入：
  - `return coastalAssetCount(for: .player) >= 2`
- 新增 helper：
  - `private func coastalAssetCount(for faction: Faction) -> Int`
  - 统计 `entities.values` 中 matching faction、alive、operational、kind 为 `.shipyard` / `.sonarBuoy` / `.coastalBattery` 的实体。

必须保持：

- 不改 `completedMissionStages` 的数据结构。

- 不改海岸建筑放置、AI、战斗、迷雾、生产、支援技能和单位数值。
- 如果复用 `isCoastalInfrastructure(_:)`，只能复用 kind 集合，不得把 AI 评分逻辑耦合进任务判定。

## 验收标准

- 任务链新增 `Secure Coast`，阶段顺序为 `Secure Oil` -> `Capture Frontline` -> `Secure Coast` -> `Deploy Combined Arms` -> `Break Red Production` -> `Destroy Red HQ`。
