# v4.39（航母警戒翼队接触类型提示）Agent B 实现提示词

## 1. 版本号

v4.39（航母警戒翼队接触类型提示）

## 2. 版本分配依据

- 当前 `md/prompt/v4（海军航母）/` 最新版本为 `v4.38（航母警戒翼队接触数提示）.md`，本轮延续 v4 海军航母阶段。

- 当前玩家仍无法从 `Ctc n` 直接判断接触主要来自空中、水面、潜艇或地面 / 结构威胁。本轮只补一个只读类型摘要，不改变战斗、AI、迷雾或命令。

## 3. 背景

当前 `GameScene.swift` 中：

- `carrierAirWingLine(for:)` 在 Carrier 未 HOLD 时显示 `Wing x/2 OK` 或 `Wing x/2 Need n`，在 Carrier HOLD 时显示 `Guard wing x/2 ...  Ctc n`。
- `boundCarrierGuardWing(for:)` 返回当前 Carrier 附近仍 HOLD 且 `guardAnchorCarrierID == carrier.id` 的 HEL/JET。
- `carrierGuardContactCount(for:guardWing:)` 遍历绑定 wing 和 `entities.values`，用 `isCarrierGuardContact(...)` 统计去重目标 id。
- `isCarrierGuardContact(...)` 同时被 HUD 接触数和 `carrierGuardPriorityTarget(for:)` 使用，已保证 contact 不绕过 `isKnownToFaction(...)`、`canAttack(...)`、Carrier 近域半径和 wing HOLD 站位半径。

## 4. 本轮目标

实现一个小范围、可验收的 HUD 反馈增量：

1. Carrier 未 HOLD 时继续显示原有 readiness：`Wing x/2 OK` 或 `Wing x/2 Need n`，不追加 contact 数或类型。
2. Carrier HOLD 时继续显示 `Ctc n`，并在 `n > 0` 时追加一个短类型摘要：
   - 只有 air 接触：`Ctc n Air`
   - 只有水面 naval 接触：`Ctc n Sea`
   - 只有 submarine 接触：`Ctc n Sub`
   - 只有 land 或 structure 接触：`Ctc n Ground`
   - 同时存在多个类型：`Ctc n Mix`
3. `Ctc 0` 不追加类型，避免无意义的 `Ctc 0 Mix`。
4. 类型摘要必须和 `Ctc n` 使用同一去重 contact 集合：同一个目标被多个 wing 看到或可攻击时只计一次，也只贡献一次类型。
5. 类型判断不能扩大情报来源，仍必须通过 `isCarrierGuardContact(...)` 过滤后才分类。
6. 本轮只读 HUD 反馈不得写入 `revealedUntil`、`visibleTiles`、`exploredTiles`、攻击目标、命令状态、AI 评分或生产状态。

## 7. 实现步骤

建议按以下小步实现：

1. 将 `carrierGuardContactCount(for:guardWing:) -> Int` 调整为返回 contact summary，例如 `(count: Int, type: String?)`，或新增 `carrierGuardContactSummary(for:guardWing:)` 并让旧 count helper 删除或只作为包装。
2. 遍历绑定 wing 与 `entities.values` 时继续用 `isCarrierGuardContact(...)` 过滤合法目标。
3. 用 `Set<Int>` 去重目标 id；只有首次插入某目标 id 时，才把该目标分类写入类型集合。
4. 增加小 helper，例如 `carrierGuardContactType(for target: GameEntity) -> String`：
   - `target.kind == .submarine` 返回 `Sub`
   - `target.kind.domain == .air` 返回 `Air`
   - `target.kind.domain == .naval` 返回 `Sea`
   - `target.kind.domain == .land` 或 `.structure` 返回 `Ground`
5. summary 生成规则：
   - `count == 0` 时 `type == nil`
   - 类型集合只有一个元素时返回该类型
   - 类型集合多于一个元素时返回 `Mix`
6. 修改 `carrierAirWingLine(for:)`：
   - 未 HOLD：保持原文案。
   - HOLD：返回 `Guard wing x/2 OK  Ctc n Type`、`Guard wing x/2 Need n  Ctc m Type`，或 `Ctc 0` 无类型。
7. 更新 `README.md`：说明 Carrier HOLD guard wing 行显示已知 contact 数和 Air / Sea / Sub / Ground / Mix 类型摘要。
8. 更新 `md/flow/flow.md`：同步 HUD 段落，说明类型摘要复用同一去重 contact 集合，遵守 `isKnownToFaction` / `canAttack` / HOLD 半径，不改变迷雾或命令。
9. 必要时更新 `md/flow/flowchart.md` 的 HUD 节点文字，不重画无关流程。
