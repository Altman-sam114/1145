# v4.29（支援按钮资金不足提示）

## 版本号

v4.29

## 版本分配依据

- `update_log.md` 当前最新正式玩法版本为 v4.28「陆军生产按钮来源提示」。
- v4.25 已让 `SCAN` / `REPR` / `AIRS` / `BARR` 在缺少 operational asset 时显示具体缺失资产短码。
- 当前支援按钮在未冷却且资产满足时只显示价格，即使玩家资金不足也需要点击后才知道不能执行。
- 本轮是 v4.25 同一 HUD 可读性路线的窄范围延伸：在支援资产满足但资金不足时显示短缺金额，不改变支援执行规则。

## 背景

当前 `DesertFrontline/GameScene.swift` 中：

- `SupportPower.cost` 定义 `SCAN` / `REPR` / `AIRS` / `BARR` 的费用。
- `supportButtonSubtitle(for:)` 当前显示优先级是：冷却秒数 -> 具体缺失资产短码 -> 价格。
- `supportIssue(for:faction:)` 是真实执行门槛，当前顺序是：冷却 -> 资金 -> operational asset。
- `selectSupportPower(_:)` 和 `executeSupportPower(_:at:faction:showFeedback:)` 都通过 `supportIssue(for:faction:)` 拒绝不可执行支援。
- v4.25 的缺资产提示只读复用 `hasOperationalSupportAsset(for:faction:)` 和 `supportAssetRequirementLabel(for:)`，没有改变费用、冷却、资产需求或效果。

本轮只让按钮 subtitle 在“无冷却、资产满足、资金不足”时提前显示资金缺口。

## 目标

1. 增强 `SCAN` / `REPR` / `AIRS` / `BARR` 按钮 subtitle。
   - 冷却中仍优先显示 `Ns`。
   - 未冷却但缺少 operational asset 时，仍优先显示 `need HQ/RAD`、`need HQ/MECH`、`need AF/CV` 或 `need BB/CV`。
   - 未冷却、资产满足、但玩家资金不足时，显示短缺金额，例如 `need $350`。
   - 未冷却、资产满足、资金足够时，仍显示 `$cost`。
2. 资金缺口必须只读读取玩家资金。
   - 推荐用 `max(0, power.cost - money(for: .player))`。
   - 不改变资金、不进入 pending、不触发冷却、不执行支援。
3. 保持真实执行链路完全不变。
   - 不改变 `supportIssue(for:faction:)` 的拒绝逻辑或消息。
   - 不改变 `selectSupportPower(_:)`。
   - 不改变 `executeSupportPower(...)`。
   - 不改变 `changeMoney`、cooldown、asset requirement、support radius、damage、repair、AI 支援使用或目标合法性。
4. 同步 README、flow 和必要的 flowchart 文案。

## 非目标

- 不改变任何支援技能的 cost、cooldown、radius、damage、repairAmount 或 asset requirement。
- 不改变支援 pending 面板、target panel、`Asset ready` / `Need` 资产行或执行反馈。
- 不改变 `supportIssue(for:faction:)` 的检查顺序，即使按钮文案选择让缺资产优先于资金短缺显示。
- 不改变 AI 支援评分、AI 支援执行、经济、任务、胜负、迷雾、潜艇暴露、伤害或维修结算。
- 不新增按钮禁用态、颜色、布局、字体、图标、动画或音效。
- 不更新 `md/test/test.md`，因为测试流程不变。
- Agent B 不更新 `update_log.md`；Agent C 在 artifact 验收通过后追加正式记录。
- 不新增 XCTest target、workflow、Xcode project 配置、外部素材或外部依赖。

2. 修改 `DesertFrontline/GameScene.swift`。

推荐实现方向：

```swift
private func supportButtonSubtitle(for power: SupportPower) -> String {
    let cooldown = supportCooldown(for: .player, power: power)
    if cooldown > 0 {
        return "\(Int(ceil(cooldown)))s"
    }
    if !hasOperationalSupportAsset(for: power, faction: .player) {
        return "need \(supportAssetRequirementLabel(for: power))"
    }
    let shortfall = power.cost - money(for: .player)
    if shortfall > 0 {
        return "need $\(shortfall)"
    }
    return "$\(power.cost)"
}
```

约束：

- 保持冷却优先和缺资产优先，避免本轮改变 v4.25 的缺资产可读性。
- 不在 subtitle helper 中调用 `supportIssue` 来避免改变按钮文案优先级。
