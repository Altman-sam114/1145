# v4.98 直升机双侧火箭齐射

## 版本号与依据

`v4.98`。当前正式版本为 `v4.97`；人工要求持续优先改善 UI 与战斗细节，本轮只补 Helicopter 对地 / 对水面目标的专用火箭反馈，因此顺序递增。

## 背景

- 参考对象固定为百度百科 lemma id `4042982` 对应的 Noble Master Games《沙漠风暴 Desert Stormfront》。已重新核对官方 press kit、trailer 与 `screen_dsf_v02_13.png`；官方画面中的绿色 Helicopter 从机身侧向地面战线发射分离弹药，弹道带短白烟点，目标端爆点紧凑。本项目只借鉴俯视轮廓、节奏和反馈原则，不复制素材或 UI。
- 当前 Helicopter 模型已有双侧 weapon pods，但实际攻击仍只使用单条通用直线 tracer，模型挂载与战斗反馈没有形成对应关系。

## 目标

1. Helicopter 攻击玩家可见 land、structure 或非 Submarine 水面舰艇时，从双侧 weapon pods 交替发射四枚约 0.045 秒错开的短火箭。
2. 每枚火箭有分离的轻微二次曲线、可见弹体 / 尾焰、阵营色亮芯和离散白色烟点；两个 pod 发射点各有紧凑闪光。
3. land / structure 目标使用紧凑多点闪光、尘环与碎屑，水面目标使用小型舰体火花和近舷水面反馈；不得遮满目标、HP 或命令 UI。
4. 每次 `fire` 仍只扣一次 HP；四枚火箭全部是视觉，不新增 splash、碰撞或第二次伤害。
5. Submarine 保持既有单 tracer 与 ASW hit / 暴露链路，不进入火箭齐射。
6. 新增 capture-only `helicopter-salvo` 场景：单选 Blue Helicopter 锁定约 45% HP Red Tank，冻结四枚不同进度火箭、双 pod 闪光、烟点、多点命中和伤害飘字。
7. GitHub Actions 扩展为十八次独立启动，新增 `simulator-helicopter-salvo.png`。

## 非目标

- 不改 Helicopter damage、range、cooldown、cost、build time、HP、speed、vision、生产来源、AI 或目标合法性。
- 不新增 splash、弹药库存、真实 projectile entity、压制、摇摆或炮塔状态。
- 不改 Fighter、Carrier、AA Truck、迷雾、声呐、经济、任务或胜负。
- 不运行本地 Xcode build、本地 simulator 或本地玩法探针。

## 架构与实现步骤

1. 保留人工 `project.pbxproj` Team ID 改动，不纳入提交。
2. 在统一 `fire(attacker:target:)` 只对玩家 Helicopter 或玩家已知 Red Helicopter 且目标不是 Submarine 时派发 `showHelicopterRocketSalvo(...)`；Submarine 和其他单位保留旧分支。
3. `showHelicopterRocketSalvo(...)` 复用 `quadraticPoint(...)` / `quadraticTangent(...)`，以四条轻微错位路径组合弹体、尾焰、烟点和两个 pod 发射闪光。普通节点约 0.8 秒内清理，`persistent` 仅用于 capture。
4. 新增 `showHelicopterRocketImpact(...)`；水面目标复用较小舰体 / 水面反馈。所有 helper 不调用 `fire`、不写 HP、不创建 gameplay entity。
5. 在 camera focus 早退前分流 `helicopter-salvo`，编排 Blue Helicopter、Red Tank、选择、目标、HP、reload、fog、投影和 persistent 视觉。
6. workflow 的清理、launch 与 manifest 加入第十八张截图。
7. 同步 README、flow、flowchart 与 test；Agent C 云端 artifact 验收通过后再更新 `update_log.md`。
