# v4.60 / AI 主攻波 HUD 摘要过期

## 版本号

v4.60

## 版本分配依据

上一轮 v4.59 已让 HUD Red 状态行显示最近一次玩家可感知 Red routine assault wave 子集摘要。当前摘要一旦出现，会一直保留到下一次可感知 wave 或 `SKRM` 重开，容易让玩家误以为旧主攻波仍是当前状态。本轮给该摘要增加过期时间，过期后恢复显示 AI 难度，属于 v4 AI / HUD 可读性的小步修正。

## 背景

当前相关事实：

- `lastEnemyAssaultWaveSummary` 存储最近可感知主攻波摘要。
- `updateEnemyAssaultWaveSummary(for:)` 只在 wave 中存在玩家已知单位时更新摘要。
- `updateHUD()` 用 `lastEnemyAssaultWaveSummary ?? "AI \(aiDifficulty.displayName)"` 显示 Red 状态。
- `restartSkirmish()` 会清空 `lastEnemyAssaultWaveSummary`。
- 当前没有摘要时间戳或过期逻辑。

## 目标

- 给 Red 主攻波 HUD 摘要增加固定持续时间。
- 摘要过期后 HUD 自动恢复显示 AI 难度。
- 新的可感知主攻波会刷新摘要内容和时间戳。
- `SKRM` 重开同时清空摘要和时间戳。

## 非目标

- 不改变 AI 选波、目标、escort 门槛、Carrier guard wing、低血撤退、生产、支援、迷雾、潜艇侦测、任务、胜负或 SKRM 地图生成。
- 不改变 v4.59 的玩家可知边界；仍只统计玩家已知的 wave 子集。
- 不新增顶部消息、音效、按钮、独立日志面板或完整战报系统。
- 不新增 CAP、巡逻、截击或 Carrier battle group 策略扩展。

## 当前架构依据

- `lastUpdateTime` 已作为场景时间源，用于支援 reveal、警报 cooldown、潜艇临时侦测等 runtime 状态。
- `updateHUD()` 每帧刷新，是判断摘要是否仍有效的最小入口。
- `updateEnemyAssaultWaveSummary(for:)` 是写入摘要的唯一入口，适合同时写入时间戳。
- `restartSkirmish()` 是清理本局 runtime 状态的集中入口。

## 实现步骤

1. 在 `GameScene.swift` 增加：
   - `private var lastEnemyAssaultWaveSummaryTime: TimeInterval = -100`
   - `private let enemyAssaultWaveSummaryDuration: TimeInterval = 12`
2. 修改 `updateEnemyAssaultWaveSummary(for:)`：
   - 写入 `lastEnemyAssaultWaveSummary` 后，同步写入 `lastEnemyAssaultWaveSummaryTime = lastUpdateTime`。
3. 增加 helper `currentEnemyAssaultWaveSummary()`：
   - 若 `lastEnemyAssaultWaveSummary` 为空，返回 nil。
   - 若 `lastUpdateTime - lastEnemyAssaultWaveSummaryTime <= enemyAssaultWaveSummaryDuration`，返回摘要。
   - 否则返回 nil。
4. 修改 `updateHUD()`：
   - 使用 `currentEnemyAssaultWaveSummary() ?? "AI \(aiDifficulty.displayName)"`。
5. 修改 `restartSkirmish()`：
   - 清空 `lastEnemyAssaultWaveSummary`。
   - 重置 `lastEnemyAssaultWaveSummaryTime = -100`。
6. 同步 README：
   - Enemy AI / HUD 描述改为“latest recently player-detectable”或类似，避免暗示永久显示。
7. 同步 `md/flow/flow.md`：
   - AI / HUD 段写清摘要持续约 12 秒，过期恢复 AI 难度。
8. `md/flow/flowchart.md` 如未描述 HUD 状态细节，可不改。
