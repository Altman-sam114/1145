# v4.80 / 三类战场命令落点反馈

## 版本号与依据

v4.80。v4.79 已由最终 `origin/main` run `29144283671` artifact 验收通过，五页单排 HUD 已释放战场空间；下一优先级是让玩家点击战场后立即分辨普通移动、攻击移动和直接攻击是否成功下达。

## 视觉参考

- Noble Master 官方产品页与 5 张 1280x800 截图：`https://noblemaster.itch.io/desert-stormfront`。
- 官方页面嵌入 trailer：`https://www.youtube.com/watch?v=7oZC6DpIUhQ`。
- 官方截图使用高对比黄橙命令色、单位周围短箭头 / 虚线圈和紧贴战场主体的低占屏反馈，适合休闲移动端 RTS 快速确认操作。
- 本轮只借鉴“瞬时、高对比、贴地、低遮挡”的原则，不复制原素材、图形或 UI 排列。

## 目标

- 普通移动显示青绿色 `MOVE` 落点：贴地椭圆、方向箭头和短刻度，快速淡出。
- 攻击移动显示琥珀色 `AMOV` 落点：更强的双层环、十字 / 推进箭头和标签，与普通移动明显区分。
- 玩家成功直接攻击已知敌方目标时显示红色 `ATK <shortCode>` 目标框：尺寸随目标 footprint 调整，并只在攻击者真实分配成功后出现。
- 三类反馈都位于 `effectsLayer`，短生命周期、无逐帧创建；不修改目的地、攻击目标、编队、伤害、射程或 AI。
- 直接攻击标记只能来自 `entity(at:)` 已通过玩家迷雾认知过滤的目标，不得泄露隐藏敌军或潜艇。
- 云端新增 move / attack-move / attack-target 三个确定性截图与独立 PID 存活检查。

## 非目标

- 不修改触摸手势、选择、编队偏移、寻路、attack-move 搜敌、直接攻击合法性或 Carrier guard release 语义。
- 不给 AI 显示玩家视角命令标记；AI 仍共享原移动与战斗执行函数。
- 不修改 HUD 五页映射、模型、地图、经济、生产、支援、任务、胜负或迷雾集合。
- 不新增外部依赖、素材、音效或 XCTest target。
- 不运行本地测试、build、模拟器、探针或本地静态检查。
- 不提交用户 `project.pbxproj` 团队号改动。

## 当前架构依据

- `handleWorldTap(...)` 在成功直接攻击时设置 `attackTarget`；目标来自 `entity(at:)`，该函数会排除玩家未知敌方实体。
- `issueFormationMove(...)` 已按 land / air / naval 计算真实 anchor，并分别调用 `showMoveMarker(...)` 或 `showAttackMoveMarker(...)`。
- `effectsLayer` 负责短暂战场视觉，现有 marker 使用 `SKShapeNode` 和 `SKAction` 后自动移除。
- `prepareCIAirCaptureScene()` 已构建确定性空战场景；`DESERT_CI_CAPTURE_MODE=1` 会冻结玩法循环，适合持久化 command marker。

## 实现步骤

1. 把 move 与 attack-move marker 改为组合节点，加入贴地环、方向 / 十字符号、标签和统一短动画；支持仅 CI 使用的 `persistent` 参数。
2. 新增 footprint-aware direct attack target marker，并在 `assignedAttackers > 0` 后调用，标签使用目标 `shortCode`。
3. `prepareCIAirCaptureScene()` 读取 `DESERT_CI_COMMAND_MARKER=move/attack-move/attack-target`，在可见战场上持久显示对应反馈；普通启动不读取或改变任何命令状态。
4. workflow 保留五页 HUD 探针，并额外以 tactical 页启动三次 command marker capture；每次解析 PID、等待、截图并用宿主机 `kill -0` 检查存活。
5. 把三张新图写入 artifact manifest，并同步 README、flow、flowchart 和测试规范。
6. commit `v4.80: 强化战场命令落点反馈`，直推 `origin/main`，只做 GitHub Actions 云端验证。

## 验收标准

- 最新 main 的 manifest SHA、run、attempt、version 一致；static checks、project lint、generic build 和 simulator launch 均成功，JUnit 0 失败。
- 原五张 HUD 截图继续正常，无黑屏 / 白屏、页签丢失或裁切。
- `simulator-command-move.png` 清楚显示青绿色 `MOVE` 贴地落点。
- `simulator-command-attack-move.png` 清楚显示琥珀色 `AMOV` 落点，视觉上不同于 MOVE。
- `simulator-command-attack-target.png` 在已知 Red 空军目标上显示红色 `ATK <shortCode>` footprint-aware 框，不遮住目标主体。
- 八次启动 PID 均在截图后存活；App 日志无启动崩溃、数组越界或未捕获异常。
- 源码审阅确认普通三类 marker 会自动移除，CI persistent marker 不进入普通启动，直接攻击 marker 不绕过 `entity(at:)` 迷雾过滤。
