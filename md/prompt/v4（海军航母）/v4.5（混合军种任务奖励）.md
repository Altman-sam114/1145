# v4.5（混合军种任务奖励）

## 版本号

v4.5

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.4「海岸据点任务奖励」。
- v4 阶段聚焦海军、航母、海岸争夺和任务链推进。v4.3 新增 `Secure Coast` 阶段，v4.4 为该阶段增加一次性资源奖励并引入 `missionReward(for:)`。
- 本轮继续沿任务链奖励推进，但只扩展一个主攻前目标：完成 `Deploy Combined Arms` 时给玩家一次性资源奖励，不新增支援冷却、免费技能、科技树、升级系统或新 HUD 按钮。

## 背景

当前 `Deploy Combined Arms` 要求玩家拥有 10 个机动单位，并且 land / air / naval 三个域都至少有一个单位。该目标发生在 `Secure Coast` 之后、`Break Red Production` 之前，是玩家从经济/海岸建设进入主动进攻的关键门槛。

v4.4 已经证明任务奖励可以安全复用 `completedMissionStages` 去重和 `changeMoney(for:by:)` 经济入口。本轮只把奖励表扩展到 `Deploy Combined Arms`，让玩家完成混编投入后获得主攻前补给，同时不扩大到完整战役奖励系统。

## 目标

1. 为 `Deploy Combined Arms` 增加一次性玩家资源奖励。
   - 奖励金额固定为 `$800`。
   - 只在 `.combinedArms` 首次完成并写入 `completedMissionStages` 时发放。
   - 使用现有 `changeMoney(for: .player, by:)`。
2. 扩展 `missionReward(for:)`。
   - `.secureCoast` 保持 `600`。
   - `.combinedArms` 返回 `800`。
   - 其他阶段仍返回 `0`。
   - 不新增新的持久状态；仍依赖 `completedMissionStages` 防止重复发奖。
3. 更新任务 HUD 文案。
   - `missionDetail(for: .combinedArms)` 简短显示 `+$800`。
   - 示例：`Field 10 units: X/10  Lx Ay Nz  +$800`。
   - 文案必须保持短句，避免挤压 HUD。
4. 保持任务完成消息逻辑。
   - v4.4 已经让 `updateMissionProgress()` 以最后完成阶段为标题并附本轮累计奖励；本轮不需要新增消息系统。
   - 同一帧连续完成 `Secure Coast` 和 `Deploy Combined Arms` 时，奖励仍在循环内按对应阶段发放，消息显示最后完成阶段标题和本轮累计奖励。
5. 文档同步。
   - `README.md`：任务面板描述中说明 Secure Coast 和 Deploy Combined Arms 有一次性资源奖励。
   - `md/flow/flow.md`：任务系统说明中同步 `missionReward(for:)` 当前奖励表。
   - `md/flow/flowchart.md`：可简短同步任务节点含海岸/混编奖励。
6. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不改变任何单位/建筑的 cost、HP、damage、range、cooldown、vision、sonar、buildTime、canAttack、footprint 或建造规则。
- 不改变生产队列、AI build pattern、AI 海岸目标权重、动态 AA / ASW、support powers、潜艇侦测、迷雾合法性、路径、damage / XP 结算或胜负条件。

关键源码事实：

- `MissionStage` 当前顺序为 `secureOil`、`captureFrontline`、`secureCoast`、`combinedArms`、`breakRedProduction`、`destroyHQ`。
- `missionDetail(for:)` 负责当前任务 HUD 详情。
- `playerMobileDomainCounts()` 统计玩家存活机动单位总数和 land / air / naval 分布。
- `isMissionStageComplete(.combinedArms)` 当前要求 `total >= 10 && land > 0 && air > 0 && naval > 0`。
- `updateMissionProgress()` 会 while 连续完成已满足条件的阶段，完成时通过 `missionReward(for:)` 发钱，并显示最后完成阶段标题加本轮累计奖励。
- `completedMissionStages.insert(stage)` 是阶段首次完成边界；该集合在 `restartSkirmish()` 中清空。
- `changeMoney(for:by:)` 是玩家 / AI 资金变化的共享入口。

## 风险和禁止项

- 风险：如果在 `missionDetail` 或每帧 HUD 刷新里发钱，会造成重复奖励；奖励必须只发生在 `updateMissionProgress()` 的阶段首次完成路径。
- 风险：如果改 `playerMobileDomainCounts()` 或完成条件，会改变任务目标范围；本轮禁止。
- 风险：奖励过高可能放大 10 单位混编后的滚雪球；本轮固定 `$800`，低于大型舰艇价格，作为主攻前补给。
- 风险：任务文案过长会挤压 HUD；保持短句。
- 禁止使用非 `Altman-sam114` 的 GitHub 账号伪装 push、CI 或 artifact 验收。
- 禁止下载大体积历史 artifact、模型、测试数据或无关产物。
