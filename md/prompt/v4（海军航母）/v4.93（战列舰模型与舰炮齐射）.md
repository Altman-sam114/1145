# v4.93 战列舰模型与舰炮齐射

## 版本号

`v4.93`

## 版本分配依据

- `update_log.md` 与 `md/prompt/v4（海军航母）/` 的最新正式版本为 `v4.92`。
- 人工总目标要求海战、模型与战斗画面持续细化；当前 Battleship 仍只有简单船体和两个矩形炮塔，舰炮攻击主要是一条直线炮迹和单个水柱。本轮围绕同一海战资产同时完善模型与开火反馈，因此递增为 `v4.93`。

## 背景

- 参考对象继续固定为百度百科 lemma id `4042982` 对应的 Noble Master Games《沙漠风暴 Desert Stormfront》，不是其他同名作品。
- 已核对 Noble Master 官方 press kit `https://www.noblemaster.com/press/sheet.php?p=desert_stormfront`、官方 trailer `https://www.youtube.com/watch?v=7oZC6DpIUhQ` 与官方海战图 `screen_dsf_v02_10.png`。官方图以清晰舰种轮廓、紧贴单位的生命信息和直接弹道表现大规模海战；本项目只借鉴可读性原则，不复制原素材或 UI。
- 当前 `addNavalUnitBody(...)` 的 Battleship 只有 74x24 hull 与两个无炮管矩形；Carrier、陆军、空军模型已更细，视觉完成度不一致。
- `fire(attacker:target:)` 已统一处理伤害、冷却、目标合法性、已知边界、伤害飘字和水面命中；`showProjectile(...)` 对 Battleship / Coastal Battery 只画单条直线，`showNavalWaterImpact(...)` 已有一组水柱 / 环 / 水滴，可复用而不另建战斗系统。

## 目标

1. 在原 Battleship gameplay footprint 与实体树内增加分层主甲板、首尾装甲、两座可辨双联装主炮、炮塔座圈 / 炮盾 / 双炮管、舰桥、阵营化舷窗、桅杆 / 雷达、二级火炮和救生艇；不改变碰撞、selection、health bar 或数值。
2. 玩家 Battleship / operational Coastal Battery 开火，或敌方炮位对玩家已知时，用 `showNavalGunSalvo(...)` 显示双发舰炮：两条分离的暖白核心 / 阵营强调炮迹、两枚亮弹体、炮口闪光和烟团。每次 attack cycle 仍只结算一次原伤害。
3. Battleship / Coastal Battery 命中玩家可见水面舰艇时，用 `showNavalSalvoImpact(...)` 组合主水柱、较小副水柱和舰体装甲命中闪光 / 火花；Artillery 对舰命中继续使用原单水柱。
4. 所有新普通节点在约 0.6 秒内清理；capture-only `persistent` 可冻结证据。未知敌方炮位不得通过新双发炮迹或炮口反馈泄露。
5. 新增 capture-only `naval-salvo` coast 场景：Blue Battleship 对约 47% HP Red Battleship 设置真实目标与 reload，冻结双发炮迹、炮口闪光、舰体命中、主 / 副水柱、双方航迹、单选目标 HP / 距离 / reload、射程圈和命令条。
6. GitHub Actions 从十二次扩展为十三次独立启动，新增 `simulator-naval-salvo.png`。

## 非目标

- 不改变 damage、armor multiplier、range、cooldown、target acquisition、AI、formation、pathfinding、economy、production、mission、victory 或 fog。
- 不让一次视觉双发造成两次伤害，不增加 splash damage、击退、燃烧、dot 或弹药系统。
- 不重做 Carrier / Submarine / Shipyard / Coastal Battery 模型，不修改海军建造和护航逻辑。
- 不新增外部依赖、图片素材或音频。
- 不运行本地 Xcode build、本地 simulator 或本地游戏探针。

## 当前架构依据

- `addNavalUnitBody(for:to:)` 是 Battleship 静态视觉的唯一入口；模型节点随实体移动、镜像、迷雾隐藏和销毁。
- `fire(attacker:target:)` 是玩家与 AI 共用的直接火力结算入口；只在这里选择新视觉 helper，不新增平行伤害路径。
- `attacker.faction == .player || isKnownToFaction(attacker, observer: .player)` 是新舰炮来源反馈的可见性门槛；目标命中继续使用 `shouldShowNavalWaterImpact(...)` 的玩家 / 已知过滤。
- `showNavalWaterImpact(...)` 已支持 `scale` 和 `persistent`，可由新的 salvo impact helper 复用两次；舰体闪光应使用独立短生命周期节点。
- `prepareCICaptureScene()` 的 coast/default 分支已布置玩家海军航迹和水面命中，可在 `DESERT_CI_COMMAND_MARKER=naval-salvo` 时改为专门编排，普通启动与 map-terrain capture 不进入该分支。

## 实现步骤

1. 开始前确认 `main`、`HEAD == origin/main`，保留人工未暂存的 `DesertFrontline.xcodeproj/project.pbxproj` `DEVELOPMENT_TEAM` 改动。
2. 在 Battleship case 内用现有 `shipHullPath(...)`、`UIColor.darker(...)` 和 SpriteKit shape nodes 构建分层船体 / 甲板 / 舰桥 / 桅杆 / 雷达 / 救生艇 / 二级炮；为两座主炮各建座圈、炮盾和两根平行炮管，保持 Blue / Red 阵营化窗口与原 xScale 镜像。
3. 新增 `showNavalGunSalvo(from:to:kind:faction:persistent:)`。只允许 Battleship / Coastal Battery；根据射向计算 direction / normal，为双发炮迹创建外层强调线、暖白核心、弹体以及双炮口 flash / smoke，普通状态定时清理。
4. `fire(...)` 对 Battleship / Coastal Battery 且炮位玩家可见时走新 salvo；其他武器继续走原 `showProjectile(...)`。隐藏 Red 炮位不创建新 helper。
5. 新增 `showNavalHullStrike(...)` 与 `showNavalSalvoImpact(...)`。后者确定性选择目标两侧的主 / 副落点，复用 `showNavalWaterImpact(...)`，并在船体生成短促暖白 / 橙色装甲闪光、火花与烟团。
6. `fire(...)` 的水面舰艇命中分支：Battleship / Coastal Battery 使用 salvo impact，Artillery 保持原单水柱；一次 attack cycle 的 `target.hp` 写入仍只有一次。
7. coast capture 在 `naval-salvo` marker 下把玩家 Battleship、Red Battleship 和背景 Carrier 放到 coast 镜头内，Red Battleship 设为约 47% HP，Blue Battleship 设真实目标 / reload 并选中，更新 fog 后冻结齐射与命中证据。
8. workflow 清理列表、启动列表和 manifest 增加 `simulator-naval-salvo.png`，总计十三次 launch / PID 存活检查。
9. 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 与 `md/test/test.md`，只记录已完成行为。

## 风险和禁止项

- 禁止把双发视觉实现为两次 `fire` / 两次 HP 扣减。
- 禁止为视觉新增 per-frame 节点或常驻全局数组；只在真实 fire / capture 时创建短生命周期效果。
- 禁止舰炮弧线夸张成导弹转弯；双线应紧凑、平行、方向明确。
- 禁止水柱、闪光或 smoke 大到完全遮挡目标舰、HP bar、任务面板或 HUD。
- 禁止未知 Red Battleship / Coastal Battery 通过新炮口或双发线泄露坐标。
- 禁止用旧 artifact、本地截图或文字汇报代替最新云端验收。
