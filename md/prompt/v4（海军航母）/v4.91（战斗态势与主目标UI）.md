# v4.91 战斗态势与主目标 UI

## 版本号

`v4.91`

## 版本分配依据

- `update_log.md` 与 `md/prompt/v4（海军航母）/` 的最新正式版本为 `v4.90`。
- 人工明确要求优先提升 UI 和战斗细节。本轮在现有五页单排 HUD、选择信息面板、命令意图线和 focus marker 上做一个完整但有限的交战信息增量，因此递增为 `v4.91`。

## 背景

- 参考对象继续固定为百度百科 lemma id `4042982` 对应的 Noble Master Games《沙漠风暴 Desert Stormfront》，不是其他同名游戏。
- Noble Master 官方 press kit、官方 trailer 和 iOS 截图体现的触屏信息原则是：单位 / 编队状态集中、动作按钮足够大、战场主体尽量不被长期面板遮挡。项目只借鉴信息层级，不复制原作素材、面板外形或排版。
- 当前右侧选择面板已经显示编队领域、总 HP、总伤害、最大射程和部分状态；世界 focus marker 只显示 `FOCUS n`。玩家在密集战斗中仍需从多个位置推断正在交战的单位数、主目标类型、目标剩余 HP、近似距离和武器是否可再次开火。
- 现有 `attackTarget`、`attackTimer`、`selectedIDs`、`sharedFocusFireTarget(...)`、`isKnownToFaction(...)`、`updateHUD()` 和 `focusFireMarkerNode` 已提供可靠状态来源，不需要新增平行战斗状态。

## 目标

1. 选中一个正在攻击的作战单位时，右侧既有选择面板优先显示目标短码、目标 HP / 百分比、近似距离，以及当前武器 ready / reload 状态。
2. 多选编队时集中显示作战单位数、正在交战数、可开火数、受损 / 危急单位数；若存在主目标，显示主目标短码、绑定攻击者数量、目标 HP / 百分比和最近距离。
3. 在选择信息面板底部增加一条紧凑主目标生命条；没有玩家已知合法主目标时隐藏，不占用长期战场空间。
4. 增强既有共享集火世界标记：继续只在至少两个选中单位共享合法已知目标时显示，但标签升级为 `FOCUS n <code> <hp>%`，并在目标下方显示分段生命条。
5. 将云端 simulator launch probe 从十次扩展为十一次，新增 `simulator-combat-ui.png`。使用 tactical 页、land focus 和 capture-only `combat-ui` marker，冻结四个玩家作战单位共同攻击一个受损敌方 Tank 的状态，同时保留 Mechanic、维修链路、炮线、爆炸、单排命令条和右侧选择面板。

## 非目标

- 不重构五页 HUD，不新增常驻大面板，不移动或缩小现有主命令按钮。
- 不改变伤害、射程、冷却、装甲倍率、目标选择、AI、迷雾、移动、经济、生产、任务或胜负。
- 不修改 AA Truck、其他单位或建筑模型；AA Truck 细化顺延。
- 不允许 UI 读取未知敌方实体或 stale / 已死亡目标，不通过目标条泄露战争迷雾。
- 不新增外部依赖或图片资源。
- 不运行本地 Xcode build、本地 simulator 或本地游戏探针。

## 当前架构依据

- `selectionInfoContent(...)` 分流 pending、空选中、单选和多选；`singleSelectionInfo(...)`、`groupSelectionInfo(...)` 是本轮战斗摘要的唯一文字入口。
- `sharedFocusFireTarget(...)` 已过滤玩家阵营、存活、非结构选中单位、存活攻击目标和 `isKnownToFaction(..., observer: .player)`，并要求至少 2 个共享攻击者。
- 可新增一个同口径但允许单个攻击者的 `primaryCombatTarget(...)`，供单选 / 多选面板和主目标生命条复用；`sharedFocusFireTarget(...)` 应复用它或维持相同过滤边界。
- `focusFireMarkerNode` 是 effects layer 中的只读选择反馈；不得写回 `attackTarget` 或影响战斗结算。
- `addSelectionInfoPanel(...)` 在既有右侧面板中创建标签，可在面板底部预创建目标生命条 back / fill，并在 `updateHUD()` 时只更新隐藏、比例、位置和颜色。
- `prepareCILandCaptureScene()` 已冻结双方地面模型和维修 / 炮击证据；capture-only `DESERT_CI_COMMAND_MARKER=combat-ui` 可为选中玩家作战单位设置共享目标和受损目标 HP，普通启动不走该分支。

## 实现步骤

1. Agent B 开始前执行 `git fetch origin`、确认 `main` 和 `origin/main`，并 `git pull --ff-only origin main`；保留人工未暂存的 `DesertFrontline.xcodeproj/project.pbxproj` `DEVELOPMENT_TEAM` 改动。
2. 新增 `primaryCombatTarget(for:)`，只统计玩家当前选中、存活、非结构、真实 `attackTarget` 存活且玩家已知的单位；按攻击者数量优先、目标 id 确定性打平。
3. 为单选战斗状态生成 `Tgt <code> HP x/y <p>% Dn` 和 `Weapon ready` / `Reload x.xs`，只在合法主目标存在时替换低优先级 Move / Value 行。
4. 为多选生成紧凑四行：领域 / Combat 数、总 HP + Wounded / Critical、主目标摘要或总伤害 / 最大射程、Engaged / Ready / 最近距离或现有最高优先状态。最长字符串必须适合 compact 选择面板。
5. 在 selection info panel 底部预创建窄生命条；`updateSelectionTargetHealthBar(...)` 依据主目标 HP 比例左对齐缩放 fill，按 >55%、>25%、其余使用绿 / 黄 / 红，目标无效时隐藏。
6. 增强 `refreshFocusFireMarker(...)` 的标签和分段目标生命条；保持 marker 只读、玩家已知过滤、至少两名共享攻击者和现有世界 zPosition。
7. land capture 在 `command_marker=combat-ui` 时把四个可攻击 Tank 的玩家单位绑定到同一敌方 Tank，把目标冻结在约 43% HP，并清理这些攻击者的 move / attack-move path；Mechanic 保持维修职责。
8. workflow 新增第十一次 `launch_hud_page tactical ci-results/simulator-combat-ui.png 4 combat-ui land`，同步清理列表和 manifest。
9. 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `md/test/test.md`，只记录已完成行为。

## 风险和禁止项

- compact 面板只有 4 行，文案必须短；禁止为了塞信息缩小到不可读字号。
- 主目标生命条必须左对齐缩放，不能随比例从中心收缩导致误读。
- 战斗 UI 不得把多个目标误称为唯一目标；显示的是攻击者最多的 `PRIMARY / FOCUS` 目标，并附攻击者数量。
- 禁止绕过 `isKnownToFaction(...)`、读取隐藏目标或保留死亡目标。
- 禁止扩大为 HUD 全面重构、战斗规则改写或单位模型迭代。
- 禁止用旧 artifact、本地截图或文字汇报代替最新云端验收。
