## 背景

当前航母 HOLD 警戒翼队已经具备：

- 选中 Carrier 并下达 `HOLD` 时，附近同阵营、存活、operational、非结构的 Helicopter / Fighter 会绑定该 Carrier 的 `guardAnchorCarrierID`。
- Carrier 仍 HOLD 时，绑定翼队通过 `boundCarrierGuardWing(for:)` 计入紧凑 `GW ... Cn` HUD 行，并通过站位逻辑维护 Carrier 附近稳定 guard station。
- `carrierGuardPriorityTarget(for:)` 在 `updateCombat(dt:)` 的目标获取阶段给绑定 HEL/JET 优先选择 Carrier 近域内的合法威胁。
- `carrierGuardContactSummary(for:guardWing:)` 已复用 `isCarrierGuardContact(...)` 统计去重接触数 `C`、接触类型 Air / Sea / Sub / Ground / Mix，以及按同一优先级排序得到的 `Tgt XXX` 短码。
- `isCarrierGuardContact(...)` 是 guard contact 合法性的统一边界，要求目标存活、敌对、非中立、可被 wing 攻击、在 Carrier 威胁半径内、在 wing HOLD 警戒半径内，并满足 `isKnownToFaction(...)`。

本轮只补充一个 HUD 可读性信息：在单选 HOLD Carrier 的紧凑 `GW ... Cn` 行中追加当前绑定警戒翼队已经实际交战的 HEL/JET 数量，例如 `Eng 1`。

## 目标

只为单选 HOLD Carrier 的紧凑 `GW ... Cn` 行追加短只读交战数：

- 统计对象：当前 Carrier 的 `boundCarrierGuardWing(for:)` 结果，即仍绑定当前 Carrier guard anchor 的 HEL/JET。
- 统计条件：翼队成员当前 `attackTarget` 非空，且该 `attackTarget` 仍满足 `isCarrierGuardContact(target, for: wing, carrier: carrier, holdPosition: holdPosition)`。
- 显示建议：在现有 `GW x/2 OK Cn ... Tgt XXX` 或 `GW x/2 Need n Cn ... Tgt XXX` 行末追加 `Eng n`，例如：
  - `GW 2/2 OK C1 Air Tgt JET Eng 1`
  - `GW 1/2 Need 1 C2 Mix Tgt SUB Eng 1`
  - `GW 2/2 OK C0 Eng 0`
- `Eng 0` 可以显示，也可以由 Agent B 根据 HUD 宽度选择省略；如果省略，必须只在 `n == 0` 时省略，非零必须显示。
- 该信息必须是只读 HUD 摘要，不改变任何命令、目标、AI、迷雾或战斗结果。

## 非目标

本轮不得实现或改动以下内容：

- 不做 CAP、巡逻、截击、新按钮、新 HUD action、新状态机、新 AI 策略或新战斗加成。
- 不改变 `isCarrierGuardContact(...)` 的 contact predicate。
- 不改变 `carrierGuardPriorityTarget(for:)` 的目标排序、返回逻辑或 tie-breaker。
- 不改变 `updateCombat(dt:)` 中 `attackTarget` 的写入、清理、冷却、开火、HOLD、AMOV 或普通目标搜索语义。
- 不改变 `fire(attacker:target:)`、伤害、命中、XP、老兵、ASW HIT、潜艇暴露或声呐侦测。
- 不改变战争迷雾、`isKnownToFaction(...)`、`visibleTiles`、`exploredTiles`、`supportRevealTiles`、潜艇 `revealedUntil`。
- 不改变 AI、移动、生产、支援技能、集结点、护航计数、SKRM 重置或任何实体状态字段。
- 不新增外部依赖、素材、测试 target 或大体积 artifact。

## 当前架构依据

关键源码边界在 `DesertFrontline/GameScene.swift`：

- `singleSelectionInfo(for:)`：单选 Carrier 时把 `carrierDeckCapabilityLine()`、`highValueNavalEscortLine(for:)`、`carrierDeckQueueAndRallyLine(for:)` 和 `carrierAirWingLine(for:)` 拼到选择面板行中。
- `carrierAirWingLine(for:)`：当前生成 `Wing x/2 ...` 或 HOLD 时的 `GW x/2 ... Cn Air/Sea/Sub/Ground/Mix Tgt XXX` 紧凑行。本轮优先在这里拼接 `Eng n`。
- `boundCarrierGuardWing(for:)`：HOLD Carrier 的 guard wing 统计口径，只包括附近 air wing 中 `holdPosition != nil` 且 `guardAnchorCarrierID == carrier.id` 的成员。
- `carrierGuardContactSummary(for:guardWing:)`：现有去重 contact / type / target 摘要，复用 `isCarrierGuardContact(...)`，不能改变其现有返回含义，除非以兼容方式新增只读 engaged count 字段或另写 helper。
- `isCarrierGuardContact(_:for:carrier:holdPosition:)`：guard contact 合法性边界，必须原样复用，不得放宽、收紧或绕过。
- `carrierGuardPriorityTarget(for:)`：只返回候选目标，不直接写入移动、迷雾或攻击状态；本轮不得改动其排序和行为。
- `updateCombat(dt:)`：当前只有在 `entity.attackTarget == nil` 时用 `carrierGuardPriorityTarget(for:) ?? nearestTarget(...)` 写入目标。本轮不得为了 HUD 统计新增写入路径。

推荐实现边界：

- 新增一个私有只读 helper，例如 `carrierGuardEngagedCount(for:guardWing:) -> Int`，或在 `carrierGuardContactSummary` 的返回 tuple 中兼容性追加 `engagedCount: Int`。
- helper 内对 `guardWing` 遍历：
  - `guard let holdPosition = wing.holdPosition else { continue }`
  - `guard let target = wing.attackTarget else { continue }`
  - 如果 `isCarrierGuardContact(target, for: wing, carrier: carrier, holdPosition: holdPosition)` 为 true，则计数 +1。
- 统计的是 HEL/JET 数量，不是去重目标数量；两个 wing 同时攻击同一合法目标时应显示 `Eng 2`。
- 对过期、死亡、不可见、离开 Carrier 近域、离开 wing HOLD 警戒半径、当前 wing 不可攻击的 `attackTarget` 不计入。

## 风险和禁止项

风险：

- HUD 行过长可能挤压选择面板文本；Agent B 可选择省略 `Eng 0`，但非零 `Eng n` 必须显示。
- 如果误把 contact 目标去重数当作 engaged count，会与目标不符；本轮统计的是正在交战的 HEL/JET 数量。
- 如果绕过 `isCarrierGuardContact(...)` 直接检查距离或可见性，可能造成 HUD 与实际 guard contact 口径不一致。
- 如果在 HUD helper 中写入 `attackTarget`、`holdPosition`、`guardAnchorCarrierID` 或迷雾状态，会破坏只读边界。

禁止项：

- 禁止扩大范围做航母 CAP、巡逻、截击或新命令按钮。
- 禁止修改 contact predicate、目标排序、攻击目标写入、迷雾、潜艇侦测、AI、移动、战斗、生产或 SKRM。
- 禁止新增外部依赖、素材或测试 target。
- 禁止无关重构 `GameScene.swift`。
- 禁止回滚或覆盖用户 / 其他 Agent 的无关改动。
- 禁止伪造本地验证；本轮明确不跑本地验证。
