# v4.21（海军生产按钮来源提示）

## 版本号

v4.21

## 版本依据

- `update_log.md` 当前最新正式玩法版本为 v4.20「多选高价值海军护航摘要」。
- `md/prompt/v4（海军航母）/` 已连续围绕海军、航母、支援、任务和 HUD 可读性做小步增量。
- v4.17 已让 `HELI` / `JET` 生产按钮 subtitle 复用 `productionSource(for:faction:)` 显示当前 `AF` / `CV` 来源或 `need AF/CV`。
- 本轮是 v4.17 同一 HUD 文案模式向海军按钮的窄范围延伸：`SHIP` / `SUB` / `CV` subtitle 显示当前 Shipyard 来源 `SY $cost`，没有 operational Shipyard 时显示 `need SY`。
- 人工指定本轮版本为 v4.21，且要求只做 Agent B 实现提示词；Agent A 本轮不实现代码。

## 背景

当前 `DesertFrontline/GameScene.swift` 已有以下事实：

- `HudAction.buildKind` 把 `.buildBattleship`、`.buildSubmarine`、`.buildCarrier` 映射到 `.battleship`、`.submarine`、`.carrier`。
- `EntityKind.shortCode` 中 Shipyard 的短码是 `SY`，Battleship / Submarine / Carrier 的按钮标题分别是 `SHIP` / `SUB` / `CV`。
- `productionButtonSubtitle(for:)` 当前只对 `.helicopter` / `.fighter` 特殊处理：调用 `productionSource(for:faction:)`，有 Airfield / Carrier 时显示 `AF $cost` 或 `CV $cost`，没有来源时显示 `need AF/CV`。
- 其它生产按钮当前仍显示 `$cost`，因此 `SHIP` / `SUB` / `CV` 上看不到当前是否有 operational Shipyard。
- `queueBuild(kind:faction:showFeedback:)` 和 `canQueueBuild(kind:faction:)` 都依赖 `productionSource(for:faction:)` 判断生产来源；缺来源、缺资金和入队行为已经由现有逻辑负责。
- `productionSource(for:faction:)` 会过滤阵营、存活、`isOperational`、`canProduce(kind)`，玩家侧还会优先当前选中的可生产来源，再按队列深度和实体 ID 选择来源。

本轮目标只是让 HUD subtitle 反映同一个来源口径，减少玩家点海军生产按钮前的信息盲区。

## 目标

1. 增强 `SHIP` / `SUB` / `CV` 生产按钮 subtitle 的生产来源可读性。
   - 当 `productionSource(for: .battleship, faction: .player)` 返回 Shipyard 时，`SHIP` subtitle 显示 `SY $<cost>` 或等价短文案。
   - `SUB` / `CV` 同理，显示 `SY $<cost>`。
   - 当没有 operational Shipyard 可生产对应海军单位时，显示 `need SY`。
2. 必须复用现有 `productionSource(for:faction:)` 口径。
   - 不要在 HUD helper 中重写一套 Shipyard 搜索规则。
   - 不要把未完工、死亡、敌方、中立或不能生产该单位的实体计入来源。
   - 保持选中 Shipyard 时的玩家优先来源逻辑与实际 `queueBuild` 一致。
3. 保持 v4.17 空军按钮行为不回归。
   - `HELI` / `JET` 继续显示 `AF $cost`、`CV $cost` 或 `need AF/CV`。
4. 保持陆军生产按钮 subtitle 不变。
   - `HMV`、`AA`、`TANK`、`ART`、`MECH` 仍显示 `$<cost>`。
5. 保持生产执行链路完全不变。
   - 不改变 `queueBuild(kind:faction:showFeedback:)`。
   - 不改变 `productionSource(for:faction:)`。
   - 不改变 `canQueueBuild(kind:faction:)`。
   - 不改变 `BuildOrder`。
   - 不改变费用、建造时间、资金检查、出兵位置、队列排序、ready pulse 或 rally point。
6. 保持 AI、Shipyard、Carrier、支援、战争迷雾、任务、胜负和按钮布局完全不变。

推荐实现方向：

```swift
private func productionButtonSubtitle(for kind: EntityKind) -> String {
    switch kind {
    case .helicopter, .fighter:
        guard let source = productionSource(for: kind, faction: .player) else {
            return "need AF/CV"
        }
        return "\(source.kind.shortCode) $\(kind.cost)"
    case .battleship, .submarine, .carrier:
        guard let source = productionSource(for: kind, faction: .player) else {
            return "need SY"
        }
        return "\(source.kind.shortCode) $\(kind.cost)"
    default:
        return "$\(kind.cost)"
    }
}
```

这是参考写法，不要求逐字照抄；但必须满足：
