# v4.69 / 启动视口自适应修复

## 版本号

v4.69

## 版本分配依据

v4.68 已完成 `AMOV` 按钮 Carrier guard release 预告并通过实现提交的云端 artifact 验收。用户反馈当前打开后出现黑屏或白屏、无法游玩，这是高优先级启动阻断问题。本轮作为 v4 系列热修，版本递增为 v4.69。

## 背景

当前 `GameView` 直接把固定 `1366x1024`、`.aspectFill` 的 `GameScene` 放进 `SpriteView`。在某些设备、窗口或首次 SwiftUI 布局时，`SpriteView` 可能未明确铺满父容器，或 `GameScene.size` 与实际窗口尺寸不一致，导致用户看到系统白底、黑色空背景或内容裁切到不可玩区域。`GameScene.didMove(to:)` 本身会创建地图、HUD 和初始单位，因此本轮优先修复 SwiftUI 宿主和 SpriteKit viewport 的启动适配。

## 目标

- `GameView` 必须明确让 `SpriteView` 铺满实际窗口。
- `SpriteView` 背后必须有黑色背景兜底，避免场景尚未渲染或视图未铺满时露出白屏。
- `SceneHolder` 继续提供非零初始 `GameScene`，但在 SwiftUI 提供有效容器尺寸后同步 `scene.size`。
- `GameScene.scaleMode` 改为 `.resizeFill`，让 HUD、camera clamp、minimap camera box 和触摸坐标以真实场景尺寸运行。
- 同步 `README.md`、`md/flow/flow.md` 和 `md/flow/flowchart.md` 中关于 SwiftUI 宿主和场景尺寸的当前事实。

## 非目标

- 不重写 `GameScene.didMove(to:)` 初始化链路。
- 不改变地图、单位、AI、战斗、HUD 内容、命令、迷雾、任务、胜负或经济规则。
- 不删除或覆盖用户本地未提交的 Xcode 签名设置。
- 不新增外部依赖、图片素材、启动页或 UIKit wrapper。
- 不更新 `md/test/test.md`，本轮不改变测试流程。

## 当前架构依据

- `DesertFrontlineApp` 只创建 `GameView`。
- `GameView` 是 SwiftUI 宿主，当前用 `SpriteView(scene: holder.scene, preferredFramesPerSecond: 60)` 承载游戏。
- `SceneHolder` 当前创建 `GameScene(size: 1366x1024)` 并设置 `.aspectFill`。
- `GameScene.layoutHUD()`、`didChangeSize(_:)`、`clampCamera(_:)` 和小地图 camera box 都依赖 `scene.size`。
- 项目 deployment target 是 iOS 17，可使用双参数 `onChange(of:)`。

## 实现步骤

1. 同步 main：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - 确认工作区已有改动归属；本轮不得覆盖用户的无关 `project.pbxproj` 签名设置。
2. 修改 `DesertFrontline/GameView.swift`：
   - 用 `GeometryReader` 包住 `SpriteView`。
   - 给 `SpriteView` 显式 `.frame(width: proxy.size.width, height: proxy.size.height)`。
   - 给 `SpriteView` 和外层容器加 `.background(Color.black)`。
   - 保留 `.ignoresSafeArea()`。
   - `SceneHolder` 将 `scene.scaleMode` 改为 `.resizeFill`。
   - 新增 `resizeScene(to:)`，只在 `width > 1`、`height > 1` 且尺寸变化时更新 `scene.size`。
   - 在 `onAppear` 和 `onChange(of: proxy.size)` 调用 `resizeScene(to:)`。
3. 修改 `README.md`：
   - 当前功能中说明 SwiftUI shell 的 SpriteKit scene 会跟随 live window 尺寸，并有黑色启动兜底。
4. 修改 `md/flow/flow.md`：
   - 更新执行流：`GameView` 使用 `GeometryReader` + 全屏 `SpriteView`，`SceneHolder` 使用非零初始尺寸并跟随真实 viewport。
5. 修改 `md/flow/flowchart.md`：
   - 同步核心逻辑图中的 `GameView` / `SceneHolder` 节点说明。
