# v4.8（终局HQ攻击指引）

## 版本号

v4.8

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.7「终局HQ任务情报」。
- v4 阶段已经围绕海军、航母、海岸争夺和任务链做了多轮小步增强。v4.7 已让终局任务详情在合法侦察到 Red HQ 时显示 HP。
- 本轮继续做一个终局 HUD / 战场反馈小增强：玩家进入 `Destroy Red HQ` 阶段并且 Red HQ 已被玩家合法认知时，按下 `AMOV` 会给出明确的 HQ 攻击指引和短暂目标标记，帮助组织最终攻势。

## 背景

当前 `Destroy Red HQ` 阶段已经能在任务面板显示玩家已知 Red HQ 的 HP，但玩家按 `AMOV` 时仍只看到通用 `Tap the map to attack-move.` 文案。终局阶段如果 Red HQ 已经被侦察到，玩家需要更明确地知道可以把 attack-move 指向 HQ 或 HQ 附近地面。

项目已有 `isKnownToFaction(_:observer:)` 作为敌方实体对玩家可感知的合法边界。本轮只复用该边界：只有 Red HQ 对玩家已知时，才显示 HQ 攻击提示和战场标记；未知时继续显示通用 AMOV 文案，不泄露位置、距离或 HP 之外的隐藏信息。

## 目标

1. 为终局阶段的 `AMOV` 准备态增加合法 HQ 攻击指引。
   - 玩家选中可 attack-move 的作战单位后点击 `AMOV`。
   - 如果当前 active mission stage 是 `.destroyHQ`，Red HQ 存活，且 `isKnownToFaction(redHQ, observer: .player)` 为 true，显示短文案，例如 `Red HQ known: tap HQ or map for attack-move.`
   - 同时在 Red HQ 位置显示短暂战场目标标记，例如橙色环和 `HQ` 标签。
2. 未知 Red HQ 不泄露信息。
   - 如果 Red HQ 存活但不被玩家已知，保留通用 `Tap the map to attack-move.`。
   - 不显示隐藏 HQ 位置、距离、标记或额外状态。
3. 保持命令语义不变。
   - `AMOV` 仍只是进入 pending attack-move，之后由玩家点击地图或可见敌人来下达命令。
   - 不自动移动、不自动攻击、不选中 Red HQ、不改目标合法性、不改小地图、不改视野。
4. 文档同步。
   - `README.md`：控制说明或任务面板说明同步终局 `AMOV` 会在 Red HQ 已知时给出 HQ 攻击提示。
   - `md/flow/flow.md`：输入与命令 / 任务或 HUD 说明同步该反馈复用 `isKnownToFaction`，不绕过迷雾。
   - `md/flow/flowchart.md`：玩家输入或任务节点简短同步已知 HQ 的 AMOV 指引。
5. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不改变 `AMOV` 的目标选择、寻路、沿途交战、编队或 `attackMoveDestination` 语义。
- 不改变 `Destroy Red HQ` 完成条件、胜利 / 失败触发、HQ HP、单位伤害、射程、冷却或 AI。
- 不显示隐藏 Red HQ 的位置、距离、HP、标记或攻击路线。
- 不改变 `isKnownToFaction(...)`、`visibleTiles`、`exploredTiles`、`supportRevealTiles`、潜艇侦测或支援侦察逻辑。
- 不新增 HUD 按钮、小地图标记、自动导航、教程弹窗、战役系统、科技树、新地图、新素材或外部依赖。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。
