# v4.53 / 航母警戒翼队分配组成反馈

## 版本号

v4.53

## 版本分配依据

上一轮 v4.52 已补齐多选 bound guard aircraft 的 `Hn` / `Jn` 组成摘要。当前航母 guard wing 的单选、多选、绑定飞机和普通翼队 HUD 已基本具备组成提示，但玩家刚下达 Carrier HOLD 时的即时反馈仍只显示数量或泛化 `CV GUARD` cue。本轮继续在 v4 海军 / 航母 / 操作反馈方向小步推进。

## 背景

当前 `issueHoldPositionOrder(units:)` 在玩家对包含 Carrier 的选择下达 `HOLD` 后：

- `assignCarrierGuardWing(for:)` 会绑定附近玩家 Helicopter / Fighter，并返回总数量。
- 顶部消息追加 `Guard wing n.`。
- 对拥有 bound wing 的 Carrier 显示短暂 deck cue，label 为 `CV GUARD`。

这些反馈能说明绑定成功，但不能马上告诉玩家本次 guard wing 是 Helicopter、Fighter 还是混合翼队。

## 目标

- Carrier HOLD 成功绑定 guard wing 时，顶部消息追加 HEL/JET 组成，例如：
  - `Guard wing 2 H1 J1.`
  - `Guard wing 2 H2.`
  - `Guard wing 1 J1.`
- 对每艘本次 HOLD 后拥有 bound wing 的玩家 Carrier，deck cue 从泛化 `CV GUARD` 改为包含该 Carrier 当前 bound wing 组成的短文本，例如：
  - `GW H1 J1`
  - `GW H2`
  - `GW J1`
- 组成口径复用 `carrierAirWingCompositionSuffix(for:)`，0 数量类型继续省略。

## 非目标

- 不改变 `assignCarrierGuardWing(for:)` 的筛选、去重、写入字段或返回语义。
- 不改变 `guardAnchorCarrierID`、`holdPosition`、station keeping、target priority、combat、AI、fog、production、support、mission、victory 或 SKRM 重置。
- 不新增 CAP、巡逻、截击、新按钮、新 HUD action、新状态机或 AI 策略。
- 不改变 v4.49-v4.52 已有选择面板 `GW` / `CV GW` / `CV GUARD` 摘要。

## 当前架构依据

- `issueHoldPositionOrder(units:)` 是玩家 HOLD 命令入口，已经拥有 `guardWingCount` 和 `holdCarriers`。
- `boundCarrierGuardWing(for:)` 是当前 Carrier bound guard wing 的统一只读统计口径。
- `carrierAirWingCompositionSuffix(for:)` 已统一输出 ` Hn Jn` 组成后缀。
- `showCarrierDeckPulse(at:faction:label:)` 支持短 label，当前已经被 Carrier launch / strike / guard cue 复用。

## 实现步骤

1. 在 `issueHoldPositionOrder(units:)` 中，`assignCarrierGuardWing(for:)` 后，按 `holdCarriers` 回查当前 bound wing。
2. 只在 `guardWingCount > 0` 时生成 feedback。
3. 顶部消息：
   - 聚合所有 selected HOLD Carrier 当前 bound wing。
   - 去重后用 `carrierAirWingCompositionSuffix(for:)` 生成总组成。
   - 返回 `Guard wing \(guardWingCount)\(compositionSuffix).`
4. deck cue：
   - 对每艘 `boundCarrierGuardWing(for:)` 非空的 Carrier 生成 per-carrier 组成。
   - label 使用 `GW\(compositionSuffix)`；若组成为空则退回 `CV GUARD`，但正常有 bound wing 时应有 `Hn` 或 `Jn`。
5. 同步 `README.md`、`md/flow/flow.md`、必要时 `md/flow/flowchart.md`。

## 验收标准

- 玩家对包含 Carrier 的选择下达 HOLD 且实际绑定 HEL/JET 时，顶部消息显示 `Guard wing n Hn/Jn.`。
- 每艘拥有 bound wing 的 Carrier 显示短暂 `GW Hn/Jn` deck cue。
- 没有可绑定 HEL/JET 时，不显示误导性 guard wing 组成或 deck cue。
- 多 Carrier 同时 HOLD 时，顶部组成按所有 relevant bound wing 聚合，deck cue 按每艘 Carrier 自身组成显示。
- 原普通 HOLD marker、release feedback、选择面板、anchor 范围圈、AI guard wing、combat、fog、production、support、mission 和 victory 语义不变。
- 最新 `origin/main` GitHub Actions run 成功，artifact 对应本轮 commit。

- 禁止修改 guard wing 绑定条件、contact predicate、目标排序、迷雾、潜艇侦测、AI、移动、战斗、生产或 SKRM。
- 禁止新增外部依赖、素材、测试 target 或大体积 artifact。
