## 背景

当前 `DesertFrontline/GameScene.swift` 已有以下事实：

- `isActiveSonarSensor(_:)` 判断 active sonar sensor：`kind.hasSonar`、实体存活，且结构必须 `isOperational`。
- `sonarRange(for:)` 当前包含 Submarine 250、Battleship 230、Carrier 210、Helicopter 185、Sonar Buoy 340。
- `sonarInfoLine(for:)` 当前对 active sonar sensor 返回 `Sonar <range>`，并由 `selectionCapabilityInfoLine(for:)` 接入单选面板。
- Sonar Buoy 的结构状态行也会显示 `Sonar <range>  No SCAN`。
- `refreshSelection()` 会对玩家选中的 active sonar sensor 显示 sonar coverage ring；这是 selection-derived visual，不改变 fog 或 detection。
- `isKnownToFaction(_:, observer:)` 是玩家能否知道敌方实体的核心边界；对敌方潜艇还会进入 `isSubmarineDetected(_:, by:)`，依赖 `revealedUntil` 或 sonar sensor 范围。
- `isSubmarineDetected(...)` 只判断潜艇是否被 observer 侦测，不应在 HUD contact count 中被改写或绕过。

当前玩家选中 Sonar Buoy 这类传感器时，只能看到 sonar range，看不到该 sensor 当前范围内有多少个玩家已知 / 已侦测的敌方潜艇接触。v4.22 只补这个选择面板可读性。

## 目标

1. 玩家单选 active sonar sensor 时，selection panel 的 sonar 信息显示当前该 sensor 范围内玩家已知 / 已侦测的敌方潜艇接触数。
   - 目标文案可为 `Sonar 340 Contacts 1`、`Sonar 340 Contact 1`、`Sonar 340 Ctc 1` 或等价短文案。
   - Sonar Buoy 是重点验证对象：玩家选中 operational Sonar Buoy 时，应显示其 340 sonar range 和 contacts 数。
   - Battleship、Carrier、Helicopter、Submarine 这些 active sonar sensor 也可复用同一 helper 显示 contacts。
2. 严守战争迷雾与潜艇隐身边界。
   - 只能统计 `entity.kind == .submarine`。
   - 只能统计 `entity.faction == .enemy`。
   - 只能统计 `entity.isAlive == true`。
   - 必须先满足 `isKnownToFaction(entity, observer: .player) == true`。
   - 必须满足 `sensor.node.position.distance(to: entity.node.position) <= sonarRange(for: sensor.kind)`。
   - 不得因 contact count 全图读取隐藏潜艇、显示隐藏潜艇数量、泄露隐藏潜艇位置或让未知潜艇影响 HUD 文案。
3. 只读接入 selection panel。
   - 推荐新增小 helper，例如 `sonarContactCount(for sensor: GameEntity) -> Int` 或 `knownEnemySubmarineContacts(for sensor: GameEntity) -> Int`。
   - helper 只能被 HUD / selection 文案读取，不能改变任何实体、fog、support 或 AI 状态。
   - `sonarInfoLine(for:)` 可从 `Sonar <range>` 扩展为 `Sonar <range> Contacts <count>`。
4. 未完工 Sonar Buoy 不显示 contacts。
   - 继续由 `isActiveSonarSensor(_:)` 的 operational 边界控制。
   - 施工中 Sonar Buoy 仍显示 offline / coastal role / pending mission 状态，不被当作 active sonar sensor。
5. 保持现有 selection panel 行数与布局策略。
   - 优先在既有 sonar info 片段内追加短 contacts 文案。
   - 不新增 HUD 按钮、面板行、图标、覆盖圈、目标标记或调试显示。
6. 同步玩家可见文档和核心流程文档。

### 3. 新增只读 contact count helper

推荐新增：

```swift
private func sonarContactCount(for sensor: GameEntity) -> Int {
    guard sensor.faction == .player,
          isActiveSonarSensor(sensor) else { return 0 }
    let range = sonarRange(for: sensor.kind)
    return entities.values.filter { contact in
        contact.kind == .submarine &&
            contact.faction == .enemy &&
            contact.isAlive &&
            isKnownToFaction(contact, observer: .player) &&
            sensor.node.position.distance(to: contact.node.position) <= range
    }.count
}
```

这是参考写法，不要求逐字照抄；但必须满足：

- 使用 `isKnownToFaction(contact, observer: .player)`，不能直接用 `isSubmarineDetected(...)` 代替完整玩家已知边界。
- 使用 `sonarRange(for: sensor.kind)`，不能硬编码 Sonar Buoy 340 或其它 range。
- 只统计敌方潜艇，不统计玩家潜艇、中立实体、敌方水面舰、死亡潜艇或未知潜艇。
- 不修改 `contact.revealedUntil`、`supportRevealTiles`、`visibleTiles`、`exploredTiles`、任何 node hidden 状态、任何 AI 字段或任何命令状态。

### 4. 扩展 sonar info 文案

推荐在 `sonarInfoLine(for:)` 中扩展：

```swift
private func sonarInfoLine(for entity: GameEntity) -> String? {
    guard isActiveSonarSensor(entity) else { return nil }
    let range = Int(sonarRange(for: entity.kind))
    if entity.faction == .player {
        return "Sonar \(range) Contacts \(sonarContactCount(for: entity))"
    }
    return "Sonar \(range)"
}
```

要求：

- 玩家 active sonar sensor 显示 range + contacts。
- 非玩家 sensor 如因现有选择逻辑进入面板，最多保留原 `Sonar <range>`，不要用玩家 contact count 泄露信息。
- 文案应短，避免现有 4 行选择面板溢出；必要时可用 `Ctc` 或 `Contact` 等短格式。
