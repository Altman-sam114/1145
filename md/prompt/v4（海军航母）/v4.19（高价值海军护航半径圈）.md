# v4.19（高价值海军护航半径圈）

## 版本号

v4.19

## 版本分配依据

- `update_log.md` 当前最新正式玩法版本为 v4.18「高价值海军护航状态面板」。
- v4.18 已让单选 Carrier / Battleship 显示 `Escort x/y`，但玩家仍需要靠目测判断哪些单位落在 620 护航统计半径内。
- 本轮是 v4.18 的直接可视化补充：选中玩家 Carrier 或 Battleship 时，在战场上显示高价值海军护航半径圈，帮助玩家理解 `Escort x/y` 统计范围。
- 本轮只新增 selection-derived visual，不改变护航统计、AI、玩家命令、移动、战斗、生产、支援、迷雾或胜负。

## 背景

当前相关事实：

- `GameEntity` 已有 `sonarCoverageNode`，选中玩家 active sonar sensor 时，`refreshSelection()` 会按 `shouldShowSonarCoverage(for:)` 显示声呐覆盖圈。
- `configureSonarCoverageNode(for:)` 配置声呐覆盖圈 path、颜色、lineWidth、glow 和隐藏状态。
- v4.18 新增 `highValueNavalEscortRadius = 620`、`highValueNavalEscortRequirement(for:)`、`highValueNavalEscortLine(for:)` 和 `nearbyNavalEscortCount(for:)`。
- v4.18 护航统计只读统计同阵营、存活、operational、非结构、可攻击、非当前实体、非 Battleship / Carrier、距离不超过 620 的附近单位。
- 目前 v4.18 只在 selection info 文案里显示 `Escort x/1` 或 `Escort x/2`，没有把 620 范围画出来。
- 人工要求不得用本地测试、本地 build、本地 static check 或 `git diff --check` 作为验收，只认 GitHub Actions artifact。

## 目标

1. 为玩家选中的高价值海军显示护航半径圈。
   - 玩家选中 Carrier 时，显示 620 护航半径圈。
   - 玩家选中 Battleship 时，显示 620 护航半径圈。
   - 多选多个玩家 Carrier / Battleship 时，显示多个半径圈。
2. 半径圈必须是只读视觉层。
   - 不改变 `nearbyNavalEscortCount(for:)` 的统计规则。
   - 不改变 AI escort / wave 逻辑。
   - 不改变玩家命令、移动、战斗、生产、支援、迷雾或目标合法性。
3. 与声呐覆盖圈保持清晰区分。
   - 不复用 `sonarCoverageNode` 表达护航。
   - 推荐给 `GameEntity` 增加独立 `escortCoverageNode` 或等价节点。
   - 颜色建议用暖黄色 / 金色低 alpha，避免被误认为声呐侦测范围。
4. 显示条件必须严格。
   - 只在该实体被选中时显示。
   - 只对玩家 `faction == .player` 显示。
   - 只对存活、operational、非结构的 Battleship / Carrier 显示。
   - 未选中、死亡、敌方、中立、非高价值海军不得显示。
5. 文档同步。
   - `README.md` 记录选中 Carrier / Battleship 会显示护航半径圈。
   - `md/flow/flow.md` 记录护航半径圈是 selection-derived visual，不改变 AI、escort 统计、迷雾或命令。
   - `md/flow/flowchart.md` 如需只在 HUD 节点短语中补「护航半径圈」或等价描述。

## 非目标

- 不实现自动护航、CAP、巡逻、follow 行为、编队锁定或新命令。
- 不改变 v4.18 `Escort x/y` 文案或统计规则，除非只是调用已有 helper 判断是否高价值海军。
- 不改变 `highValueNavalEscortRadius` 的数值。
- 不改变 `nearbyNavalEscortCount(for:)`、AI `enemyEscortCount(near:in:)`、`canJoinEnemyAssaultWave`、`isEnemyAssaultEscort`。
- 不改变声呐覆盖圈显示、声呐检测、潜艇暴露或 `isKnownToFaction(...)`。
- 不改变 HUD 按钮、selection info 行数、小地图、相机、触摸分流、任务或胜负。
- 不更新 `md/test/test.md`。
- 不更新 `update_log.md`；Agent C 在 artifact 验收通过后追加 v4.19 正式记录。
- 不新增 XCTest target、workflow、Xcode project 配置、外部素材或外部依赖。

2. 修改 `DesertFrontline/GameScene.swift`。

推荐实现方向：

- 在 `GameEntity` 增加：

```

- 在创建实体节点时，类似 sonar coverage：

```

- 在 `refreshSelection()` 中新增：

```swift
entity.escortCoverageNode.isHidden = !shouldShowHighValueNavalEscortCoverage(for: entity)
```

- 新增 helper：

```swift
private func shouldShowHighValueNavalEscortCoverage(for entity: GameEntity) -> Bool {
    selectedIDs.contains(entity.id) &&
        entity.faction == .player &&
        entity.isAlive &&
        entity.isOperational &&
        !entity.kind.isStructure &&
        highValueNavalEscortRequirement(for: entity.kind) != nil
}

private func configureEscortCoverageNode(for entity: GameEntity) {
    guard highValueNavalEscortRequirement(for: entity.kind) != nil else {
        entity.escortCoverageNode.isHidden = true
        return
    }
    let range = highValueNavalEscortRadius
    entity.escortCoverageNode.path = CGPath(
        ellipseIn: CGRect(x: -range, y: -range, width: range * 2, height: range * 2),
        transform: nil
    )
    entity.escortCoverageNode.fillColor = UIColor(red: 1.0, green: 0.74, blue: 0.24, alpha: 0.025)
    entity.escortCoverageNode.strokeColor = UIColor(red: 1.0, green: 0.78, blue: 0.30, alpha: 0.34)
    entity.escortCoverageNode.lineWidth = 2
    entity.escortCoverageNode.glowWidth = 1.0
    entity.escortCoverageNode.zPosition = -5
    entity.escortCoverageNode.isHidden = true
}
```

- 上面只是示例，不要求逐字照抄；但必须保持独立节点和只读显示。
- 不要改变 `sonarCoverageNode` 的颜色、条件或语义。
- 不要把 escort coverage 写入 fog、visibleTiles、exploredTiles、supportRevealTiles 或目标合法性。
