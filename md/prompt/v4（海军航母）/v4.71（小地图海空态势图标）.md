# v4.71 / 小地图海空态势图标

## 版本号

v4.71

## 版本分配依据

v4.70 已修复启动阶段的 HUD 布局闪退风险，并由 Agent C 验收最新 `origin/main` 云端 artifact 通过。当前云端截图证明游戏能够启动和渲染，但小地图上的结构、陆军、空军和海军仍主要使用同类圆点，混合兵种密集时不易快速判断海空态势。本轮继续在 v4 阶段做一个范围有限、可由云端截图直接验收的操作可读性增量，因此版本递增为 v4.71。

## 背景

Desert Stormfront 的移动端 RTS 体验依赖紧凑小地图和清楚的兵种轮廓。`GameScene.updateMinimap()` 当前只按结构与非结构调整圆点半径，再用阵营色区分归属。玩家能够看到位置，但无法从符号形状判断空军、海军、潜艇或高价值舰艇，也无法在小地图上快速定位当前选择。

## 目标

- 小地图用稳定形状区分领域：
  - 结构：方形。
  - 陆军：圆形。
  - 空军：三角形。
  - 水面海军：菱形。
  - 潜艇：空心或低填充菱形。
- Battleship / Carrier 使用略大的海军符号和更清楚描边。
- 当前选中的玩家实体在小地图上显示高对比外圈。
- 保持阵营色、控制点、相机框和点击跳转行为不变。
- 敌方实体继续先通过 `isKnownToFaction(..., observer: .player)`，不得泄露迷雾中的单位或潜艇。

## 非目标

- 不修改单位属性、战斗、AI、生产、移动、声呐、潜艇暴露、战争迷雾、任务、经济或胜负规则。
- 不修改主战场单位模型、海岸、水面、HUD 命令按钮布局或触摸命令。
- 不新增外部素材、第三方依赖或 XCTest target。
- 不运行本地测试、本地 Xcode build、本地模拟器或本地探针。
- 不提交工作区中无关的 `DesertFrontline.xcodeproj/project.pbxproj` 团队签名改动。

## 当前架构依据

- `updateMinimap()` 每帧清空 `minimapBlipsNode`，遍历存活实体并先过滤未知敌方实体。
- `EntityKind.domain` 已区分 land / air / naval / structure，`EntityKind` 也能识别 Submarine、Battleship 和 Carrier。
- `diamondPath(width:height:)` 与 `trianglePath(size:)` 已提供可复用的 SpriteKit 路径。
- `selectedIDs` 是当前选择的唯一来源，可用于只读绘制小地图选择外圈。
- 小地图点击仍由现有 `minimapFrame` 和 `worldPoint(fromMinimap:)` 处理，本轮不改变命中区域。

## 实现步骤

1. 确认 `main`、`origin/main` 和工作区状态，只保留并避开用户未暂存的 `project.pbxproj` 改动。
2. 在 `DesertFrontline/GameScene.swift` 中新增小地图实体符号 helper：
   - 根据结构、领域和具体海军类型创建轻量 `SKShapeNode`。
   - 继续复用 `minimapBlipColor(for:)` 的阵营色。
   - 控制图标尺寸在约 5-9 pt，避免密集编队糊成大块。
3. 修改 `updateMinimap()`：
   - 保留现有未知敌方过滤条件。
   - 使用新 helper 生成领域符号。
   - 对 `selectedIDs` 中的实体增加白色或青白色外圈。
   - 保持控制点、小地图相机框和点击跳转逻辑不变。
4. 更新 `README.md` 的小地图当前功能描述。
5. 更新 `md/flow/flow.md` 的 HUD / 小地图当前事实。
6. commit 并直接 push `origin/main`，只使用 GitHub Actions 云端验证。
