- `md/prompt/v4（海军航母）/` 当前最新提示词为 v4.22「声呐选中面板潜艇接触数」。

- 本轮沿用上述 selection-derived visual 模式，为玩家选中 Mechanic 增加只读维修范围圈，并让单选面板显示自动维修范围与范围内己方受损可修目标数，不改变维修算法或任何玩法数值。

## 背景

当前 `DesertFrontline/GameScene.swift` 已有以下事实：

- `EntityKind.mechanic` 是 War Factory 可生产的陆地机动单位，玩家和 AI 都可拥有。
- `updateRepair(dt:)` 每帧遍历所有存活 Mechanic，寻找同阵营、存活、非自身、`hp < maxHP` 的最近目标；当目标距离 Mechanic 小于 `95` 时，以 `CGFloat(dt) * 22` 自动维修，并偶发 `showRepairSpark(at:)`。
- `updateRepair(dt:)` 当前硬编码维修范围 `95`、每秒维修量 `22`，且最近目标选择没有排除结构或特定 domain。
- `singleSelectionInfo(for:)` 为单选面板入口；非结构单位会通过 `mobileStatusLine(for:)` 替换第 4 行状态。
- `mobileStatusLine(for:)` 当前对 Mechanic 返回 `Repairs nearby allies`，没有显示范围或当前可修目标数。
- `GameEntity` 目前已有 `sonarCoverageNode` 和 `escortCoverageNode` 两个只读覆盖圈节点。
- `configureSonarCoverageNode(for:)`、`shouldShowSonarCoverage(for:)`、`configureEscortCoverageNode(for:)`、`shouldShowHighValueNavalEscortCoverage(for:)` 和 `refreshSelection()` 已形成稳定模式：覆盖圈只由选择状态派生，只在玩家选中符合条件的实体时显示，不写入 fog、命令、AI、战斗或统计规则。
- 现有 `README.md` 已记录 automatic mechanic repairs，但未记录选中 Mechanic 的维修范围 / 当前可修目标反馈。

本轮要解决的是玩家可读性：选中 Mechanic 时能直接知道自动维修半径和半径内有多少己方受损可修目标，并在战场上看到该半径。该反馈必须严格只读。

## 目标

1. 玩家单选 Mechanic 时，selection panel 显示自动维修范围和范围内己方受损可修目标数。
   - 推荐短文案：`Repair 95  Damaged 2`、`Repair R95  Dmgd 2` 或等价简短格式。
   - 范围数值必须与 `updateRepair(dt:)` 当前真实范围一致：`95`。
   - 目标数必须统计该 Mechanic 范围内同阵营、存活、非自身、`hp < kind.maxHP` 的可修目标。
   - 统计口径应和 `updateRepair(dt:)` 当前目标候选口径保持一致，只是从「最近一个」变成「范围内数量」。
2. 玩家选中 Mechanic 时，战场显示维修范围圈。
   - 单选玩家 Mechanic 时显示范围圈。
   - 多选多个玩家 Mechanic 时可以显示多个范围圈；如果实现选择只支持通过 selectedIDs 显示，也应自然覆盖单选和多选。
   - 未选中 Mechanic 不显示。
   - 敌方 Mechanic 不显示给玩家范围圈。
   - 死亡或非玩家 Mechanic 不显示。
3. 维修范围圈必须是 selection-derived visual。
   - 推荐新增独立节点，例如 `repairCoverageNode`，不要复用 `sonarCoverageNode` 或 `escortCoverageNode`。
   - 颜色建议使用绿色 / 青绿色低 alpha，与声呐蓝色、护航暖黄色区分。
   - `zPosition` 应低于单位主体，不遮挡选择圈、血条、单位和 HUD；可参考现有 sonar / escort coverage。
4. 保持现有维修玩法完全不变。
   - 不改变 `updateRepair(dt:)` 的范围 `95`。
   - 不改变每秒维修量 `22`。
   - 不改变最近目标选择逻辑。
   - 不改变 repair spark 频率或视觉效果。
   - 不改变 AI 生产 Mechanic、低血撤退回修、Red `REPR` 评分、支援 Field Repair、移动、战斗、迷雾、任务或胜负。
5. 同步玩家可见文档和核心流程文档。

### 3. 新增只读维修目标统计 helper

推荐新增：

```swift
private func damagedRepairTargetCount(for mechanic: GameEntity) -> Int {
    guard mechanic.kind == .mechanic,
          mechanic.isAlive else { return 0 }
    return entities.values.filter { target in
        target.faction == mechanic.faction &&
            target.isAlive &&
            target.id != mechanic.id &&
            target.hp < target.kind.maxHP &&
            target.node.position.distance(to: mechanic.node.position) < mechanicRepairRange
    }.count
}
```

要求：

- 统计口径与 `updateRepair(dt:)` 当前候选保持一致：同阵营、存活、非自身、受损。
- 范围比较应与 `updateRepair(dt:)` 一致，保留 `< mechanicRepairRange`。

### 5. 新增维修范围覆盖圈节点

推荐在 `GameEntity` 新增：

```swift
let repairCoverageNode = SKShapeNode()
```

在 `configureEntityNode(_:)` 中参考 sonar / escort 模式：


新增 helper：

```swift
private func shouldShowMechanicRepairCoverage(for entity: GameEntity) -> Bool {
    selectedIDs.contains(entity.id) &&
        entity.faction == .player &&
        entity.isAlive &&
        entity.kind == .mechanic
}

private func configureRepairCoverageNode(for entity: GameEntity) {
    guard entity.kind == .mechanic else {
        entity.repairCoverageNode.isHidden = true
        return
    }

    let range = mechanicRepairRange
    entity.repairCoverageNode.path = CGPath(
        ellipseIn: CGRect(x: -range, y: -range, width: range * 2, height: range * 2),
        transform: nil
    )
    entity.repairCoverageNode.fillColor = UIColor(red: 0.28, green: 1.0, blue: 0.55, alpha: 0.035)
    entity.repairCoverageNode.strokeColor = UIColor(red: 0.38, green: 1.0, blue: 0.62, alpha: 0.42)
    entity.repairCoverageNode.lineWidth = 2
    entity.repairCoverageNode.glowWidth = 1.0
    entity.repairCoverageNode.zPosition = -4
    entity.repairCoverageNode.isHidden = true
}
```

上面只是参考写法，不要求逐字照抄；但必须满足：

- 节点独立于 sonar / escort coverage。
- 只对 Mechanic 配置有效范围 path。
- 范围取 `mechanicRepairRange`，不要写成其他数值。
- 颜色与现有 sonar 蓝色和 escort 暖黄色可区分。
- 只读显示，不写任何 fog、support、AI、targeting 或 repair 状态。
