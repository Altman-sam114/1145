# v4.58 / AI 航母战斗群携带警戒翼队

## 版本号

v4.58

## 版本分配依据

上一轮 v4.57 已让 Red routine attack-move 波次不再把当前已绑定 Carrier guard wing 的 HEL/JET 当作普通 idle assault candidate 抽走。当前遗留边界是：如果 Red Carrier 自身被 routine assault wave 选中出击，它已有的 bound guard wing 仍不会跟随 Carrier 加入同一 wave。本轮把“保留护航”推进为“Carrier 被接受出击时携带自身有效 guard wing”，属于 v4 海军 / 航母 AI battle group 的小步增量。

## 背景

当前相关事实：

- `updateAI(dt:)` 先调用 `updateEnemyCarrierGuardWings(enemyUnits)`，再在满足攻击条件时调用 `commandEnemyAttackers(enemyUnits)`。
- `commandEnemyAttackers(_:)` 的 `idleAttackers` 已排除 `isEnemyCarrierGuardWingReservedForAnchor(_:)`，避免 bound Red HEL/JET 被普通 assault role 选走。
- `enemyProvisionalAssaultWave(...)` 可以选入 Red Carrier，但当前 provisional wave 不包含该 Carrier 已绑定的 guard wing。
- `canJoinEnemyAssaultWave(...)` 会对 Carrier 做 escort / mixed-force 门槛判断；如果 bound wing 不进入同一候选集合，就无法作为该 Carrier 的近身护航参与门槛。
- `issueFormationMove(...)` 会清理参与编队单位的 HOLD / guard anchor 状态，所以只应在 Carrier 最终被接受进 wave 时才让 bound wing 跟随出击。

## 目标

- 当 Red Carrier 进入 routine provisional wave 时，把它当前有效、空闲、同 anchor 的 bound HEL/JET 追加到本轮 assault 候选集合。
- 这些 bound wing 可作为该 Carrier 的 escort 参与 `canJoinEnemyAssaultWave(...)` 判断。
- 最终 wave 中只有 anchor Carrier 自身也通过并保留在 wave 内时，bound wing 才会跟随同一 `issueFormationMove(... attackMove: true)` 出击。
- 如果 Carrier 因低血、Veteran / Elite 保护、escort 门槛或其他既有规则未进入最终 wave，则它的 bound wing 继续保留 guard anchor，不单独出击。

## 非目标

- 不改变玩家 HOLD / AMOV / move / attack 命令释放 guard wing 的语义。
- 不改变 Carrier guard wing 的绑定数量、半径、候选排序、HUD 文案、范围圈、释放反馈或 composition 统计。
- 不新增 CAP、巡逻、截击、新按钮、新 AI 状态机或 AI 作弊侦察。
- 不改变 Red Carrier / Battleship 的高价值海军门槛、低血撤退、Veteran / Elite 保护、capture reservation、fog、潜艇侦测、生产、支援、任务、胜负或 SKRM 重置。
- 不让 bound wing 在 anchor Carrier 未进入最终 wave 时脱离护航。

## 当前架构依据

- Red AI 的 routine assault wave 构建集中在 `commandEnemyAttackers(_:)`。
- `boundCarrierGuardWing(for:)` 和 `carrierGuardAnchor(for:)` 已能确认当前有效 Carrier guard anchor。
- `isAvailableEnemyCarrierGuardWing(_:)` 已定义 Red HEL/JET 是否空闲、可用、operational、非 reservation、非 retreating。
- 最小安全改动是在 `enemyProvisionalAssaultWave(...)` 之后、`retreatEnemyAssaultUnits(...)` 和 `enemyAssaultWave(...)` 周围扩展候选集合，并在最终 wave 前过滤掉 anchor Carrier 未入波的 bound wing。

## 实现步骤

1. 在 `GameScene.swift` 增加 helper，例如 `enemyAssaultCandidatesWithCarrierGuardWings(from:)`：
   - 输入 provisional wave。
   - 复制已有单位，使用 id set 防重复。
   - 对其中每艘 Red Carrier，枚举 `boundCarrierGuardWing(for:)`。
   - 只追加满足 `isAvailableEnemyCarrierGuardWing(wing)`、`!shouldRetreatEnemyAssaultUnit(wing)` 且 `carrierGuardAnchor(for: wing)?.id == carrier.id` 的 wing。
2. 增加 helper，例如 `enemyAssaultWaveKeepingCarrierGuardAnchors(_:)`：
   - 输入 `enemyAssaultWave(...)` 的 accepted wave。
   - 对非 bound wing 单位直接保留。
   - 对 `isEnemyCarrierGuardWingReservedForAnchor(unit)` 的 wing，只有其 `carrierGuardAnchor(for:)` 的 Carrier id 也在 accepted wave id set 内才保留。
3. 修改 `commandEnemyAttackers(_:)`：
   - `let provisionalWave = ...`
   - `let assaultCandidates = enemyAssaultCandidatesWithCarrierGuardWings(from: provisionalWave)`
   - 只对原始 `provisionalWave` 执行低血撤退。
   - 用过滤掉 retreating id 且排除已撤退 anchor Carrier 之 bound wing 的候选执行 `enemyAssaultWave(...)`。
   - 再用 `enemyAssaultWaveKeepingCarrierGuardAnchors(...)` 得到最终 `wave`。
4. 低血撤退仍只对原始 `provisionalWave` 执行，避免 anchor Carrier 未最终出击时额外追加的 bound wing 被单独改成撤退状态。
5. `enemyAssaultCandidatesWithCarrierGuardWings(from:)` 追加 wing 时排除 `shouldRetreatEnemyAssaultUnit(wing)`，避免低血 bound wing 被战斗群带走。
6. 增加过滤 helper，确保如果 anchor Carrier 本轮进入 retreating id，则其 bound wing 不进入 `enemyAssaultWave(...)` 的 escort 计算。
7. 同步 README：
   - AI 描述改为：bound Carrier guard HEL/JET 平时不被普通 routine wave 抽走，但 anchor Carrier 自身被接受出击时可随同加入 battle group。
8. 同步 `md/flow/flow.md`：
   - AI 进攻段落写清上述候选扩展和最终保留条件。
9. `md/flow/flowchart.md` 如未描述该细节，可不改。

## 验收标准

- 已绑定 Red Carrier guard wing 的 HEL/JET 仍不会作为普通 idle assault candidate 被 routine wave 抽走。
- Red Carrier 进入 provisional assault wave 时，它自身有效、空闲、同 anchor 的 bound HEL/JET 会被追加到本轮 assault candidates。
- 这些 wing 可作为 escort 帮助该 Carrier 通过 `canJoinEnemyAssaultWave(...)` 的高价值海军门槛。
- 本轮应撤退的低血 bound wing 不会被追加进战斗群；本轮 anchor Carrier 被低血撤退移出候选时，其 bound wing 不参与 escort 计算或最终出击。
- 如果 anchor Carrier 未进入最终 wave，bound wing 不会单独进入最终 `issueFormationMove(...)`。
- 玩家 Carrier guard wing、HUD、范围圈、释放反馈、combat、fog、production、support、mission 和 victory 语义不变。
- 最新 `origin/main` GitHub Actions run 成功，artifact 对应本轮 commit。
