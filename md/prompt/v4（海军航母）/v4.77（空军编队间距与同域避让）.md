# v4.77 / 空军编队间距与同域避让

## 版本号与依据

v4.77。v4.76 已由最终 `origin/main` run `29128634076` artifact 验收通过，HEL/JET 模型细节增多；现有 58pt air slot 和共享目标中心接近点容易造成机体、投影与生命条叠放。

## 目标

- air formation slot 从 58 提高到 84，保持 land 40、naval 70 不变。
- 空军移动途中对 76pt 内同阵营空军加入轻量分离方向，不引入物理系统。
- 分离方向必须保留到目的地的正向分量，避免反向或停滞。
- 空军攻击接近点按 attacker id 分配目标周围 8 个椭圆站位，避免全部飞向中心。
- 玩家和 AI 继续共享 `issueFormationMove(...)`、`updateMovement(...)` 和 `attackDestination(...)`。
- CI air scene 使用真实 formation helper 编排两方各 3 架，直接验证间距。

## 非目标

- 不新增碰撞、推挤伤害、速度加成、队形 UI、队长状态或飞行高度层。
- 不修改 land / naval slot、目标选择、攻击射程、伤害、AI 战略、Carrier guard、经济、任务、胜负或迷雾。
- 不新增依赖或 XCTest target。
- 不运行本地测试、build、模拟器或探针。
- 不提交用户 `project.pbxproj` 团队号改动。

## 实现步骤

1. 增加 air spacing、separation radius/weight 和 attack station radius 常量。
2. `formationOffsets(...)` 仅把 `.air` spacing 改为 84。
3. `updateMovement(...)` 在 desired direction 后调用空军分离 helper；只扫描同阵营存活 air，限制最小 forward dot。
4. `attackDestination(...)` 为 air 返回 8 槽椭圆站位；空军移动到站位容差 18 后停止。
5. CI 新增每方第二架 Fighter，以真实 formation offsets 放置 HEL + 2 JET，并保留导弹证据。
6. 更新 README、flow、flowchart、test 和提示词。
7. commit `v4.77: 优化空军编队间距与避让`，直推 `origin/main`，只做云端验证。
