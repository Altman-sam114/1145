# v4.72 / 海岸浅滩与浪线细节

## 版本号

v4.72

## 版本分配依据

v4.71 已通过两次最新 `origin/main` GitHub Actions artifact 验收，小地图能够区分结构、陆军、空军、海军和潜艇。总目标仍要求地图和海战画面更接近 Desert Stormfront 的等距沙漠 RTS 风格；当前水陆交界仍是硬切色块，缺少浅滩和岸边浪线。本轮继续以单一地图视觉增量递增为 v4.72。

## 背景

现有 `addTerrainDetail(for:terrain:at:)` 只在部分水格绘制一条静态白色波纹。水格与沙地、道路或山脊直接相接，海岸资产附近缺少浅水过渡和冲岸反馈。`buildTerrain()`、`isWaterCoordinate(_:)`、`isCoastal(_:)`、陆海寻路和海岸建筑合法性已经稳定，本轮只应在 `mapNode` 的静态装饰层补充视觉，不改变任何地形语义。

v4.71 的云端截图聚焦默认战场中心，海岸主体容易被 HUD 遮挡。本轮让 CI 启动 App 时通过 `SIMCTL_CHILD_DESERT_CI_CAMERA_FOCUS=coast` 传入仅用于截图的环境变量，`GameScene` 在该变量存在时把初始相机定位到玩家 Shipyard 海岸；普通玩家启动不设置该变量，默认镜头保持不变。Agent C 首次检查 v4.72 实现 run 时确认浅滩和浪线已渲染，但 runner 延迟期间 skirmish 已推进到玩家失败、Shipyard 被摧毁，无法稳定证明初始海岸资产。因此同一轮增加 `DESERT_CI_CAPTURE_MODE=1`，仅在云端截图时保持 HUD 渲染并暂停玩法推进。

## 目标

- 邻接陆地的水格增加浅水青绿色层次，形成深水到浅滩的视觉过渡。
- 沿水格与陆格相接的等距菱形边绘制柔和冲洗带和细白泡沫线。
- 开阔水格保留稀疏波纹，并略微加入等距方向变化，避免水面完全平坦。
- 每条岸边只增加少量简单 `SKShapeNode`，控制节点数量和 zPosition。
- GitHub Actions simulator launch 使用 CI 环境变量让截图聚焦玩家海岸，直接验证浅滩、浪线、Shipyard 和海军区域。
- CI 截图模式暂停经济、AI、战斗和胜负推进，避免 runner 延迟改变初始战场；普通玩家实时循环不受影响。
- 默认玩家镜头、地形分类、建造、寻路、AI、战斗、迷雾、任务和胜负完全不变。

## 非目标

- 不修改 `buildTerrain()`、`isWaterCoordinate(_:)`、`isCoastal(_:)` 或 `isPassable(...)` 的语义。
- 不新增潮汐、动态水体、粒子系统、物理碰撞、天气、昼夜或可交互海岸。
- 不修改海军单位属性、航母警戒翼、潜艇侦测、声呐、海岸建筑规则或 AI。
- 不新增外部素材、第三方依赖或 XCTest target。
- 不运行本地测试、本地静态检查、本机 Xcode build、本地模拟器或本地探针。
- 不提交工作区中无关的 `DesertFrontline.xcodeproj/project.pbxproj` 团队号改动。

## 当前架构依据

- `drawMap()` 先创建地形菱形，再调用 `addTerrainDetail(...)` 和 `addScenery(...)`，最后创建 fog 菱形。
- `terrain` 在 `drawMap()` 前完整构建，可安全只读检查相邻格。
- `tileWidth` / `tileHeight` 和 `diamondPath(width:height:)` 定义等距菱形几何。
- 相邻 row / col 的四个正交格分别对应菱形的四条边，适合生成岸线 segment。
- terrain detail 当前位于 tile 之上、scenery / entity / fog 之下；新增节点应保持同一层级。
- `didMove(to:)` 当前把 camera 设为中央 tile；可通过一个只读 CI 环境变量选择海岸焦点，同时保留默认值。
- workflow 已构建、安装、启动、等待、截图并保存 App 日志，只需在 `simctl launch` 时传入 `SIMCTL_CHILD_` 环境变量。

## 实现步骤

1. 确认本地 `main` 与 `origin/main` 一致，保留并避开用户未暂存的 `project.pbxproj` 改动。
2. 修改 `DesertFrontline/GameScene.swift`：
   - 新增返回水格岸线 segment 的纯视觉 helper，只检查四个正交邻格是否为有效非水格。
   - 对存在岸线 segment 的水格绘制一个略内缩的浅水菱形，颜色透明度保持克制。
   - 每条岸边绘制一条较宽、低透明度青白冲洗带和一条较细白色泡沫线，端点略向水格内部收缩，避免 tile 缝隙溢出。
   - 开阔水面波纹保持稀疏，不增加每帧动画。
   - 新增 CI 相机焦点 helper：仅当 `DESERT_CI_CAMERA_FOCUS == "coast"` 时使用玩家 Shipyard 附近 tile；默认仍使用原中央 tile。
   - 当 `DESERT_CI_CAPTURE_MODE == "1"` 时，`update(_:)` 只保持时间戳和 HUD 刷新，不推进玩法系统；没有该变量时执行原实时循环。
3. 修改 `.github/workflows/ci-results.yml`：
   - 在 `simctl launch` 命令前设置 `SIMCTL_CHILD_DESERT_CI_CAMERA_FOCUS=coast` 和 `SIMCTL_CHILD_DESERT_CI_CAPTURE_MODE=1`。
   - 保持 PID 解析、等待、截图、日志和 `kill -0` 门禁不变。
4. 更新 `README.md` 的地图视觉事实。
5. 更新 `md/flow/flow.md`、`md/flow/flowchart.md` 的地图渲染事实和 CI 海岸截图焦点。
6. 更新 `md/test/test.md`，说明 simulator screenshot 在云端聚焦海岸用于地图 / 海战视觉复核，默认 App 行为不受影响。
7. commit 并直接 push `origin/main`，只使用 GitHub Actions 云端验证。

## 风险和禁止项

- 风险：每条岸边两条 shape 会增加静态节点；只对邻陆水格生成，不添加动画、纹理或粒子。
- 风险：岸线端点若使用错误的 row / col 映射会画到错误菱形边；必须按四个正交邻格对应四边。
- 风险：泡沫 zPosition 过高可能压住单位或 fog；保持在 terrain detail 层。
- 风险：CI 相机焦点和截图暂停不能改变普通玩家默认镜头或实时循环；必须由环境变量严格门控。
- 禁止改动 terrain 分类、海岸建造合法性、移动代价、路径搜索、AI、战斗、经济、任务、胜负或迷雾。
- 禁止运行本地测试或构建。
- 禁止提交用户无关的 `project.pbxproj` 团队号改动。
- 禁止用旧 run、旧 artifact 或本地输出冒充 v4.72 验收。
