# v4.79 / 五页单排战术 HUD

## 版本号与依据

v4.79。v4.78 已由最终 `origin/main` run `29137984987` artifact 验收通过。当前云端 1206x2622 横屏截图中，底部 24 个按钮被固定分成两排，遮挡约 114pt 战场高度；Noble Master 官方 Desert Stormfront 的 5 张 1280x800 截图则以紧凑左侧命令面板和底部槽位优先保留战场主体。

## 视觉参考

- Noble Master 官方产品页与 5 张截图：`https://noblemaster.itch.io/desert-stormfront`。
- 官方截图将移动、编组、修理、停止等上下文命令集中展示，底部只保留紧凑槽位，战场单位和目标标记占据主要画面。
- 本轮借鉴“按任务聚合、战场优先”的信息架构，不复制原 UI、素材或按钮排列。

## 目标

- 将 24 个旧 HUD 动作完整分配到 `TACT / BUILD / AIR / SEA / SUP` 五页。
- 五个页签常驻，与当前页动作共用一排；当前页最多 7 个动作。
- 紧凑横屏命令区高度从约 114 降为 54，释放约 60pt 战场空间。
- 页签切换只重建 HUD，不清理选中单位、生产队列、建筑 / 支援 / 集结 / AMOV pending 状态。
- 当前页显示金色描边；隐藏页含 armed 动作时继续显示金色 glow，避免目标模式丢失感。
- `SKRM` 重开回到 `TACT`。
- 云端同一 run 分别启动并截图五页，每次解析 PID 并检查存活。

## 动作分配

- `TACT`：ARMY、G1、G2、HOLD、AMOV、RLY、HQ。
- `BUILD`：HMV、AA、TANK、ART、MECH、BASE。
- `AIR`：HELI、JET。
- `SEA`：SHIP、SUB、CV。
- `SUP`：SCAN、REPR、AIRS、BARR、AI、SKRM。

## 非目标

- 不删除、重命名或改变任何 `HudAction` 的执行语义、subtitle、费用、冷却、来源、pending 优先级或触摸世界命令。
- 不修改战斗、AI、生产、迷雾、地图、模型、经济、任务或胜负。
- 不新增外部依赖、音效、手势或 XCTest target。
- 不运行本地测试、build、模拟器、探针或本地静态检查。
- 不提交用户 `project.pbxproj` 团队号改动。

## 实现步骤

1. 新增 `HudPage`，集中定义标题、副标题和动作清单，确保 24 个 `HudAction` 各出现一次。
2. `layoutHUD()` 常驻绘制五个页签和当前页动作，使用单排稳定尺寸并移除旧两排分配。
3. 新增页签 frame / shape、点击路由、活动页样式与跨页 armed glow。
4. `didMove` 仅在 CI capture 下读取 `DESERT_CI_HUD_PAGE`；普通启动默认 `TACT`。
5. workflow 对 tactical / build / air / naval / support 各启动一次、截图一次、检查 PID 一次，并把五图写入 manifest。
6. 更新 README、flow、flowchart、test 和提示词。
7. commit `v4.79: 重构五页单排战术HUD`，直推 `origin/main`，只做云端验证。

## 验收标准

- manifest SHA、run、attempt、version 与最新 main 一致；static checks、lint、generic build、simulator launch 成功，JUnit 0 失败。
- 五张截图都存在且不是黑屏 / 白屏；各自高亮正确页签，并显示该页完整动作集。
- 任一截图中页签和动作均为单排、无横向裁切、无按钮文字溢出，底部战场可见面积明显大于 v4.78 双排截图。
- tactical 页仍显示 v4.78 空军、防空圈、顶标和 `AA THR2 S1 M1 C3/3`；其他页不破坏这些战场证据。
- 五次启动 PID 均在等待后存活，App 日志无启动崩溃、越界或未捕获异常。
