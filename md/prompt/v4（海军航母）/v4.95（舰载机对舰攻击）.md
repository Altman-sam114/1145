# v4.95 舰载机对舰攻击

## 版本号

`v4.95`

## 版本分配依据

- `update_log.md` 与 `md/prompt/v4（海军航母）/` 的最新正式版本为 `v4.94`。
- 人工要求先继续改善 UI 与战斗细节；v4.94 已完成岸防炮与海岸交战，但 Carrier 直接攻击仍只有单机直线淡出和一枚通用投射物。本轮只细化这条既有攻击表现及其云端取证，因此递增为 `v4.95`。

## 背景

- 参考对象固定为百度百科 lemma id `4042982` 对应的 Noble Master Games《沙漠风暴 Desert Stormfront》，不是其他同名作品。
- 已核对 Noble Master 官方 press kit `https://www.noblemaster.com/press/sheet.php?p=desert_stormfront`、官方 trailer `https://www.youtube.com/watch?v=7oZC6DpIUhQ` 和官方截图目录。官方作品以俯视战场、明确单位轮廓、短促攻击反馈和紧凑触控信息保持高密度战斗可读性；本项目只借鉴这些原则，不复制原作素材或 UI。
- 当前 `fire(attacker:target:)` 对 Carrier 调用 `launchCarrierWing(...)`，后者只显示甲板 pulse、单架 Fighter 直线飞行和通用投射物；伤害、冷却、目标合法性、飘字和水面命中仍由统一 `fire` 链路处理。

## 目标

1. 将 Carrier strike 改为三机错列编队，使用可读的弧形进场 / 俯冲航线、机体朝向和分层尾迹；普通效果在短时间内完整清理。
2. 从编队前缘释放两枚可辨识反舰弹体，提供弹体、尾焰、烟迹和目标端舰体闪光；与现有水面命中组合形成明确的对舰攻击过程。
3. 保持每次 `fire` 只扣一次 HP；新增 helper 全部为视觉，不新增 splash、延迟伤害或二次结算。
4. Carrier strike 来源和目标都继续服从玩家已知 / 可见边界，未知 Red Carrier 不得通过甲板 pulse、航线、编队或弹体泄露位置。
5. 新增 capture-only `carrier-strike` 场景：单选 Blue Carrier 锁定约 42% HP Red Battleship，冻结三机编队、弧形航线、双反舰弹体、舰体命中、水面反馈、目标 HP / 距离 / reload、护航圈、小地图和单排命令条。
6. GitHub Actions 从十四次扩展为十五次独立启动，新增 `simulator-carrier-strike.png`。

## 非目标

- 不改变 Carrier damage、range、cooldown、cost、build time、vision、生产、rally、guard wing、AI、target acquisition 或 `canAttack(...)`。
- 不增加真实舰载机实体、弹药库存、返航、巡逻、CAP、拦截、航母升级或独立状态机。
- 不改变 Battleship / Submarine / Coastal Battery 模型或既有舰炮视觉。
- 不改变迷雾、声呐、潜艇隐身、任务、经济、建造或胜负。
- 不新增外部依赖、图片素材或音频。
- 不运行本地 Xcode build、本地 simulator 或本地玩法探针。

## 当前架构依据

- `fire(attacker:target:)` 是玩家与 AI 共用的伤害结算入口；Carrier 分支只能派发视觉，`target.hp` 写入必须仍只有统一的一次。
- `launchCarrierWing(...)`、`carrierLaunchCraft(...)`、`showNavalWaterImpact(...)` 和 `showNavalHullStrike(...)` 已位于 `effectsLayer` 视觉链路，可在不新增 gameplay state 的前提下组合。
- `prepareCICaptureScene()` 已按 `DESERT_CI_COMMAND_MARKER` 分流；新 marker 必须完全受 `DESERT_CI_CAPTURE_MODE=1` 限制。
- 单选目标态势、护航范围圈、迷雾和 HUD 都复用现有 selection-derived 状态，不新增平行 UI。

## 实现步骤

1. 确认 `main`、`HEAD == origin/main`，保留人工未暂存的 `DesertFrontline.xcodeproj/project.pbxproj` `DEVELOPMENT_TEAM` 改动。
2. 将 Carrier fire 视觉限制在 `attackerKnownToPlayer`，避免未知 Red 来源通过新效果泄露。
3. 扩展 `launchCarrierWing(...)`：构造三条错列二次曲线，三机按轻微时间差进入，沿路径自动朝向；叠加低透明烟迹与阵营色尾迹。
4. 新增双反舰弹体 helper：从编队释放点飞向目标两侧，包含烟迹、亮色弹道、弹体、尾焰，并在目标端组合舰体闪光；普通节点约 1 秒内清理，persistent 只冻结 capture 证据。
5. 新增 `prepareCICarrierStrikeCaptureScene()`，写入 Carrier / Battleship 位置、目标、42% HP、reload、selection、fog 和 persistent 视觉。
6. workflow 清理列表、launch 列表和 manifest 增加 `simulator-carrier-strike.png`，总计十五次 PID / PNG 检查。
7. 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 与 `md/test/test.md`；Agent C 验收通过后再更新 `update_log.md`。
