# v4.14（海岸资产职责面板）

## 版本号

v4.14

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.13「支援按钮资产提示」。
- v4 阶段持续围绕海军、航母、海岸资产、任务面板和 HUD 可读性做小步增强。
- 本轮是一个范围很小的选择信息面板增量：让玩家单选 Shipyard / Sonar Buoy / Coastal Battery 时，能直接看到该海岸资产的职责摘要，以及它是否已经按 `Secure Coast` 任务口径计入 operational coastal asset。

## 背景

当前海岸资产相关事实：

- `Shipyard` 是海岸生产建筑，`EntityKind.canProduce(_:)` 允许它生产 Battleship / Submarine / Carrier。
- `Sonar Buoy` 是低成本、脆弱的海岸侦测结构，完工后提供有限普通视野和较大 sonar 检测范围；它不攻击、不生产、不赚钱，也不作为 `SCAN` 资产。
- `Coastal Battery` 是海岸防御结构，完工后只攻击可见水面海军目标；它不攻击潜艇，也不提供 sonar。
- `Secure Coast` 任务阶段通过 `coastalAssetCount(for:) >= 2` 完成，计数来自同一口径的 `coastalAssetBreakdown(for:)`：指定 faction、存活、已完工 `isOperational` 的 Shipyard / Sonar Buoy / Coastal Battery。
- `missionDetail(for: .secureCoast)` 已显示 `SY` / `SON` / `CB` 分项摘要。
- 单选信息面板当前由 `singleSelectionInfo(for:)` 生成 4 行，结构状态由 `structureStatusLine(for:)` 补充：
  - 未完工结构显示 `Build Ns left` 和 `Offline until operational`。
  - `Sonar Buoy` 当前显示 `Sonar <range>  No SCAN`，并通过 `selectionCapabilityInfoLine(for:)` 在攻击行追加 active sonar 信息。
  - `Coastal Battery` 当前显示 `Naval defense  No sonar`。
  - `Shipyard` 作为 factory 显示队列和 rally 状态，但单选面板没有明确写出它是 naval production coastal asset。

人工指定本轮目标：单选 Shipyard / Sonar Buoy / Coastal Battery 时，在选择信息面板显示职责摘要和 `Secure Coast` operational coastal asset 计入状态；不改建造、任务条件、战斗、sonar、支援、AI、迷雾。

## 目标

1. 增强单选海岸资产的选择信息面板。
   - 只针对玩家或任意可选实体中的 `.shipyard`、`.sonarBuoy`、`.coastalBattery` 单选信息。
   - 面板需要保留当前 HP、攻击 / 射程 / 视野、domain、队列 / rally、ASW / sonar 等既有有用信息。
   - 增加或替换一行短文案，让玩家能直接看出该建筑职责：
     - Shipyard：建议 `Naval production asset` 或 `Builds BB/SUB/CV`。
     - Sonar Buoy：建议 `Coastal sonar asset` 或保留并强化 `Sonar <range>  No SCAN`。
     - Coastal Battery：建议 `Coastal naval defense` 或保留并强化 `Naval defense  No sonar`。
   - 增加或替换一行短文案，让玩家能看出该单体是否按 `Secure Coast` 任务口径计入 operational coastal asset：
     - 已完工且存活时建议显示 `Secure Coast: counted`。
     - 未完工时建议显示 `Secure Coast: pending` 或 `Secure Coast: build pending`。
   - 文案必须短，适配 selection info panel 的 4 行限制；不要新增面板行数、弹窗或教程文本。
2. 复用任务计入口径。
   - `counted` 的判断必须与 `Secure Coast` 任务一致：Shipyard / Sonar Buoy / Coastal Battery、存活、`isOperational`。
   - 可用一个小 helper 表达，例如：

```swift
private func isCoastalMissionAsset(_ entity: GameEntity) -> Bool
private func coastalAssetRoleLine(for entity: GameEntity) -> String?
private func coastalAssetMissionLine(for entity: GameEntity) -> String?
```

   - 如果需要判断对应 faction 的总数或分项，必须复用 `coastalAssetBreakdown(for:)` / `coastalAssetCount(for:)`，不得新增并行统计规则。
   - 本轮单选面板的计入状态只描述该实体自身是否 counted / pending，不要求显示全局 `SY/SON/CB` 总数。
3. 保持现有功能语义不变。
   - Shipyard 的生产来源、队列、rally、出兵和 AI 使用不变。
   - Sonar Buoy 的普通视野、sonar range、潜艇检测、`SCAN` 资产排除规则不变。
   - Coastal Battery 的攻击目标、射程、伤害、冷却、目标搜索和潜艇不可攻击规则不变。
   - `Secure Coast` 完成条件仍是任意 2 个 operational coastal assets。
4. 文档同步。
   - `README.md`：同步选择面板会在单选 Shipyard / Sonar Buoy / Coastal Battery 时报告海岸资产职责和 `Secure Coast` counted / pending 状态。
   - `md/flow/flow.md`：在 `HUD / 小地图` 或 `MissionStage` 相关段落同步这一选择面板真实行为，并强调只是 selection info 文案，不改变任务计数或建筑能力。
   - `md/flow/flowchart.md`：如 HUD 节点已有选择信息描述，可短语同步为“海岸资产职责/计入状态”或等价说法；如果图已经足够概括且不改，Agent B 必须说明原因。
5. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

关键源码事实：

- `EntityKind.displayName` / `shortCode` 中三类海岸资产分别是 `Shipyard` / `SY`、`Sonar Buoy` / `SON`、`Coastal Battery` / `CB`。
- `EntityKind.isFactory` 中 Shipyard 是 factory；`supportsRallyPoint` 中 Shipyard 支持 rally。
- `EntityKind.hasSonar` 中 Sonar Buoy 是 sonar sensor；Coastal Battery 不是 sonar sensor。
- `EntityKind.canProduce(_:)` 中 Shipyard 生产 Battleship / Submarine / Carrier。
- `EntityKind.canAttack(_:)` 中 Coastal Battery 只攻击 naval domain，且由于 submarine 在前置分支中只允许 submarine / battleship / helicopter / fighter / carrier 攻击，因此 Coastal Battery 不攻击潜艇。
- `GameEntity.isOperational` 是结构完工边界；未完工建筑可被攻击，但不得生产、赚钱、提供视野、支援资产、基地覆盖、任务计数或开火。
- `singleSelectionInfo(for:)` 生成 selection info panel 4 行，并对结构调用 `structureStatusLine(for:)`。
- `structureStatusLine(for:)` 当前对 Radar / Sonar / Guard / SAM / Coastal Battery / factory queue 有专门文案；Shipyard 作为 factory 会进入队列 / rally 文案。
- `selectionCapabilityInfoLine(for:)` 会把 ASW attack / sonar 信息追加到攻击行；本轮不得破坏 Sonar Buoy 的 sonar 行和 Carrier / naval ASW 信息。
- `missionDetail(for: .secureCoast)` 复用 `coastalAssetBreakdown(for: .player)` 显示 `SY` / `SON` / `CB` 摘要。
- `isMissionStageComplete(.secureCoast)` 使用 `coastalAssetCount(for: .player) >= 2`。
- `coastalAssetCount(for:)` 复用 `coastalAssetBreakdown(for:)`，而 breakdown 只统计指定 faction、存活、operational 的三类海岸资产。

推荐实现方向：

- 增加小 helper 判断三类海岸资产，避免到处写 `kind == .shipyard || kind == .sonarBuoy || kind == .coastalBattery`。例如：


```swift
private func coastalAssetRoleLine(for entity: GameEntity) -> String? {
    switch entity.kind {
    case .shipyard:
        return "Naval production asset"
    case .sonarBuoy:
        return "Coastal sonar asset"
    case .coastalBattery:
        return "Coastal naval defense"
    default:
        return nil
    }
}
```

- 增加计入状态文案 helper，例如：

```swift
private func coastalAssetMissionLine(for entity: GameEntity) -> String? {
    guard isCoastalAssetKind(entity.kind) else { return nil }
    return entity.isOperational ? "Secure Coast: counted" : "Secure Coast: pending"
}
```

  如果希望更严格表达任务口径，也可判断 `entity.isAlive && entity.isOperational`；在 `singleSelectionInfo` 里传入的一般是存活 selected entity，但 helper 独立判断更稳。

## 风险和禁止项

风险：

- Selection info panel 只有 4 行，Shipyard 同时需要 HP、combat stats、role、queue/rally、Secure Coast 状态，文案过长容易截断。优先用短句组合，不新增行数。
- Sonar Buoy 的职责文案如果覆盖了现有 `Sonar <range>  No SCAN`，可能让玩家误以为它可作为 `SCAN` 资产；必须保留 No SCAN 或等价语义。

- 为了复用 helper 改动 `coastalAssetBreakdown(for:)` 时可能误改任务条件；除非必要，不要动任务计数 helper。

- 禁止改变 Shipyard / Sonar Buoy / Coastal Battery 的玩法能力、数值、AI 使用、建造限制或任务完成条件。
- 禁止把 Sonar Buoy 变成 `SCAN` 支援资产。

- 禁止让未完工建筑计入 Secure Coast。
