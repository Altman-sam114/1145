# v4.97 战斗机双弹对地对海打击

## 版本号与依据

`v4.97`。当前正式版本与最新提示词为 `v4.96`；人工要求先改善 UI 和战斗细节，本轮只补 Fighter 对水面舰艇 / 建筑目标的专用打击反馈，因此顺序递增。

## 背景

- 参考对象固定为百度百科 lemma id `4042982` 对应的 Noble Master Games《沙漠风暴 Desert Stormfront》。已核对官方 press kit、官方 trailer 和 `screen_dsf_v02_04.png`；图中飞机以两枚分离弹体和短烟迹攻击海岸目标。本项目只借鉴俯视轮廓、构图和反馈原则，不复制素材。
- Fighter 模型已有翼下挂点和可见导弹，但实战对空、对海、对建筑都复用单枚直线 guided trail，水面 / 建筑目标缺少专用命中反馈。

## 目标

1. Fighter 攻击玩家可见水面舰艇或建筑时，显示两个翼下发射点、两枚约 0.055 秒错发的可见弹体、短白烟迹、阵营色亮芯和紧凑发射闪光。
2. 两条弹道使用轻微分离的二次曲线，保持目标轮廓、HP、距离和 reload 可读。
3. 水面舰艇使用紧凑舰体火花并辅以小型近舷水面冲击；建筑使用小型闪光、尘环和碎屑，不调用大号击毁爆炸。
4. 每次 `fire` 仍只扣一次 HP；所有新增 helper 只负责视觉。
5. 新增 capture-only `fighter-strike` 场景，单选 Blue Fighter 锁定约 48% HP Red Coastal Battery，冻结双曲线、双弹体、发射闪光、结构命中和伤害飘字。
6. GitHub Actions 扩展为十七次独立启动，新增 `simulator-fighter-strike.png`。

## 非目标

- 不改 Fighter 数值、成本、生产、AI、目标合法性或空战 / 反潜表现。
- 不新增第二次伤害、splash、碰撞 projectile、弹药库存或制导状态。
- 不改 Carrier、AA Truck、Helicopter、迷雾、声呐、经济、任务或胜负。
- 不运行本地 Xcode build、simulator 或本地玩法探针。

## 架构与实现步骤

1. 保留人工 `project.pbxproj` Team ID 改动，不纳入提交。
2. 在 `fire(attacker:target:)` 内只对已知 Fighter 来源且目标为非 Submarine 水面舰艇或建筑时派发 `showFighterSurfaceStrike(...)`；air 与 submarine 保持原 `showProjectile(...)`。
3. `showFighterSurfaceStrike(...)` 复用 `quadraticPoint(...)` / `quadraticTangent(...)` 计算冻结弹体位置与朝向；普通节点在约 0.7 秒内清理，`persistent` 仅供 capture。
4. 新增紧凑 `showFighterGroundImpact(...)`；水面命中复用较小舰体 / 水面反馈。helper 不写 HP、不调用 `fire`、不创建 gameplay entity。
5. 在 camera focus 早退前分流 `fighter-strike`，编排 Blue Fighter、Red Coastal Battery、选择、目标、HP、reload、fog 和 persistent 视觉。
6. workflow 的清理、launch 和 manifest 列表加入第十七张截图。
7. 同步 README、flow、flowchart 与 test；Agent C 云端 artifact 验收通过后再更新 `update_log.md`。
