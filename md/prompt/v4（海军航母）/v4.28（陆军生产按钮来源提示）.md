# v4.28（陆军生产按钮来源提示）

## 版本号

v4.28

## 版本分配依据

- `update_log.md` 当前最新正式玩法版本为 v4.27「多选声呐编队接触数摘要」。
- v4.17 已为 `HELI` / `JET` 增加 `AF` / `CV` 生产来源提示。
- v4.21 已为 `SHIP` / `SUB` / `CV` 增加 `SY` 生产来源提示。
- 陆军按钮 `HMV` / `AA` / `TANK` / `ART` / `MECH` 仍只显示价格，本轮补齐同一 HUD 可读性模式，显示 War Factory 来源或缺失提示。

## 背景

当前 `DesertFrontline/GameScene.swift` 中：

- `HudAction.buildKind` 将陆军生产按钮映射到 `.humvee`、`.aaTruck`、`.tank`、`.artillery`、`.mechanic`。
- `EntityKind.shortCode` 中 War Factory 的短码是 `WF`。
- `productionButtonSubtitle(for:)` 已对空军和海军调用 `productionSource(for:faction:)`，显示 `AF` / `CV` / `SY` 来源或 `need AF/CV` / `need SY`。
- `queueBuild(kind:faction:showFeedback:)` 和 `canQueueBuild(kind:faction:)` 已复用 `productionSource(for:faction:)` 判断真实生产来源。
- `productionSource(for:faction:)` 会过滤阵营、存活、`isOperational`、`canProduce(kind)`，玩家侧优先当前选中的可生产来源，再按队列深度和实体 ID 选择来源。

本轮只让陆军按钮 subtitle 使用同一只读来源口径，减少玩家在 War Factory 缺失、施工中或未选中时的信息盲区。

## 目标

1. 增强 `HMV` / `AA` / `TANK` / `ART` / `MECH` 生产按钮 subtitle。
   - 有 operational War Factory 来源时，显示 `WF $<cost>`。
   - 没有可用 War Factory 来源时，显示 `need WF`。
2. 必须复用 `productionSource(for:faction:)`。
   - 不在 HUD helper 中重写 War Factory 搜索规则。
   - 不把未完工、死亡、敌方、中立或不能生产该单位的实体计入来源。
   - 保持选中 War Factory 时的玩家优先来源逻辑与实际 `queueBuild` 一致。
3. 保持现有空军和海军按钮行为不回归。
   - `HELI` / `JET` 继续显示 `AF $cost`、`CV $cost` 或 `need AF/CV`。
   - `SHIP` / `SUB` / `CV` 继续显示 `SY $cost` 或 `need SY`。
4. 保持生产执行链路完全不变。
   - 不改变 `queueBuild(kind:faction:showFeedback:)`。
   - 不改变 `productionSource(for:faction:)`。
   - 不改变 `canQueueBuild(kind:faction:)`。
   - 不改变 `BuildOrder`、费用、建造时间、资金检查、出兵位置、队列排序、ready pulse 或 rally point。
5. 同步 README 和核心流程文档中的当前真实行为。

2. 修改 `DesertFrontline/GameScene.swift`。

推荐实现方向：

```swift
private func productionButtonSubtitle(for kind: EntityKind) -> String {
    switch kind {
    case .aaTruck, .humvee, .tank, .artillery, .mechanic:
        guard let source = productionSource(for: kind, faction: .player) else {
            return "need WF"
        }
        return "\(source.kind.shortCode) $\(kind.cost)"
    case .helicopter, .fighter:
        guard let source = productionSource(for: kind, faction: .player) else {
            return "need AF/CV"
        }
        return "\(source.kind.shortCode) $\(kind.cost)"
    case .battleship, .submarine, .carrier:
        guard let source = productionSource(for: kind, faction: .player) else {
            return "need SY"
        }
        return "\(source.kind.shortCode) $\(kind.cost)"
    default:
        return "$\(kind.cost)"
    }
}
```

约束：

- 资金不足但有 operational War Factory 时仍显示 `WF $cost`，点击后由 `queueBuild` 显示资金不足。
- 不在 helper 中改变 `selectedIDs`、队列、资金、实体或 pending 状态。
- 不改变空军、海军、support、control group、`BASE`、`RLY`、`AMOV` 或其它 subtitle 逻辑。
