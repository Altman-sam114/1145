# v4.57 / AI 航母警戒翼队保留护航

## 版本号

v4.57

## 版本分配依据

上一轮 v4.56 已补齐玩家选中 HOLD Carrier 时的警戒范围可见性。当前 Red AI 已能让空闲 Carrier 绑定附近空闲 Helicopter / Fighter 作为 guard wing，但同一指挥周期后续 routine attack-move 波次仍会把这些 `destination` / `attackTarget` / `attackMoveDestination` 为空的飞机当作普通空闲攻击单位选走。本轮让已绑定 Red Carrier guard wing 的 HEL/JET 保留护航职责，属于 v4 海军 / 航母 AI 行为的小步稳定性增量。

## 背景

当前相关事实：

- `updateAI(dt:)` 先调用 `updateEnemyCarrierGuardWings(enemyUnits)`，再在满足攻击条件时调用 `commandEnemyAttackers(enemyUnits)`。
- `updateEnemyCarrierGuardWings(_:)` 会把空闲 Red Carrier 附近的空闲 Helicopter / Fighter 绑定为 guard wing，写入 `guardAnchorCarrierID`，并清空 target / destination / attack-move。
- `commandEnemyAttackers(_:)` 的 `idleAttackers` 只检查 target / destination / attack-move、capture reservation、retreat、damage 和 operational 状态。
- 因为 bound guard wing 仍然没有 target / destination / attack-move，它们会进入 `enemyProvisionalAssaultWave(...)`，再被 `issueFormationMove(... attackMove: true)` 清掉 HOLD / guard anchor 状态。

## 目标

- Red routine attack-move 波次不再把当前已绑定 Carrier guard wing 的 Red Helicopter / Fighter 作为普通 assault candidate。
- 已绑定 guard wing 继续依赖现有 Carrier HOLD anchor、station keeping 和近域已知威胁优先链路护航。
- Red Carrier 自身、未绑定 HEL/JET、其他普通攻击单位仍按原 routine wave 规则参与进攻。
- 本轮只改变 AI 普通进攻波候选筛选，不新增 CAP、巡逻、截击或独立护航状态机。

## 非目标

- 不改变玩家 HOLD / AMOV / move / attack 命令。
- 不改变 Carrier guard wing 的绑定数量、半径、候选排序、HUD 文案、范围圈、释放反馈或 composition 统计。
- 不改变 `carrierGuardPriorityTarget(for:)`、station keeping、combat、fog、潜艇侦测、生产、支援、任务、胜负或 SKRM 重置。
- 不改变 Red Carrier 是否能作为高价值海军单位加入主攻波次的现有护航门槛。
- 不新增外部依赖、素材、测试 target 或大体积 artifact。

## 当前架构依据

- Red AI 的 guard wing 绑定和 routine assault wave 都集中在 `GameScene.swift`。
- `carrierGuardAnchor(for:)` 已能根据 `guardAnchorCarrierID` 找到有效 anchor Carrier。
- `issueFormationMove(...)` 会清理参与编队单位的 HOLD / guard 状态，因此最小安全改动是在 `commandEnemyAttackers(_:)` 构造 `idleAttackers` 时排除已绑定的 Red guard wing aircraft。
- 排除只应命中 Red Helicopter / Fighter 且 `carrierGuardAnchor(for:) != nil` 的单位，避免影响玩家单位或未绑定飞机。

## 实现步骤

1. 在 `GameScene.swift` 增加小 helper，例如 `isEnemyCarrierGuardWingReservedForAnchor(_:)`：
   - 要求 `unit.faction == .enemy`。
   - 要求 `unit.kind == .helicopter || unit.kind == .fighter`。
   - 要求 `carrierGuardAnchor(for: unit) != nil`。
2. 在 `commandEnemyAttackers(_:)` 的 `idleAttackers` filter 中加入排除条件：
   - `!isEnemyCarrierGuardWingReservedForAnchor(unit)`。
3. 同步 README：
   - Current Features 的 AI 描述改为：Red routine attack waves 会保留当前已绑定 Carrier guard HEL/JET，不把它们立即作为普通进攻波成员抽走。
4. 同步 `md/flow/flow.md`：
   - AI 进攻段落删除“后续 routine wave 仍可接管这些单位”的旧事实。
   - 改为：已绑定 Red Carrier guard wing 会被排除出 routine assault candidate，直到 guard anchor 被释放或其他既有逻辑使其不再绑定。
5. `md/flow/flowchart.md` 如无相关细节可不改。
