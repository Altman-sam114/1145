# v4.7（终局HQ任务情报）

## 版本号

v4.7

## 版本分配依据

- `update_log.md` 与 `md/prompt/` 当前最新正式玩法版本为 v4.6「摧毁生产任务奖励」。
- v4 阶段已经围绕海军、航母、海岸争夺和任务链做了多轮小步增强。v4.3 到 v4.6 补齐了海岸据点、混合军种、破生产和对应奖励。
- 本轮继续沿任务链做一个终局 HUD 小增强：`Destroy Red HQ` 阶段显示玩家已知 Red HQ 的 HP 情报，帮助玩家组织最后攻势，但不新增新单位、AI 状态、战役系统或完整航母 CAP / 巡逻。

## 背景

当前 `Destroy Red HQ` 任务详情固定显示 `Destroy the Red Command HQ.`。玩家进入终局后无法从任务面板判断 Red HQ 是否已经被侦察到、还剩多少 HP，只能依赖地图选择或目视反馈。

项目已有 `isKnownToFaction(_:observer:)` 作为迷雾合法性边界，敌方实体只有在玩家可见或支援侦察覆盖时才可被玩家可靠感知。本轮只让任务 HUD 复用该边界：已知 Red HQ 时显示 HP；未知时提示先侦察或推进到 HQ。

## 目标

1. 为 `Destroy Red HQ` 阶段增加玩家合法情报文案。
   - 如果 Red HQ 存活且 `isKnownToFaction(redHQ, observer: .player)` 为 true，显示 Red HQ HP。
   - 文案保持短句，例如 `Red HQ HP 1234/1800. Finish the assault.`
   - HP 使用 `Int(redHQ.hp)` 和 `Int(redHQ.kind.maxHP)`。
2. Red HQ 未知时不泄露隐藏信息。
   - 如果 Red HQ 存活但不被玩家已知，显示侦察/推进提示，例如 `Locate Red HQ with scouts or SCAN, then finish it.`
   - 不显示隐藏 HQ 的 HP、位置、距离或状态。
3. Red HQ 已不存在时保持终局语义。
   - 如果没有存活 Red HQ，详情可显示 `Red Command HQ destroyed.`
   - 胜利状态仍由 `updateVictoryState()` 和现有 `victoryState` 文案负责。
4. 文档同步。
   - `README.md`：任务面板功能描述中说明终局阶段会在合法侦察到 Red HQ 时显示 HP 情报。
   - `md/flow/flow.md`：任务系统说明同步 `Destroy Red HQ` 详情复用 `isKnownToFaction`，不绕过迷雾。
   - `md/flow/flowchart.md`：任务节点可简短同步 HQ 情报。
5. `update_log.md` 由 Agent C 在云端 artifact 验收通过后追加正式记录。

## 非目标

- 不改变 `Destroy Red HQ` 完成条件。
- 不改变 `updateVictoryState()`、胜利 / 失败触发时机或 HQ HP / cost / damage / range。
- 不显示隐藏 Red HQ 的位置、距离、HP、可攻击状态或路径。
- 不改变 `isKnownToFaction(...)`、`isVisible(point:)`、`visibleTiles`、`exploredTiles`、`supportRevealTiles`、潜艇侦测或支援侦察逻辑。
- 不新增 HUD 按钮、小地图标记、地图 ping、任务奖励、教程弹窗、战役系统、科技树或新地图。
- 不改变 AI、生产队列、单位数值、支援技能、路径、伤害 / XP 结算或 Xcode/workflow 配置。
- 不更新 `md/test/test.md`。
- 不引入 `smalldata_test`、`develop`、`codeb/...`、候选分支或 PR 流。
