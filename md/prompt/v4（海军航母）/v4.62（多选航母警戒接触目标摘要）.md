# v4.62 / 多选航母警戒接触目标摘要

## 版本号

v4.62

## 版本分配依据

v4.61 已补齐 Red Carrier battle group 携带站位中的 guard wing。当前单选 HOLD Carrier 的 `GW` 行已经能显示合法 guard contact 的类型摘要和优先目标短码，例如 `C2 Sea Tgt SUB Eng 1`；但多选 HOLD Carrier 时 `groupCarrierGuardWingSummary(for:)` 只显示 `CV GW x/y ... Cn Eng n`，缺少同一口径的接触类型和优先目标。补齐该多选 HUD 信息能提升航母编队态势可读性，属于 v4 海军航母阶段的小步 HUD 增强。

## 背景

当前相关事实：

- `carrierGuardContactSummary(for:guardWing:)` 用 `isCarrierGuardContact(...)` 统计单选 Carrier 的合法接触数、类型和优先目标短码。
- `groupCarrierGuardWingSummary(for:)` 已聚合选中 HOLD Carrier 的 bound wing 数、需求、HEL/JET 组成、去重 contact 数和 `Eng n`。
- 多选摘要当前不显示 contact 类型或优先目标，信息弱于单选 Carrier。
- `isCarrierGuardContact(...)` 已包含 `isKnownToFaction(...)`、`canAttack(...)`、Carrier 近域半径和 wing HOLD 站位警戒半径，不会绕过迷雾。

## 目标

- 多选 HOLD Carrier 的 `CV GW` 摘要在 `C` 非零时追加接触类型：`Air` / `Sea` / `Sub` / `Ground` / `Mix`。
- 多选 HOLD Carrier 的 `CV GW` 摘要在 `C` 非零且存在优先接触时追加 `Tgt XXX`。
- 继续显示非零 `Eng n`，并保持 `C0` 时不追加类型或目标。
- 接触类型、目标和 `Eng` 均只读显示，不改变攻击目标、移动、AI、迷雾或战斗。

## 非目标

- 不改变单选 Carrier HUD 文案。
- 不改变 selected bound HEL/JET 的 `CV GUARD` 文案。
- 不改变 Carrier guard wing 绑定、站位、优先目标选择、攻击、移动、释放、范围圈或 AI Carrier battle group。
- 不新增完整 CAP、巡逻、截击、战报面板、按钮或外部依赖。

## 当前架构依据

- `groupCarrierGuardWingSummary(for:)` 是多选 HOLD Carrier 摘要的集中入口。
- `carrierGuardContactType(for:)` 已把接触归为 `Air` / `Sea` / `Sub` / `Ground`。
- `carrierGuardTargetPriority(for:target:)` 和单选摘要中的距离 / id tie-breaker 已定义优先接触选择口径。
- `isCarrierGuardContact(...)` 是合法接触边界，必须复用该边界，不读取隐藏单位或写入新状态。

## 实现步骤

1. 修改 `groupCarrierGuardWingSummary(for:)`：
   - 保留现有 bound wing、需求、组成、`C` 和 `Eng` 统计。
   - 在遍历 selected HOLD Carrier 的 guard wing contacts 时，额外聚合 contact type 集合。
   - 按单选 Carrier 相同风格维护一个 preferred target：
     - priority 越小越优先；
     - priority 相同时，距离 anchor Carrier 更近者优先；
     - 距离近似相同时，entity id 更小者优先。
2. 返回字符串时：
   - `contactIDs` 为空：保持 `C0`，不追加类型或目标。
   - `contactIDs` 非空且 type 集合只有一个：追加该类型。
   - `contactIDs` 非空且 type 集合多个：追加 `Mix`。
   - 有 preferred target：追加 `Tgt <shortCode>`。
   - `Eng n` 仍只在非零时追加，并放在目标后面。
3. 不提取大范围新抽象，除非能保持局部清晰；优先小范围编辑。
4. 同步 README：
   - Carrier feature / tactical selection panel 描述中的多选 `CV GW` 文案补充类型和 `Tgt XXX`。
5. 同步 `md/flow/flow.md`：
   - HUD / selection 段写清多选 HOLD Carrier 的 `CV GW` 摘要现在可显示 contact type 和 priority target，仍只读且遵守已知边界。
6. `md/flow/flowchart.md` 当前只概括 HUD，不需要细分本轮文案；可不改。
