# v4.38（航母警戒翼队接触数提示）Agent B 实现提示词

## 1. 版本号

v4.38（航母警戒翼队接触数提示）

## 2. 版本分配依据

- 当前 `md/prompt/v4（海军航母）/` 最新版本为 `v4.37（航母警戒翼队近域威胁优先）.md`，本轮延续 v4 海军航母阶段。
- v4.37 已让绑定到 Carrier HOLD guard wing 的 HEL/JET 优先选择 Carrier 近域内已知、可攻击且仍在自身 HOLD 警戒半径内的威胁。
- 当前玩家只能从单位行动间接判断 guard wing 是否有可处理接触。本轮只补 Carrier 单选 HUD 的只读接触数提示，不改变战斗、AI、迷雾或命令。

## 3. 背景

当前 `GameScene.swift` 中：

- `carrierAirWingLine(for:)` 在 Carrier 未 HOLD 时显示 `Wing x/2 OK` 或 `Wing x/2 Need n`，在 Carrier HOLD 时显示 `Guard wing x/2 OK` 或 `Guard wing x/2 Need n`。
- `nearbyCarrierAirWing(for:)` 统计 Carrier 附近同阵营、存活、operational 的 Helicopter / Fighter。
- `carrierGuardPriorityTarget(for:)` 已按 wing 维度筛选已知、敌对、可攻击、Carrier 近域内且仍满足 wing HOLD 站位警戒半径的目标。
- HUD 选择信息必须遵守战争迷雾边界，不能泄露隐藏潜艇或玩家未知目标。

## 4. 本轮目标

实现一个小范围、可验收的 HUD 反馈增量：

1. Carrier 未 HOLD 时继续显示原有 readiness：`Wing x/2 OK` 或 `Wing x/2 Need n`，不追加 contact 数。
2. Carrier HOLD 时，`Guard wing x/2 ...` 行追加 `Ctc n`，其中 `n` 是当前绑定 guard wing 可处理的去重近域接触数。
3. contact 统计必须满足：
   - 只考虑当前 Carrier 附近仍 HOLD 且 `guardAnchorCarrierID == carrier.id` 的 HEL/JET。
   - 目标是敌方、存活、非中立、对该翼队阵营 `isKnownToFaction(...)` 成立。
   - 至少一个绑定 wing 的 `canAttack(target.kind)` 成立。
   - 目标位于 Carrier `carrierGuardThreatRadius` 内，且对该 wing 仍满足 HOLD 站位警戒半径。
   - 按目标 entity id 去重，避免多个 wing 重复计数。
4. contact 统计应尽量复用或抽取 v4.37 的目标合法性判断，避免 HUD 口径和实际优先目标口径漂移。
5. 只读 HUD 反馈不得写入 `revealedUntil`、`visibleTiles`、`exploredTiles`、攻击目标或命令状态。

## 7. 实现步骤

建议按以下小步实现：

1. 增加小 helper，例如 `boundCarrierGuardWing(for:) -> [GameEntity]`，用于返回当前 Carrier 附近仍 HOLD 且绑定到该 Carrier 的 HEL/JET。
2. 让 `carrierAirWingLine(for:)` 在 Carrier HOLD 时复用该 helper 计算 Guard wing 数量。
3. 增加 `carrierGuardContactCount(for:guardWing:) -> Int` 或等价 helper：
   - 遍历已绑定 wing 和 `entities.values`。
   - 使用与 v4.37 优先目标一致的已知、敌对、可攻击、Carrier 近域、wing HOLD 站位半径约束。
   - 用 `Set<Int>` 去重目标 id。
4. 可选整理：抽出 `isCarrierGuardContact(_:for:carrier:holdPosition:)` 之类 predicate，让 `carrierGuardPriorityTarget(for:)` 与 contact count 共享合法性判断；但不要做大重构。
5. 修改 `carrierAirWingLine(for:)`：
   - 未 HOLD：保持原文案。
   - HOLD：返回 `Guard wing x/2 OK  Ctc n` 或 `Guard wing x/2 Need n  Ctc m`。
6. 更新 `README.md`：说明 Carrier HOLD guard wing 行会显示近域已知可处理 contacts 数。
7. 更新 `md/flow/flow.md`：同步 HUD 段落，说明 contact 数只读、去重、遵守 `isKnownToFaction` / `canAttack` / HOLD 半径，不改变迷雾或命令。
8. 必要时更新 `md/flow/flowchart.md` 的 HUD 节点文字。
