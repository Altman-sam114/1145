# v2.3（引入机动防空车）

## 版本号

v2.3

## 背景

当前空中反制能力已经包含：

- Guard Tower：完工后攻击可见 land / air / structure，是泛用防御结构。
- SAM Site：完工后只攻击可见 air 目标，是普通陆地静态防空结构。
- Fighter：空中单位，可攻击 air / naval / structure。

但玩家和 AI 的推进部队仍缺少可跟随地面集团移动的专门防空单位。v2.3 做一个保守增量：新增 AA Truck / 机动防空车，作为 War Factory 生产的陆地作战单位，只攻击可见空中目标，不攻击地面、海军、潜艇或建筑。这样前线攻防不只依赖固定 SAM Site，AI 混合编队也能更稳定地反制 Helicopter / Fighter。

## 目标

1. 新增 `EntityKind.aaTruck`。
   - Display name：`AA Truck`
   - Short code：`AA`
   - Domain：`.land`
   - 推荐数值：cost `720`，build time `5.5`，HP `170`，speed `82`，range `220`，damage `24`，cooldown `0.82`，vision `7`，footprint `30`
2. AA Truck 由 War Factory 生产。
   - `requiredFactory` 为 `.barracks`（当前项目中 War Factory 使用 `EntityKind.barracks`）。
   - `.barracks.canProduce(...)` 纳入 AA Truck。
   - 新增 HUD 生产按钮，建议 title 为 `AA`，subtitle 继续显示成本。
   - 按钮插入陆军生产区，避免影响支援技能、`BASE`、`RLY`、`SKRM` 等按钮语义。
3. AA Truck 是移动陆地单位。
   - 使用现有陆地移动、路径、道路加速、油田减速、编队、attack-move、hold、选择、控制组、迷雾显示、血条、死亡清理、老兵 XP 和 `SKRM` 重置链路。
   - 不新增结构、施工、支援资产、科技树或资源类型。
4. AA Truck 只攻击可见空中目标。
   - 可攻击 Helicopter / Fighter 等 air 目标。
   - 不攻击 land / naval / submarine / structure。
   - 不提供 sonar，不改变潜艇 `revealedUntil` / 声呐检测链路。
   - 仍通过 `isKnownToFaction(...)` 找目标，不能攻击迷雾中不可见目标。
5. AI 可生产并使用 AA Truck。
   - 将 AA Truck 纳入 AI build pattern，建议 Normal / Hard 至少出现一次；Easy 可以保守加入一次或暂不加入，但必须说明选择。
   - AI 仍通过 `queueBuild(...)` / `productionSource(...)` 排队，缺 War Factory、未完工 War Factory 或资金不足时自然跳过。
   - 新单位使用现有 `tacticalTarget(...)`、`attackDestination(...)`、attack-move 波次和防守拉兵逻辑。
6. 战斗和目标评估同步。
   - `strategicValue(...)` 给 AA Truck 合理值，建议接近 Helicopter / Tank 之间。
   - `formationPriority(...)` 可把 AA Truck 放在 Humvee / Tank 附近，保证混合编队移动稳定。
   - 不需要把 AA Truck 加入 `primaryTarget(...)`，该函数主要用于结构 / 高价值目标优先级。
7. SpriteKit 外观同步。
   - 在 `addLandUnitBody(...)` 中新增 AA Truck 外观，用现有 shape 拼一个轻型车体加防空发射架 / 双联导弹，不新增素材。
   - HUD / 选择面板无需新增专门状态行，基础 stats 和老兵信息应自然显示。
8. README / flow 文档同步。
   - README Current Features 的 Land units 加入 AA Truck。
   - README Current Features 加一句 AA Truck 是 War Factory 生产的机动防空单位，只攻击可见空中目标，不反潜、不打地面 / 海军 / 建筑。
   - README Controls / HUD 生产说明如列出生产按钮，需要加入 `AA`。
   - `md/flow/flow.md` 的 `EntityKind`、经济与生产、移动与战斗、AI、未来扩展点同步当前真实行为。
   - 必要时更新 `md/flow/flowchart.md` 的生产 / 战斗 / AI 节点。
9. 不更新 `update_log.md`。正式 v2.3 记录由 Agent C 在最新 `origin/main` artifact 验收通过后追加。

## 非目标

- 不新增科技树、升级系统、弹药系统、雷达锁定、音效、图片素材或外部依赖。
- 不改变 SAM Site、Guard Tower、Coastal Battery、Radar Outpost、Fighter、Helicopter、Submarine 或 support power 的既有行为。
- 不让 AA Truck 攻击潜艇、提供声呐、触发潜艇暴露或绕过 `isSubmarineDetected(...)`。
- 不让 AA Truck 攻击陆地、海军或建筑目标。
- 不把 AA Truck 作为 `SCAN`、`BARR`、`AIRS` 或 `REPR` 资产。
- 不让 Carrier、Airfield 或 Shipyard 生产 AA Truck；只由 War Factory 生产。
- 不拆分 `GameScene.swift`，不做大规模重构，不做全文件格式化。
- 不修改 Xcode project、CI workflow 或测试规范，除非编译必须且能说明原因。
- 除非人工明确要求，本轮不默认跑本机完整 Xcode build 或模拟器；构建结论以 GitHub Actions artifact 为准。

关键源码事实：

- `EntityKind` 集中定义单位 / 建筑静态属性，新增单位必须补齐 display name、short code、domain、cost、build time、HP、speed、range、damage、cooldown、vision、footprint、`requiredFactory`、`canProduce(...)`、`canAttack(...)`。
- 当前 War Factory 在代码中名为 `.barracks`，HUD title 是 `TANK` / `ART` 等陆军生产按钮。

- `queueBuild(...)`、`productionSource(...)` 和 `canQueueBuild(...)` 是玩家与 AI 共享生产链路。
- `addLandUnitBody(...)` 负责陆地单位 SpriteKit 外观。
- `updateCombat(dt:)`、`nearestTarget(...)`、`tacticalTarget(...)` 和 `fire(attacker:target:)` 是共享战斗链路。
- `isKnownToFaction(...)` 对敌方迷雾和潜艇隐身做合法性判断；新增机动防空不得绕过。
- `hasSonar` / `sonarRange(for:)` 是潜艇检测链路；AA Truck 不得加入。
- `aiBuildPattern()` 决定 AI 常规生产组合；`nextAvailableAIBuildKind(...)` 会跳过暂时不可生产或资金不足的单位。

### 2. 新增 EntityKind

在 `EntityKind` 中加入 `.aaTruck`，并补齐：

- `displayName`
- `shortCode`
- `domain`
- `cost`
- `buildTime`
- `maxHP`
- `speed`
- `attackRange`
- `damage`
- `attackCooldown`
- `visionTiles`
- `footprint`
- `requiredFactory`
- `canProduce(...)`
- `canAttack(...)`
- `strategicValue(...)`

`canAttack(...)` 推荐：

```swift
case .aaTruck:
    return target.domain == .air
```

注意：

- 不要把 `.aaTruck` 加入顶部 `if target == .submarine { ... }` 的反潜列表。
- 不要让 AA Truck 攻击 `target.isStructure`。
- 不要把 AA Truck 加入 `hasSonar` 或 `sonarRange(for:)`。

### 3. 纳入 HUD、生产和视觉

- `addLandUnitBody(...)` 新增 `.aaTruck` 外观。
- `formationPriority(for:)` 把 AA Truck 放到 Humvee / Tank 附近。

### 4. 同步 AI 和战斗目标

- `aiBuildPattern()` 把 `.aaTruck` 加入 Normal / Hard；Easy 可以加入一次作为基础反空，也可以不加入，但必须保持 pattern 不会因缺来源卡死。
- `strategicValue(of:)` 给 `.aaTruck` 合理值。
- 不需要新建特殊目标搜索；AA Truck 应自然通过 `canAttack(...)`、`isKnownToFaction(...)`、`nearestTarget(...)` 和 `tacticalTarget(...)` 找可见 air 目标。
- 不需要修改 `primaryTarget(...)`，除非发现现有目标排序漏掉了移动高价值目标；若修改必须说明理由。
