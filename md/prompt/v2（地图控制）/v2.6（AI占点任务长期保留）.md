# v2.6（AI占点任务长期保留）

## 版本号

v2.6

## 版本分配依据

- 人工未指定本轮版本；`update_log.md` 和 `md/prompt/` 当前最新正式玩法版本为 v2.5“AI 空军威胁防空响应”。
- v2.5 已通过 `origin/main` GitHub Actions artifact 验收，并追加正式记录。
- v1.9 已让 AI 占点 runner 在移动途中不被 attack-move 波次抢走，但遗留问题是单位到达占领半径、`destination` 清空后，后续 AI 周期仍可能把正在占领的 runner 改派到主攻波次。本轮补齐该小闭环，因此递增为 v2.6。

## 背景

当前 AI 会从空闲 Red 陆地单位中分配油井和旗点 runner，并且主攻波次会跳过已有普通 `destination` 的单位。这个策略保护了“正在移动去占点”的 runner，但单位到达目标附近后，移动系统会清空 `destination`。如果占领进度尚未完成，下一次 `commandEnemyAttackers(...)` 仍可能把这个 runner 视为 idle attacker，导致油井或旗点占领反复中断。

v2.6 做一个保守增量：AI 自己分配的占油 / 占旗任务会建立短期 reservation。只要目标仍未归 Red、单位仍有效，runner 即使已经到达占领半径且 `destination == nil`，也不会被常规占点再分配或主攻 attack-move 波次抽走。

## 目标

1. 新增 AI 占点 reservation 状态。
   - 保存 Red runner ID 与目标关系。
   - 目标至少区分 Oil Derrick 和 Control Point。
   - Reservation 只服务 AI 内部占点调度，不暴露 HUD，不影响玩家命令。
2. 分配占油 / 占旗 runner 时登记 reservation。
   - `updateAI(dt:)` 现有 oil runner 和 flag runner 分配成功后，记录该单位对应目标。
   - 同一轮仍不得把同一个 runner 同时分配给油井和旗点。
3. 主攻波次跳过已保留 runner。
   - `isAvailableEnemyCaptureUnit(...)` 应排除已有有效 reservation 的单位，避免同一单位被重新挑成新 runner。
   - `commandEnemyAttackers(...)` 的 idle attacker 筛选应排除已有有效 reservation 的单位，即使其 `destination == nil`。
4. Reservation 自动维护和清理。
   - 目标已归 Red、目标消失、单位死亡、单位不再是 Red、单位不再是可占点陆地单位时清理。
   - `cullDestroyedEntities()` 清理死亡单位时也应移除对应 reservation。
   - `restartSkirmish()` / `SKRM` 必须清空 reservation。
   - 若单位仍有 reservation 但尚未进入占领半径且没有普通 `destination`、attack target 或 attack-move 意图，可重新给它设置到目标附近的 destination，避免路径结束在占领半径外。
5. 保持既有占领规则。
   - 不改变油井 / 旗点占领半径、占领速度、进度条、收入、视野、build coverage 或任务阶段。
   - 不改变玩家占领、玩家移动 / attack-move / hold 命令。
   - 不新增单位、建筑、HUD、支援技能、资源或外部依赖。
6. 保持紧急防守语义。
   - 本轮目标是防止常规主攻波次抢走 runner；不需要阻止基地遇袭时 `triggerEnemyDefenseIfNeeded(...)` 对附近防守单位的紧急反应。
7. 文档同步。
   - README Enemy AI bullet 从“moving runners kept out of attack-move waves”扩展为“capture runners are reserved until oil / flag capture completes”这类当前真实行为。
   - `md/flow/flow.md` AI 小节同步 reservation 行为、清理条件和 `SKRM` 重置。
   - `md/flow/flowchart.md` AI 节点可加入“长期保留占点队”短语。
8. 不更新 `update_log.md`。正式 v2.6 记录由 Agent C 在最新 `origin/main` artifact 验收通过后追加。

### 2. 新增 reservation 模型和状态

建议增加一个小型私有 enum：

```swift
private enum EnemyCaptureReservation: Hashable {
    case oilDerrick(Int)
    case controlPoint(Int)
}
```

在 `GameScene` 中新增：

```swift
private var enemyCaptureReservations: [Int: EnemyCaptureReservation] = [:]
```

key 是 runner entity ID。

### 3. 新增 helper

建议新增：

- `reserveEnemyCapture(unit:target:)`
- `pruneEnemyCaptureReservations()`
- `maintainEnemyCaptureReservations()`
- `isEnemyCaptureReserved(_ unit: GameEntity) -> Bool`
- `enemyCaptureTargetPosition(for:) -> CGPoint?`
- `enemyCaptureTargetNeedsCapture(_:) -> Bool`
- `enemyCaptureRange(for:) -> CGFloat`
- `isEnemyCaptureUnit(_ unit: GameEntity) -> Bool`

行为要求：

- Reservation 有效条件应至少包含：unit 存活、faction 为 `.enemy`、非结构、land、operational、目标仍未归 `.enemy`。
- 若 unit 已有 `attackTarget` 或 `attackMoveDestination`，可以视为 reservation 被外部战斗行为打断并清理，避免和紧急防守状态长期冲突。
- 若 unit 有有效 reservation、没有 destination、没有 attack target / attack-move，且离目标超出对应占领半径，则重新 `setDestination(for:near:)`。

### 4. 接入 AI 分配和主攻筛选

- 在 `updateAI(dt:)` 的 enemy HQ 存活检查后调用 `maintainEnemyCaptureReservations()`。
- 分配 oil runner 成功后调用 `reserveEnemyCapture(unit: oilRunner, target: .oilDerrick(neutralOil.id))`。
- 分配 flag runner 成功后调用 `reserveEnemyCapture(unit: flagRunner, target: .controlPoint(targetFlag.id))`。
- `isAvailableEnemyCaptureUnit(...)` 排除 `isEnemyCaptureReserved(unit)`。
- `commandEnemyAttackers(...)` 的 idle attacker 筛选排除 `isEnemyCaptureReserved(unit)`。
- 可给 `enemyControlPointTarget(...)` 增加默认参数以便占旗分配跳过已保留目标，但默认调用仍保持原有目标排序；不要改变 `enemyWaveObjective()`、`tryEnemySupportPower()` 或 `aiBuildPattern()`。

### 5. 接入清理 / 重置

- `cullDestroyedEntities()` 移除死亡单位对应 reservation，并清理指向已死亡油井的 reservation。
- `restartSkirmish()` 清空 `enemyCaptureReservations`。
- 目标完成后下一次 `maintainEnemyCaptureReservations()` 应自动清理 reservation。

### 6. 文档更新

更新 `README.md`：

- Enemy AI bullet 改为说明占油 / 占旗 runner 会保留任务直到目标归 Red 或单位失效，避免主攻波次抢走正在占领的单位。

更新 `md/flow/flow.md`：

- AI 小节说明 reservation 的触发、保护范围、清理条件和 `SKRM` 重置。
- 明确 reservation 只影响 AI 自己的占点 runner，不改变玩家命令和占领规则。

更新 `md/flow/flowchart.md`：

- AI 节点加入“长期保留占点队”或类似短语。

- 禁止新增平行占领系统、AI 免费单位、外部依赖、HUD 调试面板或大规模 AI 重构。
