# v2.4（引入声呐浮标）

## 版本号

v2.4

## 版本分配依据

- 人工未指定本轮版本；Agent X 已完成 v2.3“引入机动防空车”闭环，并由 Agent C 下载最新 `origin/main` artifact 复核通过。
- `update_log.md` 和 `md/prompt/` 当前最新玩法版本为 v2.3。
- 本轮继续沿 `v2（地图控制）` 阶段推进，在静态防空、岸防炮和机动防空之后补齐一个保守的海岸侦测结构，增强潜艇侦测链路，因此递增为 v2.4。

## 背景

当前潜艇隐身依赖 `revealedUntil` 与声呐检测：

- Battleship / Submarine / Carrier / Helicopter 拥有 `hasSonar`。
- `isKnownToFaction(...)` 先遵守玩家战争迷雾，再对 Submarine 调用 `isSubmarineDetected(...)`。
- `SCAN` 可 8 秒区域侦察暴露潜艇；`AIRS` / `BARR` 真实命中潜艇会短暂暴露。

当前玩家和 AI 已有 Radar Outpost、Guard Tower、SAM Site、Coastal Battery 等建造层，但没有专门的海岸声呐设施。v2.4 做一个窄增量：新增 Sonar Buoy / 声呐浮标，作为可建造海岸侦测结构，完工后提供普通视野与声呐检测，不开火、不赚钱、不生产、不作为支援资产。

## 目标

1. 新增 `EntityKind.sonarBuoy`。
   - Display name：`Sonar Buoy`
   - Short code：`SON`
   - Domain：`.structure`
   - 推荐数值：cost `1150`，build time `9.0`，HP `540`，damage `0`，range `0`，cooldown 默认即可，vision `6`，footprint `44`
2. Sonar Buoy 可通过 `BASE` 建造轮换放置。
   - 要求玩家视野和己方 build coverage，继续复用结构 / 旗点覆盖规则。
   - 和 Shipyard / Coastal Battery 一样要求 `isCoastal(tile)`。
   - 不要求 oil field。
   - 仍要经过碰撞检查和旗点中心 no-build 规则。
3. Sonar Buoy 完工前不提供任何功能。
   - 未完工时不能提供视野、不能扩展 build coverage、不能提供 sonar、不能作为支援资产。
   - 保持现有施工、血条、死亡清理、迷雾隐藏、选择面板和 `SKRM` 重置链路。
4. Sonar Buoy 完工后提供声呐检测。
   - 将 `.sonarBuoy` 纳入 `hasSonar`，并在 `sonarRange(for:)` 给合理范围，建议 `300`。
   - 修改 `isSubmarineDetected(...)` 的 sensor 过滤，让结构类 sonar 必须 `isOperational` 才能生效；移动声呐单位保持原行为。
   - 不改变 `revealedUntil`、`SCAN`、`AIRS` / `BARR` 命中暴露链路。
   - 不绕过 `isKnownToFaction(...)` 的战争迷雾边界；玩家仍不能可靠看到不可见区域内的敌方潜艇。
5. Sonar Buoy 不攻击任何目标。
   - `damage` 为 `0`，`canAttack(...)` 不给 Sonar Buoy 任何攻击能力。
   - 不显示炮塔或 projectile。
   - 不计入支援技能资产；`SCAN` 仍只需要 operational HQ 或 Radar Outpost。
6. AI 可补建 Sonar Buoy。
   - 将 Sonar Buoy 纳入 `buildableStructures`，AI 可在己方结构或已占旗点覆盖附近寻找海岸位置。
   - AI 仍必须通过 `constructionIssue(...)`，不能绕过海岸、油田、陆地、碰撞或覆盖规则。
7. 战略价值和目标排序同步。
   - `strategicValue(...)`、`enemyStrategicAssaultScore(...)`、`primaryTarget(...)` 等目标排序纳入 Sonar Buoy，权重建议低于 Coastal Battery / SAM Site，高于普通 Humvee。
8. HUD / README / flow 文档同步。
   - pending construction panel 对 Sonar Buoy 显示 coast tile 需求。
   - 选择面板说明它是 sonar detector，不是 `SCAN` asset。
   - README、`md/flow/flow.md` 和必要的 `md/flow/flowchart.md` 记录当前真实行为。
9. 不更新 `update_log.md`。正式 v2.4 记录由 Agent C 在最新 `origin/main` artifact 验收通过后追加。

## 非目标

- 不新增反潜武器、反潜单位、科技树、升级系统、弹药系统、音效、图片素材或外部依赖。
- 不改变 Battleship、Submarine、Carrier、Helicopter、Radar Outpost、SAM Site、Coastal Battery 或 support power 的既有行为。
- 不让 Sonar Buoy 攻击潜艇、陆地、空中、海军或建筑目标。
- 不让 Sonar Buoy 触发潜艇 `revealedUntil`，不新增声呐 ping 主动暴露效果。
- 不把 Sonar Buoy 作为 `SCAN`、`BARR`、`AIRS` 或 `REPR` 资产。
- 不绕过战争迷雾；玩家不能可靠看到不可见区域内潜艇。
- 不修改 v2.0 旗点建造覆盖半径和 no-build 半径。
- 不拆分 `GameScene.swift`，不做大规模重构，不做全文件格式化。
- 不修改 Xcode project、CI workflow 或测试规范，除非编译必须且能说明原因。
- 除非人工明确要求，本轮不默认跑本机完整 Xcode build 或模拟器；构建结论以 GitHub Actions artifact 为准。

## 实现步骤

### 1. 开始前同步 main 并确认工作区归属

Agent B 必须先执行：

```sh
git fetch origin
git switch main
git pull --ff-only origin main
git status --short
git rev-parse HEAD
git rev-parse origin/main
```

要求：

- 当前分支必须是 `main`。
- `HEAD` 和 `origin/main` 必须是同步后的最新基线。
- 如果 `git status --short` 显示任何未提交、未跟踪或冲突文件，B 必须停止并确认归属。
- 如果唯一未跟踪文件是本提示词文件 `md/prompt/v2（地图控制）/v2.4（引入声呐浮标）.md`，B 必须明确确认它属于本轮 Agent A 产物后再继续。
- 如果没有 `origin`、没有网络、没有 push 权限或无法同步 `origin/main`，必须明确写出阻塞原因，不得伪装已完成。

### 2. 新增 EntityKind

在 `EntityKind` 中加入 `.sonarBuoy`，并补齐：

- `displayName`
- `shortCode`
- `domain`
- `cost`
- `buildTime`
- `maxHP`
- `attackRange`
- `damage`
- `visionTiles`
- `footprint`
- `hasSonar`
- `canAttack(...)`
- `strategicValue(...)`

注意：

- Sonar Buoy `damage` 应为 `0`，`canAttack(...)` 不需要新增可攻击目标。
- 不要让 Sonar Buoy 进入任何攻击、projectile 或潜艇暴露写入逻辑。

### 3. 纳入建造、地形、视觉和 HUD

- 将 `.sonarBuoy` 加入 `buildableStructures`，建议放在 Radar Outpost 后、防御建筑前后均可，但要保持 `BASE` 轮换可发现。
- `configureEntityNode(...)` 的结构 case 列表加入 `.sonarBuoy`。
- `addStructureBody(...)` 增加声呐浮标外观，用 SpriteKit shape 拼一个低矮岸基声呐基座 / 浮标天线 / 波纹标记，不新增素材。
- `constructionIssue(...)` 中让 `.sonarBuoy` 与 Shipyard / Coastal Battery 一样走 `isCoastal(tile)`。
- pending construction panel 中 Sonar Buoy 应显示 `Needs coast tile`。
- `structureStatusLine(...)` 给 Sonar Buoy 返回短句，例如 `Sonar detector  No SCAN`。

### 4. 同步声呐检测链路

- `hasSonar` 中加入 `.sonarBuoy`。
- `sonarRange(for:)` 中加入 `.sonarBuoy`，建议 `300`。
- `isSubmarineDetected(...)` 遍历 sensor 时新增结构 operational 过滤，例如结构必须 `sensor.isOperational`，移动单位保持原行为。
- 不改变 `isKnownToFaction(...)` 的玩家可见性前置判断。
- 不改变 `revealSupportTargets(...)`、`revealSubmarineHitBySupport(...)` 或 `fire(attacker:target:)` 对潜艇的 `revealedUntil` 写入。

### 5. 同步 AI 和目标排序

- `buildableStructures` 加入后，AI 缺失建筑补建会自然尝试 Sonar Buoy。
- `enemyConstructionPosition(for:)` 不需要新分支；它会通过 `constructionIssue(...)` 搜索海岸 tile。
- `enemyStrategicAssaultScore(...)` 对 `.sonarBuoy` 加侦测结构权重。
- `primaryTarget(for:)` priorities 加入 `.sonarBuoy`，建议在 Radar Outpost / 防御结构附近。
- `strategicValue(of:)` 给 `.sonarBuoy` 合理值，建议 `600` 左右。

### 6. 同步文档

更新 `README.md`：

- Base construction bullet 的建造列表加入 Sonar Buoy。
- Current Features 加一条 Sonar Buoy 说明：buildable coastal sonar detector，operational 后提供声呐侦测，不攻击、不作为 `SCAN` 资产。
- Building rules 或 Controls 的 `BASE` 轮换列表加入 Sonar Buoy。
- Enemy AI bullet 可写 rebuilds production, detection, and defense structures including Sonar Buoys，或保持简洁但要覆盖新行为。

更新 `md/flow/flow.md`：

- `EntityKind` 小节加入 Sonar Buoy 描述。
- 经济与生产 / 建造规则说明 Sonar Buoy 是海岸结构，仍需要视野与 build coverage。
- 迷雾与隐身小节说明 Sonar Buoy 完工后提供 sonar，未完工不生效，不写入 `revealedUntil`。
- AI 小节说明 AI 可补建 Sonar Buoy，仍走 `constructionIssue(...)`。
- 未来扩展点中如果仍把声呐设施当成未实现项目，改为 sonar 平衡 / 升级。

必要时更新 `md/flow/flowchart.md`：

不要更新 `update_log.md`，Agent C 验收通过后再追加正式记录。
