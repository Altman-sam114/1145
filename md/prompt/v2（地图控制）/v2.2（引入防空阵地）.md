# v2.2（引入防空阵地）

## 版本号

v2.2

## 版本分配依据

- 人工未指定本轮版本；Agent X 已完成 v2.1“引入岸防炮”闭环，并由 Agent C 记录验收。
- `update_log.md` 和 `md/prompt/` 当前最新玩法版本为 v2.1。
- 本轮继续沿 `v2（地图控制）` 阶段推进，在 Guard Tower 的陆空泛防、Coastal Battery 的海岸反舰之后补齐专门静态防空，因此递增为 v2.2。

## 背景

当前可建防御结构包括 Guard Tower 和 Coastal Battery：

- Guard Tower 完工后攻击可见 land / air / structure，不攻击 naval / submarine。
- Coastal Battery 完工后只攻击可见水面海军目标，不攻击潜艇、不提供 sonar。
- Radar Outpost 提供大范围视野和 `SCAN` 资产，不开火。

空军当前有 Helicopter / Fighter，AI 会生产混合陆海空单位。玩家与 AI 缺少专门的地面防空结构，导致前线旗点扩张后的防御选择仍偏向泛用炮塔和海岸反舰。v2.2 做一个保守增量：新增 SAM Site / 防空阵地，作为可建造、可攻击、不可生产、不可支援资产的陆地防空结构，只在完工后攻击可见空中目标。

## 目标

1. 新增 `EntityKind.samSite`。
   - Display name：`SAM Site`
   - Short code：`SAM`
   - Domain：`.structure`
   - 推荐数值：cost `1650`，build time `12.0`，HP `760`，range `260`，damage `48`，cooldown `1.55`，vision `7`，footprint `48`
2. SAM Site 可通过 `BASE` 建造轮换放置。
   - 要求玩家视野和己方 build coverage，继续复用 v2.0 的结构 / 旗点覆盖规则。
   - 作为普通陆地结构，必须放在 `isLand(tile)` 的 clear land 上。
   - 不要求 coast tile，不要求 oil field。
   - 仍要经过碰撞检查和旗点中心 no-build 规则。
3. SAM Site 完工前不提供任何功能。
   - 未完工时不能开火、不能提供视野、不能扩展 build coverage。
   - 复用现有施工、血条、死亡清理、迷雾隐藏、选择面板和 `SKRM` 重置链路。
4. SAM Site 完工后只攻击可见空中目标。
   - 可攻击 Helicopter / Fighter 等 air 目标。
   - 不攻击 land / naval / submarine / structure。
   - 不提供 sonar，不改变潜艇 `revealedUntil` / 声呐检测链路。
   - 仍通过 `isKnownToFaction(...)` 找目标，不能攻击迷雾中不可见目标。
5. AI 可补建 SAM Site。
   - 将 SAM Site 纳入 `buildableStructures`，AI 可在已有结构或已占旗点覆盖附近寻找普通陆地位置。
   - AI 仍必须通过 `constructionIssue(...)`，不能绕过玩家视野以外的共享规则、海岸、油田、陆地、碰撞或覆盖限制。
6. 战斗和目标评估同步。
   - `strategicValue(...)`、`enemyStrategicAssaultScore(...)`、`primaryTarget(...)` 等目标排序要纳入 SAM Site。
   - 玩家与 AI 共享 `canAttack(...)`、`updateCombat(...)`、`fire(...)` 和目标搜索链路。
7. HUD / README / flow 文档同步。
   - pending construction panel 对 SAM Site 显示普通 base / flag coverage 与 visible ground 需求。
   - 选择面板说明它是 anti-air defense。
   - README、`md/flow/flow.md` 和必要的 `md/flow/flowchart.md` 记录当前真实行为。
8. 不更新 `update_log.md`。正式 v2.2 记录由 Agent C 在最新 `origin/main` artifact 验收通过后追加。

## 非目标

- 不新增科技树、升级系统、导弹弹药、资源类型、音效、图片素材或外部依赖。
- 不改变 Guard Tower、Coastal Battery、Radar Outpost、Shipyard、Carrier、Helicopter、Fighter、Submarine 或 support power 的既有行为。
- 不让 SAM Site 攻击潜艇、提供声呐、触发潜艇暴露或绕过 `isSubmarineDetected(...)`。
- 不让 SAM Site 攻击陆地、海军或建筑目标。
- 不把 SAM Site 作为 `SCAN`、`BARR`、`AIRS` 或 `REPR` 资产。
- 不让未完工 SAM Site 开火、生产、赚钱、提供视野、提供 build coverage 或支援资产。
- 不改变 v2.0 旗点建造覆盖半径和 no-build 半径。
- 不拆分 `GameScene.swift`，不做大规模重构，不做全文件格式化。
- 不修改 Xcode project、CI workflow 或测试规范，除非编译必须且能说明原因。
- 除非人工明确要求，本轮不默认跑本机完整 Xcode build 或模拟器；构建结论以 GitHub Actions artifact 为准。

- `strategicValue(...)`、`enemyStrategicAssaultScore(...)`、`primaryTarget(...)` 影响 AI 和目标排序。

## 实现步骤

### 1. 开始前同步 main 并确认工作区归属

Agent B 必须先执行：

```sh

```

要求：

- 当前分支必须是 `main`。
- `HEAD` 和 `origin/main` 必须是同步后的最新基线。
- 如果 `git status --short` 显示任何未提交、未跟踪或冲突文件，B 必须停止并确认归属。
- 如果唯一未跟踪文件是本提示词文件 `md/prompt/v2（地图控制）/v2.2（引入防空阵地）.md`，B 必须明确确认它属于本轮 Agent A 产物后再继续。
- 如果没有 `origin`、没有网络、没有 push 权限或无法同步 `origin/main`，必须明确写出阻塞原因，不得伪装已完成。

### 2. 新增 EntityKind

在 `EntityKind` 中加入 `.samSite`，并补齐：

- `displayName`
- `shortCode`
- `domain`
- `cost`
- `buildTime`
- `maxHP`
- `attackRange`
- `damage`
- `attackCooldown`
- `visionTiles`
- `footprint`
- `canAttack(...)`
- `strategicValue(...)`

`canAttack(...)` 推荐：

```swift
case .samSite:
    return target.domain == .air
```

注意：

- 不要把 `.samSite` 加入顶部 `if target == .submarine { ... }` 的反潜列表。
- 不要让 SAM Site 攻击 `target.isStructure`。

### 3. 纳入建造、地形、视觉和 HUD

- 将 `.samSite` 加入 `buildableStructures`，建议放在 Guard Tower 后、Coastal Battery 前或后，保持 `BASE` 轮换可发现。
- `configureEntityNode(...)` 的结构 case 列表加入 `.samSite`。
- `addStructureBody(...)` 增加防空阵地外观，用 SpriteKit shape 拼一个低矮发射台 / 雷达小盘 / 斜置导弹，不新增素材。
- `constructionIssue(...)` 不新增特殊分支；SAM Site 应走普通 `isLand(tile)` 分支。
- pending construction panel 中 SAM Site 应显示 `Needs base/flag coverage` 和 `Requires visible ground`。
- `structureStatusLine(...)` 给 SAM Site 返回短句，例如 `Anti-air defense`。
- `showProjectile(...)` / `projectileColor(for:)` 可让 `.samSite` 使用更亮的导弹 tracer；如只复用默认红色也可接受。

### 4. 同步 AI 和目标排序

- `buildableStructures` 加入后，AI 缺失建筑补建会自然尝试 SAM Site。
- `enemyConstructionPosition(for:)` 不需要新分支；它会通过 `constructionIssue(...)` 搜索普通陆地 tile。
- `enemyStrategicAssaultScore(...)` 对 `.samSite` 加防御结构权重。
- `primaryTarget(for:)` priorities 加入 `.samSite`，建议在 `.guardTower` / `.coastalBattery` 附近。
- `strategicValue(of:)` 给 `.samSite` 合理值，建议接近 Guard Tower / Coastal Battery。
- 可在 `targetScore(for:target:)` 中给攻击者打击 SAM Site 轻量加分，但不是必须；不要让这改变目标合法性。

- Enemy AI bullet 可写 rebuilds production and defense structures including SAM Sites and Coastal Batteries，或保持简洁但要覆盖新行为。

- 战斗小节说明 SAM Site 共享攻击链路，只攻击可见空中目标，不攻击潜艇 / 不提供 sonar。
- AI 小节说明 AI 可补建 SAM Site，仍走 `constructionIssue(...)`。
