# v2.9（旗点占领奖励）

## 版本号

v2.9

## 版本分配依据

- 人工未指定本轮版本；`update_log.md` 和 `md/prompt/` 当前最新正式玩法版本为 v2.8“AI 优先反夺旗点”。
- v2.6 到 v2.8 已连续增强 AI 占旗、守旗和反夺旗点行为。
- 本轮继续沿着地图控制做一个小型经济反馈：占领前线旗点时给新归属方一次性资金奖励，让旗点争夺更有 RTS 节奏，因此递增为 v2.9。

## 背景

当前前线旗点已经提供持续收入、视野和有限 build coverage，并影响任务阶段；AI 也已经会保留占旗 runner、响应玩家抢点、优先反夺玩家旗点。但旗点完成占领时，除持续收入外没有即时经济反馈。玩家和 AI 在攻防中短时间抢回旗点时，收益感不够明显。

v2.9 只增加一次性资金奖励，不改变旗点持续收入、占领速度、视野、建造覆盖或 AI 行为逻辑。

## 目标

1. 新增旗点占领一次性奖金。
   - 建议新增常量 `controlPointCaptureBonus`，金额保持保守，例如 `$260` 左右。
   - 奖励只在旗点归属实际变化时发放。
   - 新归属方为 player 或 enemy 时都通过同一 `changeMoney(for:by:)` 入口获得奖励。
2. 玩家反馈同步。
   - 玩家成功占领旗点时，现有 “flag captured” 消息应包含 `+$<bonus>`。
   - 不新增 HUD 按钮、面板或调试 UI。
3. 保持 AI 和玩家共享规则。
   - AI 占旗成功也获得同额奖励，直接进入 `enemyMoney`，受既有 AI 生产消费逻辑使用。
   - 不给 AI 独立倍率，也不让 AI 绕过 `changeMoney(...)`。
4. 保持既有地图控制规则。
   - 不改变旗点每秒收入、占领半径、占领速度、视野、build coverage、任务阶段或胜负条件。
   - 不改变 AI 占点 reservation、旗点防守响应、反夺旗点优先级、AI 生产、支援技能或玩家命令。
   - `SKRM` 仍通过重置 `playerMoney` / `enemyMoney` 自然清空奖金影响，不需要新增持久状态。
5. 文档同步。
   - README Economy loop 或 Enemy AI/Current Features 中同步旗点占领会提供一次性资金奖励。
   - `md/flow/flow.md` 经济与占领小节同步奖金发放位置和共享经济入口。
   - `md/flow/flowchart.md` 如需要，经济 / 占领节点加入“旗点奖金”。

## 非目标

- 不新增单位、建筑、资源类型、科技树、升级、UI 或外部素材。
- 不改变 Oil Derrick 占领奖励；本轮只做前线旗点。
- 不改变持续收入数值，尤其不改 `incomePerTick(for:)` 中旗点每秒收入。
- 不改变 AI 难度收入加成、AI 生产队列、AI 目标选择或玩家命令。
- 不修改 Xcode project、CI workflow 或测试规范，除非编译必须且能说明原因。
- 按人工最新要求，本轮不得运行本地测试、本地构建或本地静态检查作为验证；验收只认 GitHub Actions artifact。

关键源码事实：

- `updateControlPointCapture(dt:)` 在旗点进度达到 4.0 后调用 `setControlPointFaction(...)`。
- `setControlPointFaction(_:,for:)` 当前负责保存旧归属、设置新归属、清理占领进度、更新视觉，并在玩家相关归属变化时刷新 fog。
- `changeMoney(for:by:)` 是玩家和 AI 共用的资金入口，支持 `.player`、`.enemy` 和 `.neutral`。
- `incomePerTick(for:)` 已给每个己方旗点提供 `$95/s` 持续收入；本轮不改变该持续收入。
- `restartSkirmish()` 重置 `playerMoney` 和 `enemyMoney`，不需要为一次性奖金新增重置字段。

## 实现步骤

### 1. 开始前同步 main 并确认工作区归属

Agent B 必须先执行：

```sh
git fetch origin
git switch main
git pull --ff-only origin main
git status --short
git rev-parse HEAD
git rev-parse origin/main
```

要求：

- 当前分支必须是 `main`。
- `HEAD` 和 `origin/main` 必须是同步后的最新基线。
- 如果 `git status --short` 显示任何未提交、未跟踪或冲突文件，B 必须停止并确认归属。
- 本提示词文件 `md/prompt/v2（地图控制）/v2.9（旗点占领奖励）.md` 属于本轮 Agent A 产物。

### 2. 新增奖金常量

- 在 `GameScene` 的旗点常量附近新增 `private let controlPointCaptureBonus = 260` 或同等级保守金额。
- 不把金额散落在多处硬编码。

### 3. 在归属变化处发奖

- 在 `setControlPointFaction(_:,for:)` 中，先保存 `previousFaction`。
- 只有 `previousFaction != faction` 时才调用奖金 helper 或直接 `changeMoney(for: faction, by: controlPointCaptureBonus)`。
- `.neutral` 不应获得奖金；当前调用方只传 player/enemy，但实现应保持安全。
- 玩家成功占旗的 `showMessage(...)` 文案改为包含 `+$\(controlPointCaptureBonus)`。

### 4. 文档更新

- README 当前功能列表同步旗点提供持续收入、视野、build coverage 和一次性占领奖金。
- `md/flow/flow.md` 经济与占领流程同步 `setControlPointFaction(...)` 发放一次性奖金，并说明玩家/AI 共享 `changeMoney(...)`。
- `md/flow/flowchart.md` 经济 / 占领节点可加入“旗点奖金”。
- 不更新 `update_log.md`。正式 v2.9 记录由 Agent C 在最新 `origin/main` artifact 验收通过后追加。

## 验收标准

- 旗点归属实际变化时，新归属方获得一次性资金奖励。
- 玩家成功占旗消息显示奖金金额。
- AI 成功占旗也通过 `changeMoney(for: .enemy, by:)` 获得同额奖励。
- 重复调用同一归属不会重复发奖。
- 持续收入、占领速度、视野、build coverage、任务阶段、AI 占点 reservation、AI 防守响应和 AI 反夺优先级不变。
- README、flow、flowchart 与真实代码行为一致。
- 最新 `origin/main` GitHub Actions artifact 通过，manifest / run / commit 对应一致。

## 风险和禁止项

- 风险：奖金过高会放大滚雪球；金额必须保守，不改变持续收入数值。
- 风险：如果在 `updateControlPointCapture(dt:)` 直接散落加钱，未来其他归属入口可能遗漏；优先放在 `setControlPointFaction(...)`。
- 禁止新增长期状态或平行经济系统。
- 禁止伪造本地测试、本地 build 或云端 artifact 结果。
- 禁止引入长期分支、PR 流或非 `main` 验证路径。
