# v2.0（旗点建造覆盖）

## 版本号

v2.0

## 版本分配依据

- 人工未指定本轮版本；Agent X 已确认 v1.9“AI 占点队保留任务”闭环完成并由 Agent C 追加记录。

- 本轮从核心单位 / AI 小修进入“地图控制 -> 前线建造”规则增强，是更接近 Desert Stormfront 节奏的重要地图控制阶段，因此新开 `v2（地图控制）` 阶段并使用 v2.0。

## 背景

当前前线旗点已经提供收入、任务进度和玩家视野。建筑放置仍只依赖己方已完工结构的基地覆盖：

- `constructionIssue(for:faction:tile:position:)` 会在玩家侧先要求目标位置可见。
- 随后调用 `hasNearbyStructure(position, faction:)`，要求 390 像素内存在己方存活、已完工结构。
- `Shipyard` 仍额外要求 coast tile。
- `Oil Derrick` 仍额外要求 oil field，并可复用 / 改建已有中立或敌方油井实体。
- `enemyConstructionPosition(for:)` 当前只围绕 Red 已完工结构搜索新建筑位置。

这导致旗点的地图控制价值主要是收入和视野，无法作为前线建造锚点。v2.0 要做一个保守增量：己方已占领旗点提供有限建造覆盖，玩家和 AI 共享同一合法性规则，让前线推进能自然衍生 Radar Outpost / Guard Tower / Shipyard / Oil Derrick 等扩张选择。

## 目标

1. 已占领旗点为对应 faction 提供有限建造覆盖。
   - 推荐覆盖半径：180 像素。
   - 该半径必须明显小于现有已完工结构覆盖半径 390，体现“有限前线覆盖”，避免抢下一个旗点后立即跨越到下一个旗点中心。
2. 玩家和 AI 使用同一建造覆盖判定：
   - 己方存活、已完工结构仍提供 390 像素覆盖。
   - 己方已占领旗点提供 180 像素覆盖。
   - 中立旗点、敌方旗点不提供本 faction 覆盖。
3. 玩家建筑放置仍必须满足玩家视野：
   - 旗点本身已在当前系统中提供玩家视野；本轮不绕过 `isVisible(point:)`。
4. 保持所有地形和碰撞规则不变：
   - `Shipyard` 仍必须放在 coast tile。
   - `Oil Derrick` 仍必须放在 oil field。
   - 普通陆地结构仍必须放在 clear land。
   - `isOccupiedForConstruction(...)` 仍必须阻止重叠。
   - 新增旗点中心 no-build 小半径，避免建筑压住旗点视觉节点。
5. AI 建造选点也要能使用已占领旗点作为前线锚点：
   - `enemyConstructionPosition(for:)` 搜索 anchor 时应包含 Red 已占领旗点位置。
   - AI 仍必须通过 `constructionIssue(...)` 最终合法性判断。
6. HUD / README / flow 文档要同步当前事实：
   - 玩家 build panel 文案从只写 base coverage 调整为 base / flag coverage。
   - README 的建筑规则、旗点能力和 AI 描述同步。
   - `md/flow/flow.md` 和必要的 `md/flow/flowchart.md` 同步。
7. 不更新 `update_log.md`。正式 v2.0 记录由 Agent C 在最新 `origin/main` artifact 验收通过后追加。

## 非目标

- 不允许中立旗点或敌方旗点给玩家 / AI 提供建造覆盖。
- 不让未占领旗点、正在被占领但未完成的旗点提供覆盖。
- 不让旗点直接生产、赚钱之外额外提供支援资产、开火、维修或声呐。
- 不改变建筑施工边界：未完工建筑仍不得生产、赚钱、提供视野、提供支援资产、扩展基地覆盖或开火。
- 不改变玩家支援技能、潜艇隐身、老兵经验、控制组、AI 生产、AI 攻击波次、经济公式、胜负条件或 `SKRM` 重置。

### 2. 明确建造覆盖半径常量

推荐在 `GameScene` 常量区新增：

```swift
private let structureBuildCoverageRadius: CGFloat = 390
private let controlPointBuildCoverageRadius: CGFloat = 180
private let controlPointNoBuildRadius: CGFloat = 56
```

并用 `structureBuildCoverageRadius` 替换 `hasNearbyStructure(...)` 中的裸 `390`。

要求：

- 不改变结构覆盖实际数值。
- 旗点覆盖半径必须明显小于结构覆盖半径。
- no-build 半径只用于阻止建筑压住旗点中心，不应影响旗点覆盖半径。

### 3. 将覆盖判定从“附近结构”升级为“附近建造覆盖”

推荐把 `hasNearbyStructure(_:, faction:)` 改名或包一层为：

```swift
private func hasNearbyBuildCoverage(_ position: CGPoint, faction: Faction) -> Bool {
    hasNearbyStructure(position, faction: faction) ||
        hasNearbyControlledFlag(position, faction: faction)
}
```

也可以直接扩展原函数，但推荐保留清晰语义：

```swift
private func hasNearbyStructure(_ position: CGPoint, faction: Faction) -> Bool { ... }

private func hasNearbyControlledFlag(_ position: CGPoint, faction: Faction) -> Bool {
    controlPoints.contains { point in
        point.faction == faction &&
            point.node.position.distance(to: position) <= controlPointBuildCoverageRadius
    }
}
```

然后 `constructionIssue(...)` 使用 `hasNearbyBuildCoverage(...)`，失败文案改为：

```swift
"Build within base or captured flag radius."
```

要求：

- `.neutral` 旗点不得给任何可建 faction 提供覆盖，除非 faction 本身是 `.neutral`；本项目不应为 neutral 建造，正常无影响。
- 已占领旗点不需要 `isOperational`，因为它不是结构。
- 不直接修改 `visibleTiles` / `exploredTiles`。

### 4. 防止建筑压住旗点中心，并在占旗后刷新 fog

在 `isOccupiedForConstruction(_:, placing:)` 中保留现有实体碰撞逻辑，并额外拒绝距离任意控制点中心过近的建筑位置：

```swift
if controlPoints.contains(where: { $0.node.position.distance(to: position) < controlPointNoBuildRadius }) {
    return true
}
```

要求：

- no-build 规则适用于玩家和 AI，因为它通过 `constructionIssue(...)` 共享。
- 不改变油井复用特殊逻辑：`Oil Derrick` 对已有油井实体的例外仍保留。
- 半径要小，只保护旗点视觉和交互中心，不应吃掉整个旗点覆盖范围。

在 `setControlPointFaction(_:, for:)` 中，当旗点阵营变化涉及玩家时，调用 `updateFog(force: true)`，让玩家刚占旗后立即获得旗点视野并能使用旗点覆盖做合法预览 / 放置。推荐：

```swift
if faction == .player || point.faction == .player {
    updateFog(force: true)
}
```

注意需要在覆盖旧 faction 前保存旧值，避免判断丢失。

### 5. 让 AI 建造 anchor 包含 Red 已占领旗点

推荐新增：

```swift
private func buildCoverageAnchors(for faction: Faction) -> [CGPoint] {
    let structureAnchors = entities.values
        .filter { $0.faction == faction && $0.isAlive && $0.kind.isStructure && $0.isOperational }
        .map(\.node.position)
    let flagAnchors = controlPoints
        .filter { $0.faction == faction }
        .map(\.node.position)
    return structureAnchors + flagAnchors
}
```

然后 `enemyConstructionPosition(for:)` 的非 Oil Derrick anchors 改为 `buildCoverageAnchors(for: .enemy)`，按距离 `enemyBaseAnchorPoint()` 排序后搜索。

要求：

- AI 仍通过 `constructionIssue(...)` 做最终合法性判断。
- Oil Derrick 的固定候选逻辑可以保持不变；如果 oil tile 在 Red 已占旗点覆盖内，则 `constructionIssue(...)` 会自然允许，否则仍拒绝。
- 不让 AI 绕过 coast / oil / land / collision 限制。

## 风险和禁止项

- 风险：旗点覆盖半径过大会让扩张过快；本轮使用 180 像素且明显小于结构覆盖 390。
- 风险：如果不加旗点中心 no-build，建筑可能压住旗点视觉节点；本轮必须保留小半径中心保护。
- 风险：如果占旗后不刷新 fog，玩家可能短暂遇到“已占旗但还不能建”的体验延迟；本轮应在玩家相关阵营变化后强制刷新 fog。
- 风险：如果 AI anchor 只加旗点但不走 `constructionIssue(...)`，可能绕过 coast / oil / collision；必须保留最终合法性检查。
- 风险：如果玩家视野要求被移除，会破坏战争迷雾；本轮禁止绕过 `isVisible(point:)`。
- 风险：如果未占领或中立旗点提供覆盖，会削弱占领价值；必须只用对应 faction 已占领旗点。
- 风险：如果未完工结构被误算覆盖，会破坏施工边界；本轮不得改变未完工结构规则。
- 禁止伪造测试结果、隐藏失败命令、把本地轻量检查说成云端构建通过。
- 禁止让 Agent C 只看 Agent B 汇报；必须核对最新 artifact。
- 禁止把旧 run、旧 artifact 或本地输出冒充本轮云端结果。
- 禁止使用非 `Altman-sam114` 的 GitHub 账号伪装完成 push、CI 或 artifact 验收。
